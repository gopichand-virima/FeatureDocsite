# Accessing the Imported Assets

1. From the left navigation panel, navigate to [Discovery Scan]{.Strong}. Click [Imported Assets.]{.Strong}

 ::: step-media
 ![Imported Assets 551x249](../../Resources/Images/Discovery/Imported-Assets/Imported Assets_551x249.png)
 
2. The Imported Assets page opens.

 ::: step-media
 ![Imported Assets 1](../../Resources/Images/Discovery/Imported-Assets/Imported Assets_1.png)
 
[Column Details:]{.Strong}

Displays imported asset details in tabular format. Each row represents a imported asset with columns such as:

- [Blueprint]{.Strong}: The type of asset (e.g., Windows Server, Linux Server, IIS Server).

- [CI]{.Strong}: Status of the configuration item (e.g., *New*, or linked CI ID such as *AST004534*).

 - [New]{.Strong}: Asset imported into the system, not yet linked to a CMDB record.

 - [Linked CI]{.Strong}: Displays a unique CI ID (e.g., AST004534). The asset is linked to an existing Configuration Item in the CMDB.

- [Host Name]{.Strong}: The hostname of the device or server.

- [IP Address]{.Strong}: The IP address of the asset.

- [Operating System]{.Strong}: The Operating System of the asset.

- [Location]{.Strong}: The specified location of the asset.

- [Re-scan]{.Strong}[ icon -- Opens the rescan dialog box where you can specify IP address, client, and type of scan to revalidate asset information.]{.inline-comment-marker}

 - [A new scan run will be listed on the Recent Scans page; refresh the page to view the latest status updates.]{.inline-comment-marker}

 - Scanned results will be displayed in Discovered items. See discovered items PRD for rest of the flow.

 - Record in Imported Assets will not be updated.

- [Records per Page Dropdown]{.Strong}: Allows users to select how many records are displayed at once (e.g., 10, 25, 100).

# Select Actions Dropdown

The [Select Actions]{.Strong} menu provides administrators with operations they can perform on imported assets:

- [Delete]{.Strong} -- Removes the selected asset record(s) from the Imported Assets list.

- [Move to CMDB]{.Strong} -- To move the asset to the Configuration Management Database after confirmation.

- [Re-scan]{.Strong} -- Opens the rescan dialog box where you can specify IP address, client, and type of scan to revalidate asset information.
