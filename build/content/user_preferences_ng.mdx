# User Notification Preferences

Use this function to manage user-level notification preferences, allowing users to control how and when they receive notifications.

User preferences enable personalization of notification delivery while ensuring critical alerts are always received.

In the main window, click **Admin > Notification Management > User Preferences**. The User Preferences window displays.

## Overview

User notification preferences control:

- Preferred notification channels
- Notification frequency and timing
- Do-not-disturb hours
- Email digest options
- Specific notification types to receive
- Mobile device registration
- Opt-out preferences

## Configure Default Preferences

Set organization-wide default preferences:

Click **Configure Defaults**.

### Default Channel Preferences

**Primary Channel**: Email

**Secondary Channel**: In-App

**Enable Channels**:

- ☑ Email notifications
- ☑ In-app notifications
- ☐ SMS notifications (opt-in only)
- ☐ Push notifications (opt-in only)
- ☐ Slack messages
- ☐ Teams messages

### Default Notification Types

Configure which notifications users receive by default:

**Ticket Notifications**:

- ☑ Assigned to me
- ☑ High priority mentions
- ☐ All ticket updates
- ☑ Tickets I created (status changes)
- ☑ Tickets I'm watching
- ☐ Team queue updates

**SLA Notifications**:

- ☑ SLA warnings (75% threshold)
- ☑ SLA breach alerts
- ☐ Daily SLA summary

**Approval Notifications**:

- ☑ Approval requests for me
- ☑ Approval decisions on my requests
- ☐ All team approvals

**System Notifications**:

- ☑ Critical system alerts
- ☑ Scheduled maintenance
- ☐ System announcements
- ☐ Feature updates

Click **Save Defaults**.

## Manage Individual User Preferences

View and edit preferences for specific users:

### Search Users

Search for user by:

- Name
- Email
- Department
- Role

Click on a user to view their preferences.

### Edit User Preferences

**Channel Preferences**:

View user's selected channels:

- Email: ✓ Enabled
- SMS: ✗ Disabled
- Push: ✓ Enabled (2 devices registered)
- In-App: ✓ Enabled

**Notification Schedule**:

- **Business Hours**: Monday-Friday, 8 AM - 6 PM
- **After Hours**: Emergency only
- **Weekends**: Critical only

**Do Not Disturb**:

- Enable DND: ☑
- DND Hours: 10 PM - 7 AM daily
- DND Days: Sundays
- Override for Critical: ☑ Allow P1 alerts

**Digest Settings**:

- Daily digest: ☑ Enabled
- Digest time: 8:00 AM
- Include: Assigned tickets, team summary
- Weekly digest: ☑ Enabled
- Weekly digest day: Monday

Make changes and click **Save**.

### Select Users

## User Self-Service

Allow users to manage their own preferences:

**Self-Service Portal**:

Navigate to **My Profile > Notification Preferences**

Users can:

- Select preferred channels
- Set DND hours
- Configure digest options
- Register mobile devices
- Opt-out of non-critical notifications
- Test notification delivery

**Restrictions**:

Administrators can lock certain settings:

- Cannot disable critical alerts
- Cannot disable SLA notifications
- Cannot exceed frequency limits
- Must have at least one channel enabled
