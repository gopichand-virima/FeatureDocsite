# Go to Page

Navigate directly to a specific page in paginated lists and reports.

## Overview

The Go to Page function allows you to quickly jump to any page in a multi-page list or report without clicking through sequential pages. This is especially useful when working with large datasets.

## Using Go to Page

### Direct Page Navigation
1. Locate the pagination controls at the bottom of the list
2. Click the **Go to Page** field or icon
3. Enter the desired page number
4. Press **Enter** or click **Go**
5. The list jumps to the specified page

### Quick Jump Options
- **First Page**: Click the first page icon
- **Previous Page**: Navigate back one page
- **Next Page**: Move forward one page
- **Last Page**: Jump to the final page
- **Specific Page**: Enter page number directly

## Pagination Information

### Page Display
The pagination area shows:
- Current page number
- Total number of pages
- Record range displayed (e.g., "Showing 1-25 of 500")
- Total record count

### Navigation Controls
Standard pagination controls include:
- ⏮️ First page
- ◀️ Previous page
- Page number input field
- ▶️ Next page
- ⏭️ Last page

## Items per Page

### Adjust Page Size
Change how many items display per page:
1. Click the **Items per Page** dropdown
2. Select from available options (25, 50, 100, 200)
3. View updates to show selected number of items

### Common Page Sizes
- **25 items**: Quick browsing
- **50 items**: Standard view
- **100 items**: Detailed review
- **200 items**: Maximum view (may impact performance)

## Keyboard Navigation

### Keyboard Shortcuts
- **Page Down**: Next page
- **Page Up**: Previous page
- **Home**: First page
- **End**: Last page
- **Ctrl + G**: Focus on "Go to Page" field

## URL Parameters

### Bookmarking Pages
- Page numbers are included in URL
- Bookmark specific pages for quick access
- Share URLs with specific page views
- Back button returns to previous pages

### URL Format
Example: `https://virima.com/list?page=5&pageSize=50`

## Page State Persistence

### Remember Page Location
- System remembers your current page
- Returns to same page after viewing details
- Maintains page position when navigating back
- Resets to page 1 when filters change

## Performance Considerations

### Large Datasets
When working with many pages:
- Use filters to reduce total pages
- Increase items per page for fewer total pages
- Use search to find specific records
- Consider export for offline analysis

### Loading Time
- Larger page sizes take longer to load
- Jumping to distant pages may have delay
- Network speed affects load time
- Consider pagination strategy for performance

## Best Practices

- Use filters before pagination to reduce results
- Adjust items per page based on screen size
- Bookmark frequently accessed pages
- Use direct page numbers for quick navigation
- Combine with sorting for better data organization
- Consider export for extensive data analysis
- Use search for finding specific records

## Accessibility

### Screen Reader Support
- Page numbers announced by screen readers
- Keyboard navigation fully supported
- ARIA labels for navigation controls
- Focus indicators for current position

## Troubleshooting

### Common Issues
- **Page not loading**: Check network connection
- **Invalid page number**: Ensure number is within valid range
- **Lost page position**: Refresh or reapply filters
- **Slow navigation**: Reduce items per page
