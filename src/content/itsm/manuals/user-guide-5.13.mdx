# ITSM - User Manual

Complete guide for IT Service Management in Virima 5.13.

## Incident Management

### Creating Incidents
Incidents can be created via:
- Manual entry through the web interface
- Email-to-ticket conversion
- Self-service portal
- API integration
- Phone call logging

### Incident Lifecycle
1. **New** - Initial creation
2. **Assigned** - Routed to appropriate team
3. **In Progress** - Being worked on
4. **Pending** - Waiting for user/vendor response
5. **Resolved** - Solution implemented
6. **Closed** - Verified and closed

## Problem Management

### Problem Identification
- Trend analysis from multiple incidents
- Proactive monitoring alerts
- User reports

### Root Cause Analysis
- Investigation and diagnosis
- Documentation of findings
- Known error creation

## Change Management

### Change Types
- **Standard Changes**: Pre-approved, low-risk
- **Normal Changes**: Require approval process
- **Emergency Changes**: Expedited approval for critical issues

### Change Advisory Board (CAB)
- Schedule regular CAB meetings
- Review pending changes
- Assess risks and impacts
- Approve or reject changes

## Best Practices

- Categorize tickets accurately for better reporting
- Document all actions in ticket notes
- Link related incidents and problems
- Keep knowledge base updated
- Monitor and optimize SLA compliance
- Regular workflow reviews and improvements
