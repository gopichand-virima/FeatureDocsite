# Normalization

Use this function to configure data normalization rules for discovered assets.

Normalization ensures that data collected from various sources is standardized and consistent across your CMDB. This includes standardizing manufacturer names, product names, software titles, and other attributes.

In the main window, click **Admin > Discovery > Normalization**. The Normalization window displays.

## Normalization Rules

The system includes pre-configured normalization rules for common manufacturers and products. You can view, edit, or add new rules.

### New Normalization Rule

From the *Select Actions* drop-down list, choose **New Normalization Rule**. The New Normalization Rule window displays.

Select the **Rule Type**:

- Manufacturer Normalization
- Product Normalization
- Software Title Normalization
- Custom Field Normalization

Enter the **Source Value** (the raw data as discovered).

Enter the **Normalized Value** (the standardized value to use).

Specify any additional matching criteria or patterns.

When all selections/entries are made, click **Add**.

### Edit Normalization Rule

Navigate to the window containing the record to edit.

Click the line containing the record. The applicable window displays.

Make the necessary modifications.

Click **Save**.

### Delete Normalization Rule

Click the line that contains the item to delete.

From the *Select Actions* drop-down list, choose **Delete**.

## Apply Normalization

To apply normalization rules to existing data:

Click **Apply Normalization** from the *Select Actions* menu.

Select the scope (all devices, specific device types, or date range).

Click **Execute**.
