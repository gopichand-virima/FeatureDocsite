# Assessing Import Meraki Window

1.  In the navigation pane, select <span class="Strong">Discovery Scan › Import from Meraki.</span>

    <span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Import-From-Meraki/Import%20Meraki%20Window.png" original-image-title=""></span>

<!-- -->

2.  The <span class="Strong">Import From Meraki</span> window opens and displays list of imported records. Each row represents a single import run.

    <span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Import-From-Meraki/Import%20Meraki%20Window_1.png" original-image-title=""></span>

- <span class="Strong">Key columns:</span>

  - <span class="Strong">Id</span>: Unique identification ID of the import run (e.g., Meraki005553), records imported from Meraki will have the prefix “Meraki” followed by a six-digit sequential number.

  - <span class="Strong">Name:</span>

    - Manual Import: The user provided name used to identify the import run.

    - Scheduled imports: The user-provided schedule name appears under the "Name" column.

  - <span class="Strong">Status</span>: Current state (e.g., Importing, Completed, Failed).

    - <span class="Strong">Importing</span>: The import process from Meraki is currently running and in progress.

    - <span class="Strong">Adding Data</span>: The imported information from Meraki is currently being processed and added to your system.

    - <span class="Strong">Completed:</span> The import from Meraki has finished successfully.

    - <span class="Strong">Failed: </span>The import has failed due to some unknown errors

      - View failed details in Meraki Logs

    - <span class="Strong">No Data Found</span>: The import completed, but no data was found to import.

  - <span class="Strong">Imported On</span>: The date and time when the import was completed.

  - <span class="Strong">Imported By</span>:

    - Manual Import: The logged-in user who initiated a manual import.

    - Scheduled Import: The logged-in user who configured scheduled import.

  - <span class="Strong">Credential</span>: Virima credential profile used to connect to Meraki.

  - <span class="Strong">Select Actions</span>: Perform bulk operations on selected or filtered rows.

- From the <span class="Strong">Select Actions</span> drop-down list, choose the operation you want to perform.

  <span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Import-From-Meraki/Import%20Meraki%20Window_2.png" original-image-title=""></span>

<!-- -->

- <span class="Strong">Delete</span>: Remove selected import record

  - Selected import record(s) will be deleted.

  - CI(s) imported during the selected import will be removed from Discovered Items.

  - CI(s) moved to CMDB will still appear under CMDB after deleting the Meraki import record.

- <span class="Strong">Export</span> – Download grid contents (selected or filtered rows) to Excel.

  - If you have selected rows, only the selected rows are exported.

  - If you have applied filters, all rows that match the current filters are exported.

  - If no filters are applied and no rows are selected, exporting will include all records from all pages.

  - The file includes the grid columns (e.g., <span class="Strong">Unique ID</span>, <span class="Strong">Name</span>, <span class="Strong">Status</span>, <span class="Strong">Imported By</span>, <span class="Strong">Imported On </span>and <span class="Strong">Credential</span>) (Refer Exported Excel Column Details)

This downloads the import records NOT the CI(s).

- <span class="Strong">Import</span> – Initiate a new Import from Meraki.
