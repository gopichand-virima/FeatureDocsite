# Notification Channels

Use this function to configure and manage notification delivery channels including email, SMS, push notifications, and third-party integrations.

Notification channels determine how notifications are delivered to users, ensuring messages reach recipients through their preferred communication methods.

In the main window, click **Admin > Notification Management > Notification Channels**. The Notification Channels window displays.

## Overview

Available notification channels:

- **Email**: Traditional email delivery
- **SMS**: Text message notifications
- **In-App**: Application notifications
- **Push**: Mobile push notifications
- **Slack**: Slack workspace integration
- **Microsoft Teams**: Teams channel integration
- **Webhook**: Custom API integrations
- **Voice**: Automated voice calls (for critical alerts)

## Configure Email Channel

Click **Email Channel Settings**.

### SMTP Configuration

**Email Provider**:

- Use Virima Email Service (recommended)
- Custom SMTP Server
- Office 365
- Gmail
- SendGrid
- AWS SES
- Other SMTP provider

**SMTP Server Settings**:

- **Server Address**: smtp.yourcompany.com
- **Port**: 587 (TLS), 465 (SSL), 25 (non-secure)
- **Security**: TLS, SSL, or None
- **Authentication**: Username and password
- **Sender Email**: noreply@yourcompany.com
- **Sender Name**: Virima Support Team

**Connection Testing**:

Click **Test Connection** to verify settings.

### Email Delivery Settings

**Delivery Options**:

- **Max Retry Attempts**: 3
- **Retry Interval**: 5 minutes
- **Timeout**: 30 seconds
- **Batch Size**: 50 emails per batch
- **Rate Limiting**: 100 emails per minute

**Bounce Handling**:

- Track bounced emails
- Auto-disable invalid addresses
- Retry soft bounces
- Report hard bounces

**Email Tracking**:

- Track email opens
- Track link clicks
- Delivery confirmation
- Read receipts

Click **Save**.

## Configure Webhook Channel

Click **Webhook Settings**.

### Webhook Configuration

**Webhook URL**: https://your-system.com/api/notifications

**HTTP Method**: POST

**Authentication**:

- None
- Basic Auth (username/password)
- Bearer Token
- API Key (header or query param)
- OAuth 2.0

**Headers**:

Add custom headers:

```
Content-Type: application/json
X-API-Key: your-api-key
```

**Payload Format**:

JSON payload structure:

```json
{
  "event": "ticket.assigned",
  "ticket_number": "{ticket_number}",
  "priority": "{priority}",
  "assigned_to": "{assigned_to}",
  "timestamp": "{timestamp}"
}
```

**Retry Settings**:

- Retry on failure: 3 attempts
- Retry delay: 1 minute
- Timeout: 30 seconds

**Test Webhook**:

Click **Send Test Payload**.

Verify your system receives the webhook.

Click **Save**.

## Configure Voice Channel

Click **Voice Channel Settings**.

### Voice Call Configuration

**Provider**:

- Twilio Voice
- AWS Connect
- Custom voice gateway

**Call Settings**:

- **From Number**: Your phone number
- **Voice**: Select voice type (male/female)
- **Language**: Select language
- **Speed**: Normal, slow, fast

**Message Template**:

Text-to-speech message:

```
This is an urgent alert from Virima. 
Ticket {ticket_number} with priority {priority} requires immediate attention.
Please log in to the system for details.
Press 1 to acknowledge this alert.
```

**Call Handling**:

- Max call duration: 60 seconds
- Retry if no answer: 2 times
- Retry interval: 5 minutes
- Leave voicemail: Yes

**Usage Controls**:

- Only for P1 incidents
- After-hours only
- Approval required
- Cost alerts

Click **Save**.

## Channel Priority and Fallback

Configure channel priority and failover:

Click **Channel Priorities**.

### Primary and Fallback Channels

**User Preference Priority**:

1. User's preferred channel (from profile)
2. Email (default fallback)
3. In-app (always available)

**Delivery Assurance**:

If primary channel fails:

- Try secondary channel
- Try tertiary channel
- Log delivery failure

**Critical Alert Handling**:

For P1 incidents, use all channels:

1. Push notification (immediate)
2. SMS (within 1 minute)
3. Email (confirmation)
4. Voice call (if no acknowledgment in 5 min)

Click **Save**.
