# Problem Model

Use this function to configure and populate problem templates that can be reused for defining problem records. These models can be used for commonly occurring problems to skip or reduce form completion time.

In the main window, click **Admin > Problem Management > Problem Model**. The Problem Model window displays.

## New Problem Model

From the *Select Actions* drop-down list, choose **New Problem Model**. The New Problem Model window displays.

Enter the problem details that should be pre-populated:

- **Name**: Template name
- **Description**: Detailed description of the problem type
- **Category**: Select problem category
- **Subcategory**: Select problem subcategory
- **Priority**: Set default priority
- **Impact**: Set default impact level
- **Urgency**: Set default urgency level
- **Assignment Group**: Default assignment group
- **Assignee**: Default assignee
- **Root Cause**: Known or suspected root cause
- **Workaround**: Temporary workaround if available

Configure additional pre-filled fields as needed.

When all selections/entries are made, click **Add**.

## Edit Problem Model

Navigate to the window containing the record to edit.

Click the line containing the record. The applicable window displays.

Make the necessary modifications.

Click **Save**.

## Delete Problem Model

Click the line that contains the item to delete.

From the *Select Actions* drop-down list, choose **Delete**.
