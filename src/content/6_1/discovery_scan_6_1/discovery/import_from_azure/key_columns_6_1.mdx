# Key columns

- <span class="Strong">Blueprint Name</span> – The discovery blueprint that matched the resource (e.g., <span class="Emphasis">Azure Virtual Network</span>, <span class="Emphasis">Azure Neo4j</span>)

- <span class="Strong">Asset Name</span> – The discovered resource name (cluster/service/task/instance, etc.).

  - The asset name is the AZURE instance ID.

- <span class="Strong">Status</span><span class="inline-comment-marker">: Indicates the current state of the CI imported from AZURE. When a CI is newly imported (does not exist in CMDB), the status is shown as "New."</span>

- <span class="Strong">Asset ID</span>: A unique, sequential identification number assigned to each discovered AZURE record. The ID shall have a prefix “DSC”+ 6 digits Unique sequential number.

- <span class="Strong">Host Name: </span>The host or endpoint name captured by discovery.

- <span class="Strong">Subscription Id</span>: The unique identifier for the Azure subscription or management group associated with the asset. In Virima, this column displays the value entered for either a Subscription Id or a Management Group Id when adding Azure credentials.

Users can select which columns to display when importing AZURE and sort the grid by any column.
