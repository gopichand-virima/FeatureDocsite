# Importing Scan Schedule(s)

1.  Click <span class="Strong">Select Actions </span>→ <span class="Strong">Import</span>.  
    A modal like <span class="Strong">Import Schedule Scan</span> opens with <span class="Strong">Click here to download the sample file</span>, <span class="Strong">Browse</span>, and <span class="Strong">Upload</span>.

<span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Scheduled-Scans-and-Imports/Scheduled%20Scans%20and%20Imports_1_13.png" original-image-title=""></span>

2.  (Optional) Click the link to download the <span class="Strong">sample template</span> that looks as below. Follow the instructions while populating the data

<span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Scheduled-Scans-and-Imports/Scheduled%20Scans%20and%20Imports_1_14.png" original-image-title=""></span>

3.  Prepare your file and keep the <span class="Strong">headers exactly as per the sample file</span>, and fill values for:

    - <span class="Strong">Name</span> is mandatory and must contain a string

    - <span class="Strong">Parameters (IP Range) </span>to be imported must be in proper format. The maximum number of IP(s) allowed is 1024

    - <span class="Strong">Client Name is mandatory. Can specific either Client name or Client ID and is case sensitive</span>

    - <span class="Strong">Timezone</span>, <span class="Strong">Active</span> (true/false)

    - <span class="Strong">Schedule</span> fields (<span class="Strong">second/minute/hour/day/month/weekday</span>)

    - <span class="Strong">Location</span>

    - <span class="Strong">Active </span>(true/false)

    - <span class="Strong">Recurring</span> (true/false)

    - <span class="Strong">Probe: </span>Probe name to be executed when the scan is triggered

    - <span class="Strong">recipients</span> (for network scans)

4.  <span class="Strong">Browse</span> to file location

5.  <span class="Strong">Select file</span>

6.  <span class="Strong">Upload</span>. file

7.  Close the dialog

8.  verify new/updated jobs in the grid.

<span class="Strong">Important behaviors</span>

- If a row’s schedule is in the <span class="Strong">past</span>, it’s <span class="Strong">skipped</span> during import (silently).

- <span class="Strong">Active = true</span>: Record will appear in scheduled scan and Import job list and scan will be triggered at designated time

- <span class="Strong">Active = false/blank</span>: Record will appear in scheduled scan and Import job list but a scan shall not be triggerred

- In <span class="Strong">time fields</span>:

  - /* is <span class="Strong">not allowed</span> for <span class="Strong">minute</span> (row skipped).

  - /* in <span class="Strong">day/hour/month/weekday</span> = “every …”.

  - <span class="Strong">Month/weekday</span> must be <span class="Strong">numeric</span> (no “Jan”/“Mon”).

- If there are errors, you’ll get an <span class="Strong">error report</span> to fix and re-upload.
