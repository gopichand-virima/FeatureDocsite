# **All Tab**

- Displays every record included in the imported file, regardless of whether it matches an existing CI or is a new CI.

 - [Blueprint Name]{.Strong}: Name of the Blueprint

 - [CI]{.Strong}: CMDB CI ID matching the imported CI

 - [Asset Name]{.Strong}: The name of the asset as recognized in the import file or system.

 - [Asset ID]{.Strong}: New Asset ID is created with prefix of "IMP" to denote that this record was imported. The format is "IMP" + 6 digits (auto generated sequential number).

- Clicking the hyperlinked CI ID under CI column will display the corresponding record in CMDB.

 ![Import Data Files 1 13](../../Resources/Images/Discovery/Import-Data-Files/Import Data Files_1_13.png)

- If we click on any other column values regardless of whether CI exists in CMDB or not, the user is redirected to Imported Assets view to view the imported asset data.

 ![Import Data Files 1 14](../../Resources/Images/Discovery/Import-Data-Files/Import Data Files_1_14.png)

# **Layout and Sections**

- [Header bar]{.Strong}

 - CI identifier and name at the top (e.g., *AST001516*).

 - [Select Actions]{.Strong} menu for record-level actions.

 - [Business Service Map]{.Strong} button (opens the service dependency map for this CI).

- [Tabs across the top]{.Strong}

 - [Details]{.Strong} -- the editable profile for the CI (core and custom fields).

 - [Components]{.Strong} -- componentized items attached to the CI.

 - [ITSM]{.Strong} -- quick access to incidents/requests/problems/changes/known errors/releases/knowledge tied to the CI.

 - [Relationships]{.Strong} -- Shows connections between this CI and others

 - [SLA]{.Strong} -- applicable service levels.

 - [Maintenance]{.Strong} -- Schedule maintenance tasks for a CI, such as rebooting or running specific commands.

 - [Private Properties]{.Strong} configured private properties at the blueprint level for specific users with access.

 - [Tasks]{.Strong} -- Tracks tasks or actions assigned to this CI

 - [Comments]{.Strong} -- audit notes / collaboration thread.

 - ****History** - **Shows modification history, including timestamps and users

 - [Vulnerability]{.Strong} (if enabled) -- Lists known vulnerabilities, threat severity, and risk details

- [Primary Details]{.Strong}

 - [Asset Id:]{.Strong} The CMDB's unique identifier for this CI (e.g., AST003156).

 - [Asset Name:]{.Strong} Provided Asset name; here it includes the host and IP (ADSERVERLD @ 10.14.80.36).

 - ****Version**:** Optional version/build of the asset or software (blank in the screenshot).

 - ****Blueprint**:** The blueprint associated (e.g., **Windows Server**).

 - ****Confidence Level**:** Data quality score from discovery/enrichment (e.g., **Very High**).

 - [IP Address:]{.Strong} Last known IP (e.g., 10.14.80.36).

 - [Host Name:]{.Strong} DNS/host name (e.g., ADSERVERLD).

 - [Created On:]{.Strong} When this CI was first created in CMDB.

 - [Last Modified On:]{.Strong} Most recent update to any field on this record.

 - [Last Seen On:]{.Strong} Last successful "sighting" in discovery (empty if not yet observed).

 - [Last Scanned On:]{.Strong} Timestamp of the most recent discovery scan for this CI (empty if not scanned).

 - [Warranty Start Date / Warranty End Date:]{.Strong} Warranty lifecycle dates (editable).

 - [Location / Department / Vendor:]{.Strong} Organizational metadata fields for ownership/placement (editable).

 - ****Tags**:** Free-form labels/metadata you can use for grouping, search, or policy (editable).

 - [Update Picture:]{.Strong} Upload/replace the icon shown for this asset.

- [Owner]{.Strong}

 - Person card for the primary owner: **Name**, **Email**, **Phone** (with avatar); quick remove/edit where permitted.

- [Second Level Support]{.Strong} / [Third Level Support]{.Strong}

 - Support group or contact cards with **Name**, **Email**, **Phone**. Used for routing escalations.

- **Manager**

 - Manager contact card (Name/Email/Phone) tied to the CI or its owning team.

- [Related Records]{.Strong}

 - Shortcuts (often with counts) to ITSM items associated to this CI:

 - **Open Incidents**, **Open Requests**, **Open Problems**

 - **Open Known Errors**, **Releases**, **Open Knowledge**

 - Clicking a link takes you to the filtered list for that record type with this CI pre-applied.
