# Knowledge Approval

The knowledge approval process ensures articles are accurate, complete, and properly formatted before publication.

## Approval Workflow

### Submit for Approval
1. Complete article
2. Self-review
3. Click **Submit for Approval**
4. Article moves to pending status
5. Approvers notified

### Review Process
Approver actions:
1. Receive notification
2. Review article content
3. Check accuracy
4. Verify formatting
5. Test procedures
6. Make decision

### Approval Decision
- **Approve**: Article published
- **Reject**: Returned to author with comments
- **Request Changes**: Specific edits needed

## Approval Criteria

### Content Quality
- Accurate information
- Complete instructions
- Clear language
- Proper grammar/spelling
- Appropriate detail level

### Technical Accuracy
- Procedures tested
- Commands verified
- Screenshots current
- Links working
- Examples valid

### Formatting Standards
- Proper headings
- Numbered steps
- Consistent style
- Readable layout
- Images sized appropriately

### Searchability
- Relevant keywords
- Proper categorization
- Clear title
- Good description
- Appropriate tags

## Approval Roles

### Knowledge Contributor
- Create articles
- Submit for approval
- Revise based on feedback
- Update published articles

### Subject Matter Expert (SME)
- Review technical accuracy
- Verify procedures
- Validate solutions
- Provide expert feedback

### Knowledge Manager
- Final approval authority
- Ensure quality standards
- Manage knowledge base
- Set approval policies

## Approval Notifications

### To Author
- Article submitted confirmation
- Approval/rejection notice
- Comments from reviewer
- Publication confirmation

### To Approvers
- New article pending
- Reminder if not reviewed
- Escalation if delayed
- Review deadline

## Handling Rejections

### Review Feedback
- Read reviewer comments
- Understand concerns
- Identify needed changes
- Ask questions if unclear

### Make Revisions
- Address all feedback
- Test changes
- Update article
- Add revision notes

### Resubmit
- Submit revised article
- Reference previous feedback
- Note changes made
- Request re-review

## Expedited Approval

For urgent articles:
1. Mark as **Urgent**
2. Provide justification
3. Senior approver notified
4. Fast-track review
5. Publish quickly

**Use Cases:**
- Critical security issues
- Major incident workarounds
- Time-sensitive procedures
- Emergency updates

## Best Practices

### For Contributors
- Self-review thoroughly
- Test all procedures
- Include all details
- Format properly
- Submit complete articles

### For Approvers
- Review promptly
- Provide clear feedback
- Test procedures
- Be constructive
- Communicate clearly
