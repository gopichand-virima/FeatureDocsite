# Business Rules

Use this function to automate tasks, such as modifying the ITSM records or creating a new record based on action details when trigger conditions are met. This reduces the manual effort of creating and updating records for repeatable actions.

Business rules can be set to run at the time when any Add or Edit actions occurs.

In the main window, click **Admin > Others > Business Rules**. The Business Rules window displays.

!

!


### New Business Rule

- From the *Select Actions* drop-down list, choose an **New Business Rule**.

!

- Complete the fields referring to the table below.
- When all selections/entries are made, click **Add**.

### Business Rules Fields

| Field | Description |

| --- | --- |

| **Rule Name** | Name for this rule. |

| Description | Description for this rule. |

| Order/Priority | The order in which this rule should be executed when there are multiple rules set for the selected module. Must be a numeric value. |

| Module | Name of this module for which the rule applies. Click the drop-down list and select an available module, such as Incidents, CMDB, and so forth. |

| Turn on cascade execution | Enable this option to execute the next enabled business rule based on the Order/Priority for the same module. |

| Enable rule | Specifies if the rule is enabled or disabled. |

| Execute On | Specifies on what action the rule is executed. |

| Associated Blueprints | Click **Add** and select the Blueprint(s) to associate with this business rule. |


### Trigger Conditions

After adding a business rule, add the trigger conditions. You can combine two or more conditions using "AND" or "OR" logical operators.

- Open an existing business rule.
- Click **Add condition**.
- Move to the *Trigger Conditions* *Details* section.

!

- To ignore the conditions, select *Ignore Conditions*.
- Select the *Property* and *Operator*. Then enter the applicable value.
- Perform any of the actions below for additional conditions.
- When all selections/entries are made, click **Save**.

### Add/Delete Condition Block

- To add a condition block (which also adds the Action Details and Notifications blocks), click **Add Condition**.
- To delete a condition block, click **Delete Condition**. Then confirm the deletion.

### Apply AND Condition

To insert an AND condition:

- Click the **plus + sign icon**.
- Enter the condition details.
- Add more AND conditions, as required.
- If there are no more conditions to add, click **Add**.

### Apply OR condition

To insert an OR condition:

- Click the **or icon**.
- Enter the condition details.
- Include more OR conditions, as required.
- If there are no more conditions to add, click **Add**.

### Delete Single Condition

- Locate the condition to delete.
- Click the **red x** **icon**.


### Action Details

After a business rule has been added, the action details are added.

- Open an existing business rule.
- Move to the *Action Details* section.

!

- In the *Action Type* field, click the drop-down list and select the action to perform, such as updating the triggered record.
- Select the *Property* and *Operator*. Then enter the applicable value.
- Perform any of the actions below for additional conditions.
- When all selections/entries are made, click **Save**.

### Add/Delete Action Details

- To add a single action row, click the **plus + sign**.
- To delete a single action row, click the **minus - sign**. Then confirm the deletion.

### Add/Delete Action Details Block

- To add an action detail block, click **Add Action**. The window updates and an additional Action block displays.
- To delete an action detail block, click **Delete Action**. Then confirm the deletion.


### Notifications

- To add a user notification when a rule is triggered, click **Add**.
- In the Send Report To dialog box, search for a select the applicable users, groups, roles, and/or departments.
- Click **Save**.
- To delete a notification, click the **X** next to the name (of the user, groups and so forth).


### Edit Business Rule

Not all functions include an option to edit record line items shown in the main window.


- Navigate to the window containing the record to edit.
- Click the line containing the record. The applicable window or dialog box displays.
- Make the necessary modifications.
- Click **Save** or **Add**, as applicable.


### Delete Business Rule

Deleting is a permanent action and cannot be undone. Deleting may affect other functionality and information in the application such as data in configured reports, fields in windows, selectable options, etc. Therefore, be sure to understand the potential effects before making a deletion.


- Click the line that contains the item to delete.
- From the *Select Actions* drop-down list, choose **Delete**. If a confirmation message is displayed, type **Delete** and click **Delete**to confirm.
