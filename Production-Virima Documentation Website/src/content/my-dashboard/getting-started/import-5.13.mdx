# Import

Import data into Virima from external files and systems to populate or update records.

## Overview

The Import function allows you to bulk load data into Virima from various file formats, making it easy to migrate data, perform bulk updates, and integrate with external systems.

## Supported Import Formats

### File Formats
- **Excel (XLSX, XLS)**: Spreadsheet files with multiple sheets
- **CSV**: Comma-separated values files
- **JSON**: Structured data format
- **XML**: Extensible markup language files
- **TXT**: Tab-delimited or pipe-delimited text files

## Import Process

### Basic Import Steps
1. Navigate to the module where you want to import data
2. Click the **Import** button
3. Select **Choose File** and browse to your import file
4. Map file columns to Virima fields
5. Configure import options
6. Review validation results
7. Click **Import** to begin the process

### Import Wizard
The import wizard guides you through:
- File selection and upload
- Column mapping
- Data validation
- Import preview
- Error handling
- Final confirmation

## Data Validation

### Pre-Import Validation
System validates:

- Valid reference values

- Data type matching

### Validation Results
Review validation report showing:
- Number of valid records

- Suggested corrections

## Import Options

### Import Mode
Choose how to handle existing records:
- **Insert Only**: Create new records only
- **Update Only**: Update existing records only
- **Insert and Update**: Create new and update existing (upsert)
- **Replace**: Delete existing and import new

### Match Criteria
Define how to match existing records:
- Match by ID field
- Match by unique identifier
- Match by multiple fields
- Custom matching rules

### Error Handling
Configure how to handle errors:
- **Stop on Error**: Halt import if any error occurs
- **Skip Errors**: Skip invalid records and continue
- **Log Only**: Import all, log errors for review
- **Interactive**: Prompt for each error

### Field Transformations

### Default Values
- Set default values for empty fields
- Apply conditional defaults

## Import Limits

### Record Limits
- Standard import: Up to 5,000 records
- Bulk import: Up to 50,000 records
- Large import: Contact administrator
- Split large files if needed

### File Size Limits
- Maximum file size: 50 MB (default)
- Larger files may require special handling
- Contact administrator for limits increase

## Best Practices

### Before Import
- Backup existing data
- Validate source data quality
- Test import with small sample
- Review field mappings
- Check for duplicates
- Clean data before import

### During Import
- Monitor import progress
- Review validation messages
- Address errors promptly
- Don't interrupt process
- Keep import file accessible

### After Import
- Verify imported data accuracy
- Review import logs
- Check relationships and references
- Update related records if needed
- Document import process
- Archive import files

## Troubleshooting

### Common Issues
- **File format errors**: Ensure correct file type and encoding
- **Mapping errors**: Verify field names and types match
- **Validation failures**: Fix data to meet requirements
- **Duplicate records**: Check match criteria settings
- **Timeout errors**: Reduce batch size or split file

### Resolution Steps
1. Review error messages carefully
2. Check import logs for details
3. Validate source data
4. Test with smaller dataset
5. Contact administrator if needed
