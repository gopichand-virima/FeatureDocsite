# Create Knowledge Article

Create knowledge articles to share solutions, procedures, and best practices with your organization.

## When to Create Knowledge

Create articles for:
- Frequently asked questions
- Common issues and solutions
- Step-by-step procedures
- Best practices
- Troubleshooting guides
- Configuration instructions
- Tips and tricks

## Creating an Article

### Method 1: From Knowledge Base

1. Navigate to **Knowledge Management**
2. Click **New Article**
3. Select article template
4. Complete article fields
5. Click **Submit for Approval**

### Method 2: From Incident

Create from resolved incident:
1. Open resolved incident
2. Click **Create Knowledge Article**
3. System pre-fills from incident
4. Enhance and format content
5. Submit for approval

### Method 3: From Problem

Document problem solution:
1. Open resolved problem
2. Click **Create Knowledge Article**
3. Include root cause and solution
4. Add troubleshooting steps
5. Submit

## Article Components

### Title *
Clear, descriptive title:
- What the article covers
- Include key terms
- Keep concise
- Example: "How to Reset Password in Active Directory"

### Short Description *
Brief summary (1-2 sentences):
- What problem it solves
- Who should use it
- Example: "Step-by-step guide for IT administrators to reset user passwords in Active Directory"

### Category *
Article classification:
- Hardware
- Software
- Network
- Security
- Procedures
- Best Practices

### Keywords/Tags
Search terms:
- Add 5-10 relevant keywords
- Include synonyms
- Common error messages
- Product names

### Article Body *
Main content:

**Structure:**
1. **Overview**: What this article covers
2. **Prerequisites**: What's needed
3. **Procedure**: Step-by-step instructions
4. **Expected Results**: What should happen
5. **Troubleshooting**: Common issues
6. **Related Information**: Links to related articles

### Attachments
Supporting files:
- Screenshots
- Diagrams
- Templates
- Scripts
- Documentation

## Writing Effective Articles

### Use Clear Language
- Avoid jargon
- Define technical terms
- Write for target audience
- Use active voice

### Structure Content
- Use headings
- Number steps
- Bullet points for lists
- Short paragraphs

### Include Examples
- Real scenarios
- Sample data
- Expected outputs
- Before/after comparisons

### Add Visuals
- Screenshots with annotations
- Flowcharts
- Diagrams
- Video tutorials

## Article Templates

### How-To Template

```markdown
# How to [Task]

## Overview
Brief description of what this procedure accomplishes.

## Prerequisites
- Required access/permissions
- Tools needed
- Skills required

## Procedure
1. Step one
2. Step two
3. Step three

## Expected Results
What should happen when completed successfully.

## Troubleshooting
Common issues and solutions.

## Related Articles
Links to related content.
```

### Troubleshooting Template

```markdown
# Troubleshooting [Issue]

## Symptoms
How the problem manifests.

## Cause
Why this issue occurs.

## Solution
Step-by-step fix.

## Prevention
How to avoid in future.

## Related Information
Links to related articles.
```

### FAQ Template

```markdown
# Frequently Asked Questions - [Topic]

## Question 1
Detailed answer.

## Question 2
Detailed answer.

## Additional Resources
Links to related content.
```

## Article Review Process

### Draft Status
- Article created
- Being edited
- Not visible to users
- Can save and return

### Pending Approval
- Submitted for review
- Knowledge manager notified
- Awaiting approval
- Cannot edit

### Approved
- Published to knowledge base
- Visible to authorized users
- Available in search
- Can be updated (with new approval)

### Rejected
- Returned to author
- Comments provided
- Needs revisions
- Resubmit when ready

## Best Practices

### Research First
- Search existing articles
- Avoid duplicates
- Link to related content
- Update existing if appropriate

### Test Procedures
- Follow your own steps
- Verify accuracy
- Test on different systems
- Get peer review

### Keep Updated
- Review regularly
- Update for system changes
- Refresh screenshots
- Remove obsolete information

### Gather Feedback
- Monitor article ratings
- Read user comments
- Track article usage
- Improve based on feedback

## After Publishing

### Promote Article
- Share with team
- Add to training materials
- Link from related articles
- Include in newsletters

### Monitor Usage
- Track views
- Check ratings
- Review feedback
- Measure effectiveness

### Maintain Content
- Regular reviews
- Update as needed
- Archive when obsolete
- Version control
