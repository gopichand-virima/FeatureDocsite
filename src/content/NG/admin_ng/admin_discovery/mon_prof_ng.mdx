# Discovery Monitoring Profile

Use this function to monitor machines based on certain defined properties, which are set as Trigger Conditions at a given frequency. If a condition matches, it creates an ITSM record with pre-defined values based on Action Details.

## [Workflow Example]

::: Bullet1
IF the [Trigger Condition](Mon-Prof-Trigger-Conditions.mdx) *Property* is **RAM(MB)** and *Operator* **Is Empty**\...

::: Bullet1
AND the [Action Details](Mon-Prof-Action-Details.mdx) *Target Module* is **Incident**, then an incident will be created in the application\...

::: Bullet1
THEN **[Notifications](Mon-Prof-Notifications.mdx){style="font-weight: normal;"}** will be sent to the configured users, roles, groups, or departments.
