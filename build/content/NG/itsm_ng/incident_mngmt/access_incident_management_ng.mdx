# Access Incident Management

The Incident Management module provides tools for logging, tracking, and resolving IT service incidents. This guide explains how to access and navigate Incident Management features.

## Accessing from the Main Menu

1. Log in to your Virima instance
2. Navigate to the **ITSM** module from the main navigation
3. Select **Incident Management** from the dropdown menu

## Quick Access Options

### Dashboard Widgets
Add Incident Management widgets to your dashboard:

1. Navigate to your **Dashboard**
2. Click **Add Widget**
3. Select from available widgets:
   - **My Open Incidents**: Incidents assigned to you
   - **Team Incidents**: Your team's incident queue
   - **Critical Incidents**: High-priority incidents
   - **Recent Incidents**: Latest incident activity
   - **SLA Violations**: Incidents breaching SLA

### Quick Create
Create incidents quickly from anywhere:

1. Click the **+** icon in the top navigation
2. Select **New Incident**
3. Fill in required details
4. Click **Submit**

### Global Search
Find incidents using global search:

1. Click the search icon
2. Enter incident number or keywords
3. Select the incident from results

## Navigation Structure

The Incident Management module includes:

- **Incidents**: View and manage all incidents
- **My Incidents**: Incidents assigned to you
- **Team Queue**: Incidents assigned to your team
- **Categories**: Browse incidents by category
- **SLA Dashboard**: Monitor SLA compliance
- **Reports**: Incident metrics and analytics

## Main Incident List

The incident list displays key information:

### Default Columns
- **Incident Number**: Unique identifier
- **Title**: Brief description
- **Status**: Current state
- **Priority**: Urgency and impact combination
- **Assigned To**: Current owner
- **Created**: Date and time opened
- **SLA Status**: Time remaining or violated

### Customizing Columns
1. Click the **Column** selector
2. Check/uncheck columns to show/hide
3. Drag columns to reorder
4. Click **Save View** to preserve settings

## Incident Queues

### My Incidents
View incidents assigned to you:
1. Navigate to **Incident Management**
2. Select **My Incidents** view
3. See all incidents requiring your action

### Team Queue
View your team's incidents:
1. Select **Team Queue** view
2. See all incidents assigned to your team
3. Filter by team member if needed

### Unassigned Queue
View unassigned incidents:
1. Select **Unassigned** view
2. Pick up incidents from the queue
3. Assign to yourself or team members

## User Permissions

Access levels vary by role:

### Incident Manager
- Full access to all incidents
- Can reassign any incident
- Access to all reports
- Configure incident settings

### Incident Resolver
- View assigned incidents
- Update incident status
- Add work notes
- Resolve incidents

### Incident Creator
- Create new incidents
- View own incidents
- Add comments
- View status updates

### End Users
- Submit incidents via self-service
- View own incident status
- Communicate with support
- Rate resolution quality

Contact your System Administrator for role changes.
