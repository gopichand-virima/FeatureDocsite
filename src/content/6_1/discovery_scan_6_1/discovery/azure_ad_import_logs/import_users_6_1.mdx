# Import Users

While importing users into Virima, the system checks if the user already exists. If not, Virima creates a new user.

For consecutive Azure AD imports, the value of a user gets updated if the value has been modified in the Azure AD.

After configuring Azure AD, follow these steps:

1.  Click the **Import** **icon** located to the right of the selected entry.

<span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Azure-AD-Users-Import-Logs/Azure%20AD%20User%20Import%20Logs_8.png" original-image-title=""></span>

2.  The <span class="Strong" style="font-weight: bold;">Azure AD Import</span> dialog box displays.

    1.  In the <span class="Strong" style="font-weight: bold;">Azure AD Import</span> dialog box:

        1.  <span class="Strong" style="font-weight: bold;">Reflect AD Account Status</span> – Enable to match user login status with Azure AD.

        2.  <span class="Strong" style="font-weight: bold;">Activate User Account</span> – Enable to make imported users active by default.

        3.  <span class="Strong" style="font-weight: bold;">Send Activation Notification</span> – Enable to email users upon account activation.

<span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Azure-AD-Users-Import-Logs/Azure%20AD%20User%20Import%20Logs_9.png" original-image-title=""></span>

3.  When all selections/entries are made, click <span class="Strong" style="font-weight: bold;">Import</span>.

4.  A confirmation message displays. Click <span class="Strong" style="font-weight: bold;">OK</span>.
