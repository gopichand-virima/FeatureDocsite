# Import from Azure

Use this function to identify information existing in AZURE, a cloud service owned by Microsoft. The discovery of AZURE is based on account information rather than an IP range.

|     |                                                           |
|-----|-----------------------------------------------------------|
|     | Discovery Servers are not used in this type of discovery. |

<div class="numlist1" madcap:autonum="1. ">

In the navigation pane, select **Discovery Scan /> Import from Azure**. The Import from AZURE window displays.

</div>

<div class="Ind50">

<span class="image placeholder" original-image-src="../Resources/Images/Discovery/import-from-azure.PNG" original-image-title="" madcap:conditions="Default.OldUI" style="width: 622px;height: 162px;"></span>

</div>

<div class="Ind50">

<span class="image placeholder" original-image-src="../Resources/Images/Discovery/import-from-azure-new.PNG" original-image-title="" madcap:conditions="Default.NewUI" style="width: 622px;height: 105px;"></span>

</div>

<div class="numlist2" madcap:conditions="Default.OldUI" madcap:autonum="2. ">

From the *Select Options* drop-down box, choose **Import**. The Import from AZURE dialog box displays.

</div>

<div class="numlist2" madcap:conditions="Default.NewUI" madcap:autonum="3. ">

Click on the **Import** icon. The Import from Azure dialog box displays.

</div>

<div class="Ind50">

<span class="image placeholder" original-image-src="../Resources/Images/Discovery/import-from-azure-dialog-box-new.PNG" original-image-title="" madcap:conditions="Default.OldUI"></span>

</div>

<div class="Ind50" madcap:conditions="Default.Omit" style="color: #dc143c;background-color: #ffff00;">

GET V6 IMAGE

</div>

<div class="numlist2" madcap:conditions="Default.OldUI,Default.NewUI" madcap:autonum="4. ">

In the *Name* field, enter a name for the credential.

</div>

<div class="numlist2" madcap:autonum="5. " madcap:conditions="Default.OldUI">

In the *Credential* field, click the drop-down list and select the type of credential, such as established in Azure.

</div>

<div class="numlist2" madcap:conditions="Default.NewUI" madcap:autonum="6. ">

In the *Credential* field, click the drop-down list and select the type of credential, such as Azure Subscription.

</div>

<div class="numlist2" madcap:conditions="Default.NewUI,Default.OldUI" madcap:autonum="7. ">

Select the information to import.

</div>

<div class="numlist2" madcap:conditions="Default.NewUI,Default.OldUI" madcap:autonum="8. ">

Click **Add** to import the information. Otherwise, click **Cancel** to cancel the operation.

</div>

Edit Import from Azure

Use this function to edit the values of the configured parameters and update the information in the application.

<div class="NumList1" madcap:autonum="1. ">

In the navigation pane, select **Discovery Scan /> Import from Azure**.

</div>

<div class="NumList2" madcap:autonum="2. ">

Select the record to edit. The Edit Report window displays the configured CI details into the following categories: **All**, **Authorized**, and **Unauthorized**.

</div>

<div class="Bullet2">

*Authorized* represents all assets that are already present inside CMDB.

</div>

<div class="Bullet2">

*Unauthorized* represents new assets.

</div>

<div class="Ind50" madcap:conditions="Default.NewUI">

<span class="image placeholder" original-image-src="../Resources/Images/Discovery/import-azure-edit-new.PNG" original-image-title="" style="width: 623px;height: 177px;"></span>

</div>

<div class="NumList2" style="color: #000000;" madcap:autonum="3. ">

To move a record CMDB, do the following:

</div>

<div class="NumLista" madcap:autonum="a. ">

Select a record.

</div>

<div class="NumListb" madcap:autonum="b. ">

Click **Move to CMDB**.

</div>

<div class="NumListb" madcap:autonum="c. ">

In the Alert Message window, click **Continue** to move the record. The process runs in the background and no further action is required.

</div>
