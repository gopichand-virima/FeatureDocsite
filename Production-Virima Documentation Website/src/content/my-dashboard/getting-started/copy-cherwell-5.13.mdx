# Copy to Cherwell

Transfer records from Virima to Cherwell Service Management for integrated ITSM operations.

## Overview

The Copy to Cherwell function enables seamless data transfer from Virima to Cherwell Service Management, allowing you to create incidents, configuration items, and other records in Cherwell directly from Virima.

## Prerequisites

### Integration Setup
Before using this function, ensure:
- Cherwell integration is configured by your administrator
- You have valid Cherwell credentials
- Proper API permissions are set up
- Field mapping is configured between systems

## Copying Records to Cherwell

### Basic Copy Operation
1. Select the record you want to copy
2. Click the **Copy to Cherwell** button or menu option
3. Review the mapped field values
4. Modify fields if needed
5. Click **Create in Cherwell** to complete the transfer

### Field Mapping
- Fields are automatically mapped based on configuration
- Required Cherwell fields must be populated
- Custom field mappings may be available
- Unmapped fields can be manually entered

## Supported Record Types

### Incidents
- Copy incident details to Cherwell
- Transfer attachments and comments
- Maintain relationships and dependencies
- Sync status updates

### Configuration Items
- Transfer CI information
- Copy asset attributes
- Include relationships and dependencies
- Sync inventory data

### Other Records
- Service requests
- Change requests
- Problem records
- Knowledge articles (if configured)

## Post-Copy Actions

### Verification
After copying to Cherwell:
- Verify the record was created successfully
- Check that all fields transferred correctly
- Review any error messages or warnings
- Confirm attachments were included

### Linking Records
- Virima maintains a reference to the Cherwell record
- View the Cherwell record ID in Virima
- Click the link to open the record in Cherwell
- Track synchronization status

## Troubleshooting

### Common Issues
- **Authentication errors**: Verify Cherwell credentials
- **Field validation errors**: Check required field values
- **Permission errors**: Confirm user permissions in Cherwell
- **Mapping errors**: Contact administrator to review field mappings

## Best Practices

- Review field mappings before copying
- Ensure required fields are populated
- Verify data accuracy before transfer
- Document integration workflows
- Train users on proper usage
- Monitor integration logs for errors
