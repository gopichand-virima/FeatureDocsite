# Users

Use this function to create new users in the application.

 --- ---------------------------------------------------------------------------------------------------
   [Role Access](../Admin%20-%20Other/Role-Access.mdx) permissions must be set before adding a user.
 --- ---------------------------------------------------------------------------------------------------

In the main window, click **Admin \> Users \> Users**. The Access Management window displays.

![Users v6 1](../Resources/Images/Admin - Users/Users_v6_1.png)

![arrow border](../Resources/Images/arrow-border.PNG)

[[[Closed]{.image .placeholder .MCDropDown_Image_Icon original-image-src="../../Skins/Default/Stylesheets/Images/transparent.gif" original-image-title="" height="12" width="16" User](javascript:void(0)){.MCDropDownHotSpot .dropDownHotspot .MCDropDownHotSpot_ .MCHotSpotImage aria-expanded="false"}]{.MCDropDownHead .dropDownHead}

 -- ----- -----------------------------------------------------------------
 1. From the *Select Actions* drop-down list, choose **New User.**\

 -- ----- -----------------------------------------------------------------

 -- ----- ---------------------------------------------------
 2. To complete the fields, refer to the table below.
 -- ----- ---------------------------------------------------

 -- ----- ------------------------------------------------------
 3. When all selections/entries are made, click **Add**.

+--------------------------------------------------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| Authentications | Check based on the organization authentication process. Choices are AD, AZURE AD, SAML, and Normal. |

| Password | The password associated with the username. The password must be validated by entering it a second time in the **Re-Enter Password** field. To display the password, click **Show Password** checkbox or click the **Show** icon. |
+--------------------------------------------------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| User Profile Fields | Requires **First Name** and **Last Name**. **Display Name** is optional and, by default, is set to "FirstName LastName," customize if needed. |
+--------------------------------------------------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| User Information Fields | Specifies the information to display within the application. |

| [Location](/admin_org_details/locations_6_1.mdx) | Specifies the location of the user. |

| | |
| | Click the **plus + sign** then search for and select the applicable role(s). |

| [Departments](/admin_org_details/departments_6_1.mdx) | Associates the user with a department. |

| | Click **Add** then search for and select the applicable department. |
| | |
| | Click the **plus + sign** then search for and select the applicable department. |
+--------------------------------------------------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| [Groups](User-Groups.mdx) | Associates the user with a group. |
| --- | --- |
| | Click **Add** then search for and select the applicable group. |
| | |
| | Click the **plus + sign** then search for and select the applicable group. |
+--------------------------------------------------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| Report To | Specifies to whom the user reports. |
| --- | --- |
| | Click **Add** then search for and select the applicable user. |
| | |
| | Click the **plus + sign** then search for and select the applicable user. |
+--------------------------------------------------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| [Designation](/admin_org_details/designations_6_1.mdx) | Assigns a designation to the user. |
| --- | --- |
| | Click **Add** then search for and select the applicable designation. |
| | |
| | Click the **plus + sign** then search for and select the applicable designation. |

| Landing Page | Specifies the page that displays to the user on first accessing the application. |
+--------------------------------------------------------------+----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| Language | Specifies the users language. |

Use this function to import new users by uploading a file.

 -- ----- ---------------------------------------------------------------------------------------------------------------------
 1. From the *Select Actions* drop-down list, choose **Import New User(s).** The Attach Data Files dialog box displays.
 -- ----- ---------------------------------------------------------------------------------------------------------------------

[[[Closed]{.image .placeholder .MCDropDown_Image_Icon original-image-src="../../Skins/Default/Stylesheets/Images/transparent.gif" original-image-title="" height="12" width="16" Edit](javascript:void(0)){.MCDropDownHotSpot .dropDownHotspot .MCDropDownHotSpot_ .MCHotSpotImage aria-expanded="false"}]{.MCDropDownHead .dropDownHead}

Use this function edit multiple users at the same time.

 -- ----- --------------------------------------------------
 1. In the list, highlight each user record to edit.
 -- ----- --------------------------------------------------

 -- ----- -----------------------------------------------------------------
 2. From the *Select Actions* drop-down list, choose **Bulk Edit**.
 -- ----- -----------------------------------------------------------------

 -- ----- ----------------------------------------------------
 3. In the Confirmation message window, click **Yes**.
 -- ----- ----------------------------------------------------

 -- ----- ---------------------------------------------------------------------------------------------------
 4. Refer to the information under \"New Users\" for information on the available fields and options.
 -- ----- ---------------------------------------------------------------------------------------------------

 -- ----- ----------------------------------------------------------------------------------------------------------
 5. When all selections/entries are made, click **Update**. The details for all the select users is updated.
 -- ----- ----------------------------------------------------------------------------------------------------------

[[[Closed]{.image .placeholder .MCDropDown_Image_Icon original-image-src="../../Skins/Default/Stylesheets/Images/transparent.gif" original-image-title="" height="12" width="16" MFA](javascript:void(0)){.MCDropDownHotSpot .dropDownHotspot .MCDropDownHotSpot_ .MCHotSpotImage aria-expanded="false"}]{.MCDropDownHead .dropDownHead}

Use this function to enable multi-factor authentication for one or more users.

 -- ----- --------------------------------------------------------------------------------------------------------------------------------
 1. In the list, highlight each user for which MFA should be enabled. If not selection is made, MFA will be enabled for all users.
 -- ----- --------------------------------------------------------------------------------------------------------------------------------

 -- ----- -----------------------
 2. Click **Enable MFA**.
 -- ----- -----------------------

 --- -----------------------------------------------------------------
   To reset MFA, open the user\'s profile and click **Reset MFA**.
 --- -----------------------------------------------------------------

 --- -----------------------------------------------------------------------------------------
   Not all functions include an option to edit record line items shown in the main window.
 --- -----------------------------------------------------------------------------------------

 -- ----- -------------------------------------------------------
 1. Navigate to the window containing the record to edit.
 -- ----- -------------------------------------------------------

 -- ----- -------------------------------------------------------------------------------------
 2. Click the line containing the record. The applicable window or dialog box displays.
 -- ----- -------------------------------------------------------------------------------------

 -- ----- -----------------------------------
 3. Make the necessary modifications.
 -- ----- -----------------------------------

 -- ----- -------------------------------------------
 4. Click **Save** or **Add**, as applicable.
 -- ----- -------------------------------------------

When editing an existing item, other options may be available, such as adding [Tasks](/itsm/release_mngmt/tasks_6_1.mdx){.MCTopicPopup .MCTopicPopupHotSpot .a}, [Comments](/discovery/comments_6_1.mdx){.MCTopicPopup .MCTopicPopupHotSpot .a}, and [Attachments](/discovery/attachments_6_1.mdx){.MCTopicPopup .MCTopicPopupHotSpot .a} and viewing [History](/common_topics/history_6_1.mdx){.MCTopicPopup .MCTopicPopupHotSpot .a}.

 --- ---------------------------------------------------------------------------------------
   The options change based upon the function; therefore, not every option may be shown.
 --- ---------------------------------------------------------------------------------------

In this window, you can also:

Click **Email Preferences** and select all the functions for which the user will receive email notifications. Then click **Save**.
