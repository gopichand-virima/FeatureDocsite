# Reports - User Manual

Complete guide for reporting and analytics in Virima 5.13.

## Report Categories

### Asset Reports
- Asset inventory summary
- Software license compliance
- Hardware lifecycle status
- Asset utilization analysis
- Warranty expiration tracking

### Service Management Reports
- Incident volume and trends
- SLA compliance metrics
- Change success rates
- Problem management statistics
- Request fulfillment metrics

### Compliance and Security
- Vulnerability assessment summary
- Patch compliance status
- Access control audit
- Configuration compliance
- Risk assessment reports

### Financial Reports
- IT spending analysis
- Cost allocation by department
- Budget vs. actual comparison
- TCO analysis
- ROI calculations

## Custom Report Builder

### Data Sources
Connect to data from:
- CMDB
- ITSM tickets
- Discovery scans
- Asset inventory
- Vulnerability scans
- Project data
- Risk register

### Report Components
- **Fields**: Select data columns
- **Filters**: Define data criteria
- **Grouping**: Organize data hierarchically
- **Sorting**: Order results
- **Calculations**: Aggregate data (sum, average, count)

### Visualization Options
- Tables (tabular data)
- Bar charts (comparisons)
- Line charts (trends over time)
- Pie charts (distribution)
- Heat maps (intensity visualization)
- Gauges (KPI metrics)
- Geographic maps (location data)

## Dashboard Creation

### Dashboard Components
- **Widgets**: Individual report visualizations
- **Filters**: Interactive data filtering
- **Date Range Selectors**: Time-based analysis
- **KPI Cards**: Key metric highlights

### Layout Options
- Grid-based layout system
- Responsive design
- Customizable widget sizes
- Drag-and-drop arrangement

## Report Scheduling

### Delivery Options
- Email delivery (PDF, Excel)
- Shared folder export
- Cloud storage integration
- FTP upload

### Schedule Frequencies
- One-time
- Daily (specific time)
- Weekly (specific day and time)
- Monthly (specific date)
- Quarterly
- Custom cron expressions

## Best Practices

- Name reports descriptively
- Use filters to improve performance
- Schedule resource-intensive reports during off-peak hours
- Limit dashboard widgets to 8-10 for optimal performance
- Document custom report logic
- Regular review and cleanup of unused reports
- Use report templates for consistency
- Implement proper access controls
