# Transformation

Use this function to configure data transformation rules for discovered assets.

Data transformation allows you to modify, enrich, or restructure discovery data before it is stored in the CMDB. This includes field mapping, value conversion, and data enrichment.

In the main window, click **Admin > Discovery > Transformation**. The Transformation window displays.

## Transformation Rules

### New Transformation Rule

From the *Select Actions* drop-down list, choose **New Transformation Rule**. The New Transformation Rule window displays.

Enter a *Rule Name* and *Description*.

Select the **Source Data Type** (Device, Software, Network, etc.).

Configure the transformation:

**Field Mapping**:

- Source Field: Select the field from discovery data
- Target Field: Select the CMDB field to populate
- Transformation Type: Direct copy, Formula, Lookup, Script

**Value Transformation**:

- **Pattern Matching**: Use regex to extract values
- **Value Replacement**: Replace specific values
- **Concatenation**: Combine multiple fields
- **Calculation**: Apply formulas

**Conditional Rules**:

- Apply transformation only when conditions are met
- Set if/then/else logic

**Script Transformation** (Advanced):

- Write custom JavaScript or Python scripts
- Access to full discovery payload
- Complex data manipulation

When all selections/entries are made, click **Add**.

### Edit Transformation Rule

Navigate to the window containing the record to edit.

Click the line containing the record. The applicable window displays.

Make the necessary modifications.

Click **Save**.

### Test Transformation Rule

Before applying a rule to production data:

Select a transformation rule.

Click **Test Rule**.

Provide sample input data.

Review the transformed output.

Make adjustments as needed.

### Delete Transformation Rule

Click the line that contains the item to delete.

From the *Select Actions* drop-down list, choose **Delete**.

## Transformation Templates

Pre-built transformation templates are available for common use cases:

- **Hardware Manufacturer Standardization**
- **Software Version Formatting**
- **IP Address Classification**
- **Serial Number Normalization**
- **Custom Field Population**

To use a template:

Click **Load Template**.

Select a template from the list.

Customize as needed.

Click **Apply**.

## Apply Transformations

To apply transformation rules to existing data:

Click **Apply Transformations** from the *Select Actions* menu.

Select the scope (all devices, specific types, or date range).

Select which rules to apply.

Click **Execute**.
