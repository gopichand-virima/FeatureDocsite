# Viewing Intune Import Record

Use this function to view the imported Intune resources

1.  Select an import record to open its details.

2.  Report window displays imported CI(s) in these categories:

    - All

    - Authorized

    - Unauthorized

    <span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Import-from-Intune/Import%20INTUNE%20Window_8.png" original-image-title=""></span>

- <span class="Strong" style="font-weight: bold;">All</span> – Every resource returned by the import.

- <span class="Strong" style="font-weight: bold;">Authorized</span> – Assets that are already present in the CMDB.

  <span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Import-from-Intune/Import%20INTUNE%20Window_9.png" original-image-title=""></span>

<!-- -->

- <span class="Strong" style="font-weight: bold;">Unauthorized</span> – New assets not yet listed in the CMDB.

  <span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Import-from-Intune/Import%20INTUNE%20Window_10.png" original-image-title=""></span>

# <span class="Strong">Common controls (top-right of the grid)</span>

- <span class="Strong" style="font-weight: bold;">Move to CMDB</span> – Promote the selected records to CMDB (Refer Move to CMDB).

- <span class="Strong" style="font-weight: bold;">Auto Refresh</span> – Refresh the grid on set periodic interval. Use this when import is still in progress so that the imported items show up without manual refresh.

# Key columns

- <span class="Strong" style="font-weight: bold;">Blueprint Name</span> – The discovery blueprint that matched the resource (e.g., <span class="Emphasis">Intune Virtual Network</span>, <span class="Emphasis">Intune Neo4j</span>)

- <span class="Strong" style="font-weight: bold;">Asset Name</span> – The discovered resource name (cluster/service/task/instance, etc.).

  - The asset name is the Intune instance ID.

- <span class="Strong" style="font-weight: bold;">Status</span>: Indicates the current state of the CI imported from Intune. When a CI is newly imported (does not exist in CMDB), the status is shown as "New."

- <span class="Strong" style="font-weight: bold;">Asset ID</span>: A unique, sequential identification number assigned to each discovered Intune record. The ID shall have a prefix “DSC”+ 6 digits Unique sequential number.

- <span class="Strong" style="font-weight: bold;">Host Name: </span>The host or endpoint name captured by discovery.

Users can select which columns to display when importing Intune and sort the grid by any column.

# Move items to the CMDB

After importing Intune data, promote it to the CMDB.

1.  In <span class="Strong" style="font-weight: bold;">All</span>, <span class="Strong" style="font-weight: bold;">Authorized</span> or <span class="Strong" style="font-weight: bold;">Unauthorized</span>, select one or more rows.

2.  Click <span class="Strong" style="font-weight: bold;">Move to CMDB</span>.

    <span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Import-from-Intune/Import%20INTUNE%20Window_11.png" original-image-title=""></span>

<!-- -->

3.  In the confirmation dialog, click <span class="Strong" style="font-weight: bold;">Continue</span>.  
    The move job runs <span class="Strong">in the background</span>—no further action is required. Use <span class="Strong" style="font-weight: bold;">Auto Refresh</span> to watch status changes.

Tip: If no rows are selected, your environment will move <span class="Strong">all records that match the current filters</span>. If you want to limit scope, always select the exact rows to move.

4.  Upon successful move to CMDB, the records will be deleted from “Import from Intune” and “Discovered Items” View.
