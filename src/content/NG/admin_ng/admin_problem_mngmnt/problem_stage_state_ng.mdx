# Problem Stage and State

Use this function to manage problem lifecycle stages and their associated states.

The following stages and states are pre-configured. These can be modified to match your organization's problem management process.

| Order | Problem Stage | Problem States | Description |
| 1 | New | - New<br>- Assigned | Initial stage when a problem is identified. |
| 2 | Investigation | - Root Cause Analysis<br>- In Progress<br>- Pending | Stage when the problem is being investigated to find root cause. |
| 3 | Known Error | - Known Error<br>- Workaround Available | Problem identified as a known error with documented workaround. |
| 4 | Resolution | - Fix in Progress<br>- Testing<br>- Pending Validation | Stage when a permanent solution is being implemented. |
| 5 | Closed | - Resolved<br>- Closed | Final stage when the problem is permanently resolved. |

In the main window, click **Admin > Problem Management > Problem Stage and State**. The Problem Stage and State window displays.

## Stage Functions

### New Problem Stage

From the *Select Actions* drop-down list, choose **New Problem Stage**. The New Problem Stage window displays.

Enter a *Name* and *Description*.

In the *Stage Order* field, click the drop-down list and select the order for this stage.

Specify if this stage should trigger any automated actions:

- Send notifications
- Create known error record
- Update related incidents
- Change assignment

Click **Add**.

### Edit Problem Stage

Navigate to the window containing the stage record to edit.

Click the line containing the stage. The stage window displays.

Make the necessary modifications.

Click **Save**.

### Delete Problem Stage

Click the line that contains the stage to delete.

From the *Select Actions* drop-down list, choose **Delete**.

## State Functions

### New Problem State

Click the line item containing the stage to add a corresponding state(s).

In the new window, click the **State** tab.

Click **New Problem State**.

Enter a *Name*, *Description* and *Action Name*.

Configure state properties:

**Notify**: Click **Add** to select users or roles to notify when a problem enters this state.

**Approver Roles**: Click **Add** to select roles that can approve transitions from this state (if approval is required).

**Requires RCA**: Check if Root Cause Analysis is required in this state.

**Create Known Error**: Check if a known error record should be created when entering this state.

When all selections/entries are made, click **Save**.

### Edit Problem State

Select the stage containing the state.

Click the **State** tab.

Click the line containing the state to edit.

Make the necessary modifications.

Click **Save**.

### Delete Problem State

Select the stage containing the state.

Click the **State** tab.

Select the state to delete.

From the *Select Actions* drop-down list, choose **Delete**.

## State Transitions

Configure which state transitions are allowed:

Click **Manage Transitions**.

For each state, define:

- Allowed next states
- Required fields for transition
- Approval requirements
- Notification triggers

Click **Save**.
