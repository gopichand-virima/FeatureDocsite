# Import CI’s

# Import CI without using existing Field Mapping Template

1.  From the <span class="Strong" style="font-weight: bold;">Select Actions</span> menu, click <span class="Strong" style="font-weight: bold;">Import CI's</span>.

    <span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Import-Data-Files/Import%20Data%20Files_1_3.png" original-image-title=""></span>

<!-- -->

2.  The <span class="Strong" style="font-weight: bold;">Attach CI Data Files</span> window appears, allowing you to proceed with uploading and mapping your configuration item data files.

    <span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Import-Data-Files/Import%20Data%20Files_1_4.png" original-image-title=""></span>

<!-- -->

3.  In the <span class="Strong" style="font-weight: bold;">Attach CI Data Files</span> window:

    - Select a mapping type as None

      - <span class="Strong" style="font-weight: bold;">None</span> – Import without a template.

    - Click <span class="Strong" style="font-weight: bold;">Browse</span> to select a file (.xls, .xlsx, or .csv).

    - (Optional) Downlo<span class="inline-comment-marker">ad the sample template for reference.</span>

    <span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Import-Data-Files/Import%20Data%20Files_1_5.png" original-image-title=""></span>

<span class="Strong" style="font-weight: bold;">Expected Columns & What to Enter</span>

- <span class="Strong" style="font-weight: bold;">Asset Name</span>  
  The name of the asset (e.g., server host + IP).

- <span class="Strong" style="font-weight: bold;">IP Address</span> <span class="Emphasis">(possible correlator)</span>  
  The host’s primary IP (e.g., 10.14.80.36). If your blueprint uses IP as a correlator, this helps match to an existing CI.

- <span class="Strong" style="font-weight: bold;">Host Name</span> <span class="Emphasis">(possible correlator)</span>  
  Host name of the device (e.g., ADSERVERLD). Another common correlator.

- <span class="Strong" style="font-weight: bold;">Mac Address</span> <span class="Emphasis">(possible correlator)</span>  
  Physical MAC address (e.g., 00:50:56:B4:01:5). Use standard notation and only one MAC per row.

- <span class="Strong" style="font-weight: bold;">Blueprint Name</span> <span class="Strong">(Mandatory)</span>  
  The CMDB blueprint that defines the CI’s schema (e.g., <span class="Strong">Windows Server</span>). Must exactly match an existing blueprint name.

- <span class="Strong" style="font-weight: bold;">Asset Id</span> <span class="Emphasis">(Mandatory only for updates)</span>  
  The CMDB ID of an <span class="Strong">existing</span> CI (e.g., AST003156).

- <span class="Strong" style="font-weight: bold;">Acquisition Date</span>  
  A datetime field for asset lifecycle. <span class="Strong">Format must be</span> MM/DD/YYYY HH:MM:SS (e.g., 12/22/2022 07:00:00 AM).

<span class="Strong" style="font-weight: bold;">Mandatory Import Rules</span>

- <span class="Strong" style="font-weight: bold;">For New CIs:</span>

  - <span class="Strong">Blueprint Name</span> is required.

  - At least <span class="Strong">one correlator</span> (IP Address, Host Name, or Mac Address, as defined by the blueprint) must be present.

- <span class="Strong" style="font-weight: bold;">For Updating Existing CIs:</span>

  - <span class="Strong">Asset Id</span> is required.

  - Correlators (IP, Host Name, MAC) help validate matches but are not strictly required if Asset Id is provided.

- <span class="Strong" style="font-weight: bold;">Date/Time Fields:</span>

  - Must follow the format MM/DD/YYYY HH:MM:SS.

4.  Click <span class="Strong" style="font-weight: bold;">Upload</span>.

5.  A preview of the data to import appears, showing how each attribute is mapped to the column headers in your sample file.

<span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Import-Data-Files/Import%20Data%20Files_1_6.png" original-image-title=""></span>

6.  If the default mapping is incorrect, use the dropdown to change the attribute name.

7.  Click <span class="Strong" style="font-weight: bold;">Submit</span> to proceed.

    - Ensure all correlator properties are mapped.

    - If a property is highlighted in <span class="Strong">red</span>, it means it is not properly mapped to an application field; use <span class="Strong">Skip Column</span> if you do not wish to map that property.

    - If a property is highlighted in <span class="Strong">green</span>, it means it is successfully mapped to an application field.

- If a record in your Excel (or CSV) import file does not have a value for the “Blueprint Name” column, that record will not appear in the Import Data Files.

- If a record has a “**Blueprint Name**” but does not have any correlator (such as IP Address, Host Name, or MAC Address—as required by the blueprint), that record will also not be shown in the Import Data Files.

8.  When prompted, a confirmation pop-up will ask if you want to save the current mapping as a template.

<span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Import-Data-Files/Import%20Data%20Files_1_7.png" original-image-title=""></span>

9.  Click <span class="Strong" style="font-weight: bold;">Yes</span> to save the template.

10. Enter a name for your template in the pop-up and click <span class="Strong" style="font-weight: bold;">Save and Continue</span>.

    - You can reuse a saved field mapping template for future CI imports as long as the selected blueprint matches the blueprint associated with the template.

<span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Import-Data-Files/Import%20Data%20Files_1_8.png" original-image-title=""></span>

11. After import, a new row appears in the file list, displaying the imported file name, status, uploaded on, and uploaded by fields.

- The system shows an error message if a user tries to upload without a file.

<span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Import-Data-Files/Import%20Data%20Files_1_9.png" original-image-title=""></span>

# Import CI using existing Field Mapping Template

1.  From the <span class="Strong" style="font-weight: bold;">Select Actions</span> menu, click <span class="Strong" style="font-weight: bold;">Import CI’s</span>.

<span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Import-Data-Files/Import%20Data%20Files_1_3.png" original-image-title=""></span>

2.  The <span class="Strong" style="font-weight: bold;">Attach CI Data Files</span> window appears, allowing you to proceed with uploading and mapping the configuration item data files.

<span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Import-Data-Files/Import%20Data%20Files_1_10.png" original-image-title=""></span>

3.  In the <span class="Strong" style="font-weight: bold;">Attach CI Data Files</span> window:

    - Select a mapping type:

      - <span class="Strong" style="font-weight: bold;">Choose Template</span> – Use a predefined import template.

      - Click <span class="Strong" style="font-weight: bold;">Choose Template</span>, then select a template from the dropdown.

    - Click <span class="Strong" style="font-weight: bold;">Browse</span> to select a file (.xls, .xlsx, or .csv).

    - (Optional) Download the sample template for reference.

4.  Click <span class="Strong">**Upload**.</span>

- If a user attempts to upload a file in an unsupported format such as .txt on the Import Data Files page, the system immediately rejects the file and displays an error message.

<span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Import-Data-Files/Import%20Data%20Files_1_11.png" original-image-title=""></span>

5.  A preview of the data to import displays, with attributes mapped to column headers in the sample file.

<span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Import-Data-Files/Import%20Data%20Files_1_6.png" original-image-title=""></span>

6.  Click the dropdown to change the attribute name if the default mapping is incorrect.

7.  Click on <span class="Strong">**Submit**.</span>

- Ensure all correlator properties are mapped.

- If a property is highlighted in <span class="Strong" style="font-weight: bold;">red</span>, it means it is not properly mapped to an application field; use <span class="Strong">Skip Column</span> if you do not wish to map that property.

- If a property is highlighted in <span class="Strong" style="font-weight: bold;">green</span>, it means it is successfully mapped to an application field.

8.  A confirmation pop-up asks you to save the current mapping as a template.

<span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Import-Data-Files/Import%20Data%20Files_1_7.png" original-image-title=""></span>

9.  Click <span class="Strong" style="font-weight: bold;">Yes </span>to save a template.

10. A pop-up prompts you to name the template.

11. Enter a name and click <span class="Strong" style="font-weight: bold;">Save and Continue</span>.

    1.  The system saves the template with the current mapping.

    2.  Use the saved template for future CI imports.

<span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Import-Data-Files/Import%20Data%20Files_1_8.png" original-image-title=""></span>

12. A new row is displayed with imported file name, status, uploaded on and uploaded by fields.

    - <span class="Strong" style="font-weight: bold;">File Name</span>: Displays the name of the uploaded Excel file.

    - <span class="Strong" style="font-weight: bold;">Status</span>: Shows the current state of the file <span class="inline-comment-marker">(e.g., Processed, Processing).</span>

    - <span class="Strong" style="font-weight: bold;">Uploaded By</span>: Indicates which user uploaded the file.

    - <span class="Strong" style="font-weight: bold;">Uploaded On</span>: Timestamp showing the exact date and time of the upload.

13. Review imported CIs in the detailed view before moving them into the CMDB.
