# Request Timescale

Use this function to configure a resolution according to a priority basis.

The following applies to this function:

<div class="Bullet1">

A Request must be either in Assigned or Work In Progress state before the Response Time ends.

</div>

<div class="Bullet1">

Auto Escalation process is triggered if assignee doesn’t resolve a Request from Assigned/Work-In-Progress to Fulfilled/Closed state before the Resolution Time ends.

</div>

<div class="Bullet1">

Notification email will be delivered to the configured user in case of Before/After the Response/Resolution Time violation.

</div>

<div class="Bullet1">

Response time and the Resolution Time are defined as per the Service Level Agreements between the Client and the Service Provider.

</div>

<div class="Bullet1">

If the support level teams do not resolve a Request within the specified time frame, then it is declared as SLA breach

</div>

<div class="Bullet1">

SLA time will not be recalculated on the following case: Root cause CI/ Assignee is changed.

</div>

In the main window, click **Admin /> Request Management /> Request Timescale**. The Request Timescale window displays.

In the main window, click the **gear** icon, and select **Request Management /> Request Timescale**.
