# Incident Types

Use this function to configure different types of incidents with customized color codes for visual identification.

Incident types help categorize incidents based on their nature, source, or handling requirements. Each type can be assigned a unique color code for easy visual recognition in lists and dashboards.

In the main window, click **Admin > Incident Management > Incident Types**. The Incident Type window displays.

## New Incident Type

From the *Select Actions* drop-down list, choose **New Incident Type**. The New Incident Type window displays.

Enter a *Name* and *Description* for this type.

In the *Color* field, click on the color and choose a color from the picker OR type a 6-digit hex code.

Configure additional properties:

**Default Priority**: Set a default priority for this incident type (optional).

**Default Category**: Set a default category for this incident type (optional).

**Requires Approval**: Check if incidents of this type require approval before closure.

**Auto-Assign to Group**: Select a default assignment group (optional).

Click **Add**.

## Common Incident Types

Examples of incident types you might configure:

- **Service Request**: User requests for standard services
- **System Outage**: Critical system failures
- **Performance Issue**: Degraded system performance
- **Security Incident**: Security-related incidents
- **Network Issue**: Network connectivity problems
- **Hardware Failure**: Physical equipment failures
- **Software Bug**: Application defects
- **Access Request**: User access and permissions
- **Password Reset**: Password-related requests
- **Data Issue**: Data quality or integrity problems

## Edit Incident Type

Navigate to the window containing the record to edit.

Click the line containing the record. The applicable window displays.

Make the necessary modifications.

Click **Save**.

## Delete Incident Type

Deleting is a permanent action and cannot be undone. Ensure no incidents are currently using this type before deletion.

Click the line that contains the item to delete.

From the *Select Actions* drop-down list, choose **Delete**.

## Type-Based Routing

To configure automatic routing based on incident type:

Select an incident type.

Click **Configure Routing**.

Define routing rules:

- **Assignment Group**: Auto-assign to specific group
- **Assignee**: Auto-assign to specific user
- **Workflow**: Attach specific workflow
- **Notification Template**: Use custom notification

Click **Save**.
