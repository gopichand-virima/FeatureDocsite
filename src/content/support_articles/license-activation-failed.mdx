# Virima License Activation Failed Error

**Article ID:** K000246234  
**Last Modified:** December 02, 2025  
**Category:** Licensing  
**Severity:** Critical

---

## Issue

When attempting to activate a Virima license, the activation process fails with one of the following error messages:

- "License activation failed: Invalid license key"
- "Unable to contact licensing server"
- "License key already in use"
- "Subscription not found or expired"
- "Module entitlements not available"

## Environment

- **Product:** Virima Platform (All Modules)
- **Versions:** 5.13, 6.1, 6.1.1, NextGen
- **Deployment:** SaaS, On-Premises, Hybrid
- **License Types:** Enterprise, Professional, Standard, Trial

## Symptoms

- License activation wizard shows error message
- Cannot access Virima modules or features
- Platform shows "License Expired" or "No Valid License" warning
- Module functionality restricted or unavailable
- Trial license not converting to paid subscription
- License status shows "Invalid" or "Inactive" in admin panel

## Cause

License activation can fail due to:

1. **Invalid or Incorrect License Key**
   - Typo in license key entry
   - Copy/paste including extra spaces or line breaks
   - Wrong license key for the instance
   - License key from different environment (dev vs. prod)

2. **License Server Connectivity Issues**
   - Firewall blocking outbound HTTPS to license.virima.com
   - Proxy server not configured
   - DNS resolution failure for licensing server
   - SSL certificate validation error

3. **License Already Activated**
   - License key already in use on another instance
   - License not properly deactivated before reinstallation
   - Maximum activations reached

4. **Subscription Status Problems**
   - Subscription expired or cancelled
   - Payment failure causing suspension
   - Subscription not yet provisioned in system
   - Purchase order not processed

5. **Instance Identification Mismatch**
   - Instance ID changed after hardware replacement
   - VM snapshot/clone with duplicate instance ID
   - On-premises to SaaS migration without deactivation

6. **Module Entitlement Issues**
   - License key doesn't include requested modules
   - Module count exceeds license limits
   - Feature flag not enabled for subscription tier

## Resolution

### Step 1: Verify License Key Format

License keys follow this format:
```
VIRIMA-XXXXX-XXXXX-XXXXX-XXXXX-XXXXX
```

**Common entry mistakes:**
- Extra spaces: `VIRIMA-XXXXX- XXXXX-XXXXX` ❌
- Missing hyphens: `VIRIMAXXXXXXXXXXXXXXXXX` ❌
- Wrong case: `virima-xxxxx-xxxxx` ❌
- Incomplete: `VIRIMA-XXXXX-XXXXX-XXXXX` ❌

**Correct license key entry:**
```
VIRIMA-A1B2C-D3E4F-G5H6I-J7K8L-M9N0P
```

**Tips:**
- Copy from original email (not PDF which may add line breaks)
- Paste into notepad first to remove formatting
- Verify no trailing spaces
- Ensure all 5 segments are present

### Step 2: Check Network Connectivity to License Server

**Test connectivity:**
```bash
# Test HTTPS connection to license server
curl -v https://license.virima.com/api/v1/health

# Expected response:
# HTTP/1.1 200 OK
# {"status": "healthy", "timestamp": "2025-12-02T10:30:00Z"}

# Check DNS resolution
nslookup license.virima.com

# Expected output:
# Server: 8.8.8.8
# Address: 8.8.8.8#53
# 
# Non-authoritative answer:
# Name: license.virima.com
# Address: 54.xxx.xxx.xxx
```

**If connection fails, check firewall:**

Required outbound rules:

| Protocol | Port | Destination | Purpose |
|----------|------|-------------|---------|
| HTTPS | 443 | license.virima.com | License validation |
| HTTPS | 443 | *.virima.com | Platform services |
| DNS | 53 | DNS Server | Name resolution |

**Windows Firewall:**
```powershell
# Allow outbound HTTPS to Virima licensing
New-NetFirewallRule -DisplayName "Virima License Server" `
  -Direction Outbound `
  -Protocol TCP `
  -RemoteAddress license.virima.com `
  -RemotePort 443 `
  -Action Allow
```

**Linux (iptables):**
```bash
# Allow outbound HTTPS
sudo iptables -A OUTPUT -p tcp --dport 443 -d license.virima.com -j ACCEPT
```

### Step 3: Configure Proxy Settings

If your organization requires proxy for internet access:

**SaaS Platform:**
1. Go to **Settings → System → Network**
2. Enable **Use Proxy Server**
3. Configure:
   - **Proxy URL:** http://proxy.company.com:8080
   - **Proxy Username:** (if required)
   - **Proxy Password:** (if required)
4. **Bypass Proxy For:** (leave empty for Virima)
5. Click **Test Connection**
6. Click **Save**

**On-Premises (Configuration File):**

Edit `/opt/virima/config/system.conf`:
```ini
[proxy]
enabled = true
host = proxy.company.com
port = 8080
username = proxyuser
password = proxypass
bypass_list = localhost,127.0.0.1,*.internal.com
```

Restart Virima service:
```bash
sudo systemctl restart virima
```

### Step 4: Verify License Status in Portal

1. Log in to [Virima Customer Portal](https://portal.virima.com)
2. Navigate to **My Subscriptions**
3. Locate your subscription
4. Verify:
   - **Status:** Active (not Suspended, Expired, or Cancelled)
   - **Activation Count:** Under maximum allowed
   - **Valid Until:** Future date
   - **Modules:** Correct modules listed

**If subscription shows "Suspended":**
- Check for payment issues
- Contact billing@virima.com
- Resolve any outstanding invoices

**If activation count is at maximum:**
- Deactivate unused instances (see Step 5)
- Or contact sales to increase activation limit

### Step 5: Deactivate License from Previous Instance

**If reinstalling or migrating:**

**Option A: Deactivate from old instance (preferred)**

1. Log in to old instance
2. Go to **Settings → License → Manage License**
3. Click **Deactivate License**
4. Confirm deactivation
5. Activation slot is now available

**Option B: Deactivate from Customer Portal**

1. Log in to [Virima Customer Portal](https://portal.virima.com)
2. Navigate to **My Subscriptions → Active Instances**
3. Locate the instance to deactivate
4. Click **⋮** (More Actions)
5. Select **Deactivate License**
6. Confirm with reason (e.g., "Hardware replacement")
7. Activation slot freed within 5 minutes

**Option C: Contact Support (if instance inaccessible)**

Email support@virima.com with:
- License key
- Instance ID or hostname
- Reason for deactivation
- Confirmation you've backed up data if applicable

### Step 6: Manual License Activation

**Web UI Method:**

1. Log in to Virima instance as Administrator
2. Go to **Settings → License**
3. Click **Activate License**
4. Enter license key: `VIRIMA-A1B2C-D3E4F-G5H6I-J7K8L-M9N0P`
5. Select **Activation Method:** Online
6. Click **Activate**
7. Wait for confirmation message
8. Verify license status shows "Active"

**Offline Activation (air-gapped environments):**

1. On Virima instance, go to **Settings → License**
2. Click **Activate License → Offline Activation**
3. **Copy Instance ID** displayed on screen
4. On internet-connected computer:
   - Go to [https://license.virima.com/offline](https://license.virima.com/offline)
   - Enter license key
   - Enter instance ID
   - Click **Generate Activation File**
   - Download `virima-activation.lic`
5. Transfer `virima-activation.lic` to Virima instance (USB, secure file transfer)
6. In Virima UI, click **Upload Activation File**
7. Select downloaded file
8. Click **Activate**

**CLI Method (On-Premises):**

```bash
# Activate license via command line
sudo /opt/virima/bin/license-manager activate \
  --key "VIRIMA-A1B2C-D3E4F-G5H6I-J7K8L-M9N0P" \
  --online

# Check license status
sudo /opt/virima/bin/license-manager status

# Expected output:
# License Status: Active
# License Type: Enterprise
# Valid Until: 2026-12-31
# Modules: Discovery, CMDB, AssetManagement, ServiceMapping, CloudDiscovery
# Activations: 1/5
```

### Step 7: Verify Module Entitlements

**Check licensed modules:**

1. Go to **Settings → License → Entitlements**
2. Verify all required modules show "Enabled"

**Common module codes:**

| Module | Code | Description |
|--------|------|-------------|
| IT Discovery | DISC | Network and agent-based discovery |
| CMDB | CMDB | Configuration management database |
| Asset Management | ASST | Hardware and software asset tracking |
| Service Mapping | SMAP | Application dependency mapping |
| Cloud Discovery | CLUD | AWS, Azure, GCP discovery |
| Software License Management | SWLM | License compliance |
| IT Operations | ITOP | Monitoring and analytics |
| Vulnerability Management | VULN | Security vulnerability tracking |
| Contract Management | CNTR | Contract and vendor management |
| Business Service Management | BSVC | Business service modeling |

**If module is not entitled:**

- Verify your subscription tier includes the module
- Check if module requires separate add-on purchase
- Contact sales@virima.com to add module to subscription

### Step 8: Clear License Cache

**If license shows as invalid after successful activation:**

**SaaS Platform:**
1. Go to **Settings → License**
2. Click **⋮** (More Actions) → **Clear License Cache**
3. Confirm action
4. Wait 2-3 minutes
5. Refresh page
6. Verify license status

**On-Premises:**
```bash
# Stop Virima service
sudo systemctl stop virima

# Clear license cache
sudo rm -rf /opt/virima/data/cache/license/*

# Start Virima service
sudo systemctl start virima

# Verify license status
sudo /opt/virima/bin/license-manager status
```

### Step 9: Check System Clock

License validation requires accurate system time:

**Verify system time:**
```bash
# Linux
date
timedatectl status

# Should show correct time within 5 minutes of actual

# Windows
Get-Date
w32tm /query /status
```

**If time is incorrect:**

**Linux:**
```bash
# Enable NTP synchronization
sudo timedatectl set-ntp true

# Or manually set time
sudo timedatectl set-time '2025-12-02 10:30:00'

# Set timezone
sudo timedatectl set-timezone America/New_York
```

**Windows:**
```powershell
# Sync with time server
w32tm /resync /force

# Configure NTP server
w32tm /config /manualpeerlist:"time.windows.com" /syncfromflags:manual /update
```

### Step 10: Request Trial License Extension

**If trial expired before evaluation complete:**

1. Email support@virima.com with:
   - Company name
   - Instance URL or ID
   - Original trial start date
   - Reason for extension request
   - Estimated evaluation completion date

2. Include business case:
   - Features being evaluated
   - Integration requirements
   - Timeline for purchase decision

**Trial licenses typically extended for:**
- 14 days (standard)
- 30 days (with justification)
- 60 days (complex evaluations with executive approval)

## Verification

After successful license activation:

1. License status shows **"Active"** with green indicator
2. All purchased modules appear under Entitlements
3. No license warning banners in UI
4. Module functionality fully accessible
5. License expiration date shows correct future date

**Verify via CLI:**
```bash
sudo /opt/virima/bin/license-manager status --verbose

# Expected output includes:
# ✓ License is valid and active
# ✓ All modules entitled
# ✓ Activation successful
# ✓ License server reachable
# ✓ Next validation: [timestamp]
```

## Additional Information

### License Validation Frequency

| License Type | Validation Frequency | Grace Period |
|--------------|---------------------|--------------|
| SaaS | Every 24 hours | 72 hours |
| On-Premises | Every 7 days | 30 days |
| Trial | Every 24 hours | 24 hours |
| Offline | Every 90 days | N/A |

### License Activation Limits

| Subscription Tier | Concurrent Activations |
|-------------------|------------------------|
| Enterprise | 5 production + unlimited non-prod |
| Professional | 2 production + 2 non-prod |
| Standard | 1 production + 1 non-prod |
| Trial | 1 instance |
