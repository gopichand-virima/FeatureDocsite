# Manage Knowledge

Effective knowledge management ensures articles remain current, accurate, and useful.

## Knowledge Lifecycle

### Draft
- Article being created
- Not visible to users
- Can edit freely
- Save progress

### Pending Approval
- Submitted for review
- Awaiting approval
- Cannot edit
- Reviewers notified

### Published
- Live in knowledge base
- Visible to users
- Searchable
- Available for use

### Under Review
- Scheduled review
- Being updated
- May remain published
- Version control

### Retired
- No longer current
- Not in search results
- Archived for reference
- Historical record

## Managing Articles

### Edit Article
1. Find article
2. Click **Edit**
3. Make changes
4. Add change notes
5. Submit for approval

### Version Control
- Track all versions
- View change history
- Revert if needed
- Compare versions

### Article Reviews
Schedule regular reviews:
1. Set review frequency
2. Assign reviewer
3. System sends reminders
4. Update as needed

### Archive Articles
When no longer needed:
1. Select article
2. Click **Archive**
3. Add reason
4. Confirm archiving

## Quality Management

### Article Ratings
Users rate articles:
- 1-5 star rating
- Average displayed
- Helps identify quality
- Guides improvements

### User Feedback
Collect comments:
- Was this helpful?
- Suggestions
- Issues found
- Missing information

### Usage Analytics
Track metrics:
- Views
- Search hits
- Problem resolution rate
- Time on page

### Quality Review
Regular assessment:
- Accuracy check
- Completeness review
- Clarity evaluation
- Update recommendations

## Categories and Tags

### Organize by Category
- Hardware
- Software
- Network
- Procedures
- Best Practices
- Troubleshooting

### Tag Articles
Add searchable tags:
- Product names
- Error codes
- Symptoms
- Solutions

### Category Management
1. Navigate to **Admin** > **Knowledge**
2. Manage categories
3. Add/edit/delete
4. Set permissions
