# CI: Relationships

Use this function to manage the relationships between sources and targets. The Discovery application detects the Connected To, Runs On, Communicates With, Installed On, Component Of, Physically Connected To, Attaches and Used By relationships during a scan.


Refer to [CMDB Process Jobs](/itsm/config_mngmt/config_items/config_item_details/cmdb_process_jobs_ng.mdx) for more details.


From the navigation pane, select **ITSM > Configuration Management > Configuration Items**. The Configuration Items window displays.

From the navigation pane, select **Configuration Management > Configuration Items**. The Configuration Items window displays.

Select a record in the list. A new window opens and the **Details** tab displays.

Click the **Relationships** tab. A list of all existing relationships displays.

! !

Choose any of the following: **Add Source**, **Add Target**, or **[Service Map](/itsm/config_mngmt/conf_items/service_map_ng.mdx)**.

To remove any relationship, select it in the list, and click **Delete**.

## Add Source or Add Target

Click the **Add Source** or **Add Target button**. The Add Relation dialog box displays.

! !

From the drop-down list, select either the **Source** or **Target**.

From the drop-down list, select the applicable blueprint from the available list.

In the *Search* field, enter the search criteria to locate the corresponding asset. See also [Advanced Search](/common_topics/advanced_search_ng.mdx).

From the *Choose Relationship* drop-down list, select the relationship the Source or Target will have such as Assigned To, Belongs To, and so forth.

From the *Relation* drop-down list, select the relationship the Source or Target will have such as Assigned To, Belongs To, and so forth.

When all selections are made, click **Add**.
