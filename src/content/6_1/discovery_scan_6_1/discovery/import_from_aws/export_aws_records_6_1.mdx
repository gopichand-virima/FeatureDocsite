# Exporting AWS Records

Exports data for the selected imported AWS record as an Excel spreadsheet and sends an email notification when the spreadsheet is ready for download.

- From the <span class="Strong">Select Actions</span> drop-down list, choose <span class="Strong">Export</span>.

  <span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Import-From-AWS/Import%20From%20AWS_1_12.png" original-image-title=""></span>

<!-- -->

- Click <span class="Strong">Continue.</span>

- The system downloads an<span class="inline-comment-marker"> Excel (.xlsx) file of</span> the list with the details,

  - <span class="Strong">Unique ID</span> – The Virima import record ID (e.g., AWS000578).

  - <span class="Strong">Account Id</span> – The "Account Id" column displays the value entered for any credential type (Account Id, Organization Unit Id, or Root Id).

  - <span class="Strong">Credential</span> – The credential set used (e.g., aws, AWS_ACCOUNT).

  - <span class="Strong">Imported By</span> – The user who <span class="inline-comment-marker">initiated the import (e.g., </span><span class="inline-comment-marker_1">Admin Virima</span><span class="inline-comment-marker">).</span>

    - <span class="inline-comment-marker">Manual Import: The logged-in user who initiated a manual import.</span>

    - <span class="inline-comment-marker">Scheduled Import: The logged-in user who initiated a scheduled import.</span>

  - <span class="Strong">Name</span> – The AWS service or resource type imported (e.g., nat, load, AWS Batch, ECS, AWS EKS6).

  - <span class="Strong">Imported On</span> – The timestamp when that record was imported (as displayed in the UI).
