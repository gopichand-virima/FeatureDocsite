# Accessing the Import AWS Window

Use this procedure to view and manage imported AWS resource metadata in Virima.

1.  In the left navigation pane, expand <span class="Strong">Discovery Scan</span>.

2.  Select <span class="Strong">Import From AWS</span>.

    <span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Import-From-AWS/Import%20From%20AWS_1.png" original-image-title=""></span>

<!-- -->

3.  <span class="Strong">Import From AWS</span> page opens.

The page displays a list of <span class="Strong">imported runs </span>from your AWS accounts. Each row represents a single import run.

<span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Import-From-AWS/Import%20From%20AWS_1_1.png" original-image-title=""></span>

- <span class="Strong">Key columns:</span>

  - <span class="Strong">Id</span>: Unique identification ID of the import run (e.g., AWS000686), records imported from AWS will have the prefix “AWS” followed by a six-digit sequential number.

  - <span class="Strong">Name:</span>

    - Manual Import: The user provided name used to identify the import run.

    - Scheduled imports: The user-provided schedule name appears under the "Name" column.

  - <span class="Strong">Account Id</span><span class="inline-comment-marker">: While adding credentials for AWS account configuration, regardless of the credential type selected (Account Id, Organization Unit Id, or Root Id), the value entered will always be displayed in the "Account Id" column.</span>

  - <span class="Strong">Imported On</span>: The date and time when the import was completed.

  - <span class="Strong">Status</span><span class="inline-comment-marker">: Current state (e.g., Importing, Completed, Failed).</span>

    - <span class="Strong">Importing:</span> The import process from AWS is currently running and in progress.

    - <span class="Strong">Completed:</span> The import from AWS has finished successfully.

    - <span class="Strong">Failed:</span> The import has failed, due to some unknown errors.

      - View failure details from the AWS logs.

  - <span class="Strong">Imported By</span><span class="inline-comment-marker">: </span>

    - <span class="inline-comment-marker">Manual Import: The logged-in user who initiated a manual import.</span>

    - <span class="inline-comment-marker">Scheduled Import: The logged-in user who configured scheduled import.</span>

  - <span class="Strong">Credential</span>: Virima credential profile used to connect to AWS.

  - <span class="Strong">Select Actions</span>: Perform bulk operations on selected or filtered rows.

- <span class="inline-comment-marker">From the </span><span class="Strong">Select Actions</span><span class="inline-comment-marker"> drop-down list, choo</span>se the operation you want to perform.

  <span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Import-From-AWS/Import%20From%20AWS_1_2.png" original-image-title=""></span>

<!-- -->

- <span class="Strong">Delete</span><span class="inline-comment-marker">: Remove selected import record</span>

  - Selected import record(s) shall be deleted

  - CI(s) imported during the selected import shall be removed from Discovered Items

  - CI(s) moved to CMDB will still appear under CMDB after deleting the AWS import record.

- <span class="Strong">Export</span> – Download grid contents (selected or filtered rows) to Excel.

  - If you have selected rows, only the selected rows are exported.

  - If you have applied filters<span class="inline-comment-marker">, all </span>rows that <span class="inline-comment-marker">match the current filters are exported.</span>

  - If no filters are applied and no rows are selected, exporting will include all records from all pages.

  - <span class="inline-comment-marker">The file includes the grid columns (e.g., </span><span class="Strong">Unique ID</span><span class="inline-comment-marker">, </span><span class="Strong">Account Id</span><span class="inline-comment-marker">, </span><span class="Strong">Credential</span><span class="inline-comment-marker">, </span><span class="Strong">Imported By</span><span class="inline-comment-marker">, </span><span class="Strong">Name</span><span class="inline-comment-marker">, and </span><span class="Strong">Imported On</span><span class="inline-comment-marker">.) (Refer </span>Exported Excel Colum Details<span class="inline-comment-marker">)</span>

This downloads the import records NOT the CI(s).

- <span class="Strong">Import</span><span class="inline-comment-marker"> – Initiate a new Import from AWS.</span>
