# CMDB - User Manual

Complete guide for managing your Configuration Management Database in Virima 5.13.

## Overview

The CMDB serves as the single source of truth for your IT infrastructure, tracking assets, services, and their interdependencies.

## CI Management

### CI Types
- Hardware (Servers, Workstations, Network Equipment)
- Software (Applications, Licenses, Databases)
- Services (Business Services, IT Services)
- Documentation (Policies, Procedures, Manuals)

### CI Attributes
Each CI contains:
- Identification information
- Technical specifications
- Ownership and responsibility
- Lifecycle status
- Custom attributes

## Relationship Management

### Relationship Types
- **Runs On**: Application runs on Server
- **Depends On**: Service depends on Application
- **Connected To**: Network connections
- **Uses**: Resource utilization
- **Part Of**: Component relationships

## Best Practices

- Maintain accurate and up-to-date CI information
- Regularly review and validate relationships
- Use standardized naming conventions
- Leverage automated discovery where possible
- Implement proper access controls
- Regular data quality audits
