# Viewing a discovered Intune record

1.  In <span class="Strong" style="font-weight: bold;">Import from Intune</span>, open the <span class="Strong" style="font-weight: bold;">All</span> tab. Click any row (e.g., an Asset Name).

2.  The <span class="Strong" style="font-weight: bold;">Discovered Item</span> view for that asset opens.

# Discovered item view overview

<span class="Strong" style="font-weight: bold;">Primary Details (left panel)</span>  
Quick facts about the CI:

- <span class="Strong" style="font-weight: bold;">Asset ID</span> – Virima’s unique identifier for the discovered CI (e.g., DSC69303). Use this to search or reference the record.

- <span class="Strong" style="font-weight: bold;">Blueprint</span> – The discovery blueprint that matched the resource (e.g., <span class="Emphasis">Intune Virtual Network</span>, <span class="Emphasis">Intune Neo4j</span>). It indicates the CI’s class/type.

- <span class="Strong" style="font-weight: bold;">Created On</span> – Timestamp when the CI was first created in the CMDB from discovery.

- <span class="Strong" style="font-weight: bold;">Last Modified On</span> – The most recent time the CI’s data was updated (any field change or enrichment).

- <span class="Strong" style="font-weight: bold;">Last Seen On</span> – The last successful discovery of assets. If this gets old, the asset may be offline or unreachable.

  - The "Last Seen On" field is not available for assets imported from Cloud sources.

- <span class="Strong" style="font-weight: bold;">IP Address</span> – Last known IP detected for the CI (blank for types that don’t expose an IP).

- <span class="Strong" style="font-weight: bold;">Host Name</span> – Host/endpoint name captured by discovery (for cloud services, often the workload/task name).

- <span class="Strong" style="font-weight: bold;">Authorization status</span> – A badge indicating <span class="Strong">Authorized</span> or <span class="Strong">This CI is unauthorized</span>:

  - <span class="Emphasis" style="font-weight: bold;">Authorized</span> = the CI is recognized/approved to exist in the CMDB; surfaced from Authorize tab of Import from Intune import records view.

    - An authorized asset always has an associated CMDB ID, such as "<span class="Strong">Update to /<CMDB Asset ID/></span>".

  - <span class="Emphasis" style="font-weight: bold;">Unauthorized</span> = the CI is new or not yet approved; typically surfaced from the <span class="Emphasis">Unauthorized</span> tab in import views.

- <span class="Strong" style="font-weight: bold;">Tags</span> – If any  
  Tags are Intune metadata key-value pairs attached to resources (like EC2 instances, S3 buckets, etc.).

# <span class="Strong">Top-right actions</span>

- <span class="Strong" style="font-weight: bold;">Move to CMDB</span> – Promotes this discovered item into the CMDB.

- <span class="Strong" style="font-weight: bold;">Rescan Now</span> – Runs a targeted re-discovery for this single asset.

# <span class="Strong">Tabs (main panel)</span>

<span class="Strong">**Details**: </span>A complete inventory of the Configuration Item (CI) is captured, with the specific details shown depending on the blueprint associated with the asset.
