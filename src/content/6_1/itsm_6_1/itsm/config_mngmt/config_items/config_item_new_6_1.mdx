# New Configuration Item (CI)

Use this function to configure a new Configuration Item (CI) by entering values for the parameters present.

<div class="numlist1" madcap:conditions="Default.OldUI" madcap:autonum="1. ">

In the navigation pane, select **ITSM /> Configuration Management /> CMDB**. The Configuration Items window displays.

</div>

<div class="numlist1" madcap:conditions="Default.NewUI" madcap:autonum="1. ">

In the navigation pane, select **ITAM /> Configuration Management /> CMDB**. The Configuration Items window displays.

</div>

<div class="Ind50" madcap:conditions="Default.OldUI">

<span class="image placeholder" original-image-src="../../../Resources/Images/ITSM/config-items.PNG" original-image-title="" madcap:conditions="Default.OldUI"></span>

</div>

<div class="Ind50" madcap:conditions="Default.NewUI">

<span class="image placeholder" original-image-src="../../../Resources/Images/ITSM/config-items-new.PNG" original-image-title="" madcap:conditions="Default.NewUI"></span>

</div>

<div class="NumList2" madcap:autonum="2. ">

From the *Select Actions* drop-down list, choose **New**. The Add New CI dialog box displays.

</div>

<div class="Ind50">

<span class="image placeholder" original-image-src="../../../Resources/Images/ITSM/add-new-ci.PNG" original-image-title="" madcap:conditions="Default.OldUI"></span>

</div>

<div class="Ind50">

<span class="image placeholder" original-image-src="../../../Resources/Images/ITSM/add-new-ci-new.PNG" original-image-title="" madcap:conditions="Default.NewUI"></span>

</div>

<div class="NumList2" madcap:autonum="3. ">

From the *Blueprint* drop-down list, select the applicable type of configuration item. Based on the selection, the fields in the dialog box change.

</div>

<div class="NumList2" madcap:autonum="4. ">

Add the required (/*) information.

</div>

<div class="NumList2" madcap:autonum="5. ">

For *Allow Edit Without Change Request*:

</div>

<div class="Bullet2">

If this option *is* checked, the CI can be edited after it's created.

</div>

<div class="Bullet2">

If this option *is not* checked, and an edit is required, either of the following must occur:

</div>

<div class="emdash1" madcap:autonum="- ">

A change request must be created.

</div>

<div class="emdash1" madcap:autonum="- ">

The user must have a role with relevant access (as per SACM Role Access).

</div>

<td class="TableStyle-Note-Indent-BodyA-Column2-Body1"><p>If a user has the ability to edit the CI, this option can be enabled/disabled on-the-fly by doing the following:</p>
<div class="NumList1" madcap:autonum="1. ">
While viewing the Configuration Items window, select the applicable item.
</div>
<div class="NumList2" madcap:autonum="2. ">
In the resulting window, from the <em>Select Actions</em> drop-down list, do one of the following:
</div>
<div class="emdash1" madcap:autonum="- ">
To allow editing, select <strong>Enable Editing</strong>.<br />
<span class="image placeholder" data-original-image-src="../../../Resources/Images/Discovery/enable-editing.PNG" data-original-image-title=""></span><br />
&#10;</div>
<div class="emdash1" madcap:autonum="- ">
To disallow editing, select <strong>Disable Editing</strong>.<br />
<span class="image placeholder" data-original-image-src="../../../Resources/Images/Discovery/disable-editing-ci.PNG" data-original-image-title=""></span><br />
&#10;</div></td>
</tr>

</table>

<div class="NumList2" madcap:autonum="3. ">

When all selections/entries are made, click **Add**. A notification email is sent to the configured user to receive CI creation actions.

</div>
