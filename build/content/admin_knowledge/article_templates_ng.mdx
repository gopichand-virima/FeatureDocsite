# Article Templates

Use this function to create and manage templates for knowledge base articles.

Article Templates provide standardized formats for different types of knowledge articles, ensuring consistency and making it easier for authors to create high-quality content.

In the main window, click **Admin > Knowledge > Article Templates**. The Article Templates window displays.

## Template List

View all available templates:

**Template Columns**:

- **Template Name**: Descriptive name
- **Type**: How-To, Troubleshooting, FAQ, etc.
- **Usage Count**: Times used
- **Status**: Active, Draft
- **Last Modified**: Last update date
- **Created By**: Template author

## Create New Template

From the *Select Actions* drop-down list, choose **New Template**. The New Template window displays.

### Basic Information

**Template Details**:

- **Template Name**: How-To Guide Template
- **Description**: Standard format for step-by-step instructions
- **Article Type**: How-To Guide
- **Status**: ☑ Active

**Template Category**:

- Technical Documentation
- User Guides
- Troubleshooting
- FAQ
- Policy and Procedures

### Template Structure

Define the structure and sections:

**Section 1: Overview**:

- **Section Name**: Overview
- **Required**: ☑ Yes
- **Help Text**: "Provide a brief summary of what this guide covers"
- **Content Type**: Rich text
- **Max Length**: 500 characters

**Section 2: Prerequisites**:

- **Section Name**: Prerequisites
- **Required**: ☐ Optional
- **Help Text**: "List any requirements or prerequisites"
- **Content Type**: Bulleted list
- **Example Content**:
  ```
  - Access to admin panel
  - Basic understanding of the system
  - Required permissions configured
  ```

**Section 3: Step-by-Step Instructions**:

- **Section Name**: Instructions
- **Required**: ☑ Yes
- **Help Text**: "Provide detailed step-by-step instructions"
- **Content Type**: Rich text with numbered steps
- **Include Screenshots**: ☑ Recommended

**Section 4: Notes and Tips**:

- **Section Name**: Additional Notes
- **Required**: ☐ Optional
- **Help Text**: "Add helpful tips or common pitfalls"
- **Content Type**: Rich text

**Section 5: Related Articles**:

- **Section Name**: Related Articles
- **Required**: ☐ Optional
- **Content Type**: Article links
- **Auto-suggest**: ☑ Based on tags

### Content Fields

**Standard Fields** (included in all articles):

- **Title**: Required, max 200 characters
- **Summary**: Required, max 500 characters
- **Keywords/Tags**: Optional, for search
- **Category**: Required dropdown
- **Author**: Auto-populated
- **Last Reviewed**: Date field

**Custom Fields for This Template**:

**Difficulty Level**:

- Type: Dropdown
- Options: Beginner, Intermediate, Advanced
- Required: Yes

**Estimated Time**:

- Type: Number
- Unit: Minutes
- Required: No

**Platform/Module**:

- Type: Multi-select
- Options: ITSM, CMDB, Discovery, Asset Management
- Required: Yes

**Version Applicability**:

- Type: Multi-select
- Options: 5.13, 6.1, 6.1.1, NextGen
- Required: Yes

### Formatting Guidelines

**Style Guidelines**:

```markdown
# Main Title (H1)
Brief overview paragraph

## Prerequisites (H2)
- Prerequisite 1
- Prerequisite 2

## Step-by-Step Instructions (H2)

### Step 1: First Action (H3)
1. Open the application
2. Navigate to Settings
3. Click on Configuration

**Note:** Important information appears in bold.

*Tip:* Helpful suggestions appear in italics.

### Step 2: Second Action
Continue with numbered steps...

## Additional Notes (H2)
Any supplementary information

## Related Articles (H2)
- Link to related article 1
- Link to related article 2
```

**Formatting Tools**:

- Headings (H1-H6)
- Bold, italic, underline
- Bullet and numbered lists
- Code blocks
- Tables
- Images and screenshots
- Hyperlinks
- Callout boxes (Info, Warning, Tip)

### Validation Rules

**Required Elements**:

- Title must be present
- At least one main section required
- Minimum content length: 100 characters

**Quality Checks**:

- ☑ Check spelling
- ☑ Validate links
- ☑ Ensure images are accessible
- ☑ Verify formatting consistency

**Approval Requirements**:

- Review by subject matter expert
- Approval by knowledge manager
- Technical accuracy verification

Click **Save Template**.

## Pre-Built Templates

### How-To Guide Template

**Structure**:

```
Title: How to [Perform Task]

Overview
Brief description of what will be accomplished

Prerequisites
- Required access or permissions
- Prerequisite knowledge
- Required tools or resources

Step-by-Step Instructions
1. First step with details
2. Second step with details
3. Continue...

Screenshots
[Embedded images showing key steps]

Tips and Best Practices
- Helpful tip 1
- Helpful tip 2

Troubleshooting
Common issues and solutions

Related Articles
Links to related content
```

### Troubleshooting Template

**Structure**:

```
Title: Troubleshooting [Issue]

Problem Description
What users are experiencing

Symptoms
- Symptom 1
- Symptom 2

Possible Causes
- Cause 1
- Cause 2

Resolution Steps
1. Check [item]
2. Verify [setting]
3. Test [functionality]

Prevention
How to avoid this issue in the future

When to Escalate
Criteria for escalating to higher support

Related Issues
Links to similar problems
```

### FAQ Template

**Structure**:

```
Title: Frequently Asked Questions - [Topic]

Introduction
Brief overview of the topic

Questions and Answers

Q: First question?
A: Detailed answer with explanation

Q: Second question?
A: Detailed answer with explanation

Q: Third question?
A: Detailed answer with explanation

Additional Resources
Links to detailed guides

Contact Information
Where to get more help
```

### Policy and Procedure Template

**Structure**:

```
Title: [Policy/Procedure Name]

Purpose
Why this policy exists

Scope
Who this applies to

Policy Statement
The actual policy

Procedures
Step-by-step process to follow

Responsibilities
Who is responsible for what

Definitions
Key terms explained

Related Policies
Links to related policies

Revision History
Changes and updates
```

## Template Variables

Use variables for dynamic content:

**System Variables**:

- `{{current_user}}`: Current user's name
- `{{current_date}}`: Today's date
- `{{article_author}}`: Article author name
- `{{last_modified}}`: Last modification date
- `{{version_number}}`: Current version

**Custom Variables**:

- `{{product_name}}`: Virima
- `{{support_email}}`: support@company.com
- `{{support_phone}}`: 1-555-123-4567
- `{{company_name}}`: Your Company

**Usage Example**:

```
If you need assistance, contact {{support_email}} 
or call {{support_phone}}.

Last Updated: {{last_modified}} by {{article_author}}
```

## Clone Template

Create a copy of existing template:

Select a template.

From the *Select Actions* drop-down list, choose **Clone Template**.

Enter new template name.

Modify as needed.

Click **Save**.

Control who can use templates:

- Edit template: Knowledge managers only
- Delete template: Admins only
