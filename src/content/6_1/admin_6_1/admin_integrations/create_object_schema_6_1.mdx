# Create an Object Schema in Jira

You can create a new Object Schema directly from Virima:

- Go to the **Schema Wizard** in Jira Credentials.  

- Select **Create Object Schema**.  

- In the **Add New Object Schema** wizard, enter:
  - **Schema Name** (mandatory)
  - **Key** (mandatory, 2–10 alphanumeric characters only)
  - **Description**
- Select **Add**.  
  The schema is added to Jira and appears in the Schema Wizard list.

# Jira Credentials

Use this function to specify the credentials for the Jira domain.

In the main window, click **Admin > Integrations > Jira Credentials**.

Enter a *Username*, the applicable *Domain* and *API Key*.

Click **Next**.

Choose one or both:

*Asset Management* – Sync Virima metadata into Jira Assets (CMDB).

*Service Management* – Sync Jira Incidents and Changes into Virima.

Click **Next**.

**Asset management setup:**

Choose specific object schemas from the dropdown or select *All Object Schemas* to include all available schemas.

Select **Yes** to sync Virima Blueprints into Jira schemas. **No** to skip.

**Service management setup:**

Select *All Projects* or choose specific projects from the dropdown. Click **Next**.

Select *All incident type issues* or choose specific types.

Select *Incident type issues*, choose how to retrieve issues by clicking **Assets** or **Custom** (link Asset Name to Jira incidents or map properties), and click **Next**.!

Select *All change type issues* or choose specific types.

Select *Change types issues*, choose how to retrieve change issues by clicking **Assets** or **Custom** (link Asset Name to Jira changes or map properties), and click **Save**.!

A popup will confirm the process. You will receive an email notification once it finishes. Select **Yes** to confirm.
