# Knowledge Standards

Knowledge standards ensure consistency, quality, and usability across all knowledge articles.

## Writing Standards

### Style Guidelines
- Clear, concise language
- Active voice preferred
- Second person (you/your)
- Present tense
- Professional tone

### Grammar and Spelling
- Proper grammar
- Correct spelling
- Punctuation rules
- Consistent terminology

### Formatting Requirements
- Use headings (H2, H3)
- Numbered steps for procedures
- Bullet points for lists
- Bold for emphasis
- Code blocks for commands

## Content Standards

### Article Structure
1. Title (clear and descriptive)
2. Overview/Summary
3. Prerequisites
4. Main Content/Procedure
5. Expected Results
6. Troubleshooting
7. Related Articles

### Required Elements
- Descriptive title
- Short description
- Category assignment
- Keywords/tags
- Last reviewed date
- Article owner

### Quality Criteria
- Accurate information
- Tested procedures
- Complete instructions
- Clear screenshots
- Working links

## Technical Standards

### Screenshot Guidelines
- Relevant content only
- Highlight important areas
- Consistent size
- PNG format preferred
- Alt text for accessibility

### Code Examples
- Use code blocks
- Include comments
- Test before publishing
- Show expected output
- Provide context

### Links
- Use descriptive text
- Verify links work
- Update broken links
- Internal links preferred
- External links noted

## Review Standards

### Regular Reviews
- Quarterly for critical articles
- Annually for standard articles
- After system changes
- Based on usage
- Triggered by feedback

### Review Checklist
- [ ] Information still accurate
- [ ] Procedures still work
- [ ] Screenshots current
- [ ] Links functional
- [ ] Category appropriate

## Compliance

### Documentation
- Author attribution
- Version history
- Review dates
- Change log

### Accessibility
- Screen reader compatible
- Alt text on images
- Keyboard navigation
- Clear headings
