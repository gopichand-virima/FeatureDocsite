# Notification Queue

Use this function to monitor, manage, and troubleshoot the notification delivery queue.

The Notification Queue displays all pending, in-progress, and recently delivered notifications, helping identify delivery issues and manage notification flow.

In the main window, click **Admin > Notification Management > Notification Queue**. The Notification Queue window displays.

## Overview

The notification queue shows:

- Pending notifications waiting to be sent
- Notifications currently being delivered
- Recently sent notifications
- Failed notifications requiring retry
- Scheduled notifications (future delivery)
- Notification delivery status and history

## View Queue Items

The queue displays notification entries:

**Queue Columns**:

- **Queue ID**: Unique identifier
- **Status**: Pending, Sending, Sent, Failed, Scheduled
- **Notification Type**: Email, SMS, Push, etc.
- **Recipient**: User or email address
- **Subject**: Notification subject line
- **Priority**: High, Normal, Low
- **Created**: When queued
- **Scheduled For**: Delivery time
- **Attempts**: Delivery attempts made
- **Channel**: Delivery method

## Manage Failed Notifications

View and retry failed notifications:

**Failed Notification Reasons**:

- **Invalid Recipient**: Email address invalid
- **SMTP Error**: Mail server rejected
- **Timeout**: Delivery timeout exceeded
- **Rate Limit**: Too many notifications
- **Bounced**: Hard or soft bounce
- **Blocked**: Recipient blocked sender
- **API Error**: Third-party API failed
- **Configuration Error**: Channel not configured

### Retry Failed Notifications

Select failed notifications.

From the *Select Actions* drop-down list, choose **Retry Delivery**.

**Retry Options**:

- Retry now
- Schedule retry for later
- Change delivery channel
- Update recipient address

Click **Retry**.

### Bulk Retry

Retry multiple failed notifications:

Filter by Status = Failed.

Select multiple items (checkboxes).

Click **Retry Selected**.

Choose retry strategy and click **Confirm**.

## Cancel Pending Notifications

Cancel notifications before delivery:

Select pending or scheduled notifications.

From the *Select Actions* drop-down list, choose **Cancel**.

**Cancellation Reason**:

- Sent by mistake
- Duplicate notification
- Recipient no longer needs it
- Notification rule error
- Testing purposes

Click **Confirm Cancellation**.
