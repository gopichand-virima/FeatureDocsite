# Import Asset/CI Relations

After moving the CI into the CMDB, to import asset-to-CI relationships:

1. Select the file containing relationship data.

2. From the [Select Actions]{.Strong} menu, click [Import Asset/CI Relations]{.Strong}.

 ![Import Data Files 1 19](../../Resources/Images/Discovery/Import-Data-Files/Import Data Files_1_19.png)

3. The [Attach Asset/CI Relations Files]{.Strong} window appears, allowing you to upload and map your asset-to-CI relationship data files.

 ![Import Data Files 1 20](../../Resources/Images/Discovery/Import-Data-Files/Import Data Files_1_20.png)

4. In the **Attach Asset/CI Relations File** window:

 - Click [Browse]{.Strong} to select a supported file format (.xls, .xlsx, or .csv).

 - (Optional) Download the sample file for correct formatting.

# [Relationship Import --- Columns & Rules]{.inline-comment-marker}

Creates links between two existing CIs in CMDB. Use it after the CIs are already in CMDB (or after you have moved imported CIs into CMDB).

![Import Data Files 1 21 559x249](../../Resources/Images/Discovery/Import-Data-Files/Import Data Files_1_21_559x249.png)

**Supported formats:** .xls, .xlsx, .csv

[Mandatory columns (must be present & filled):]{.Strong}

- [Source Asset Id]{.Strong}\
 The CMDB ID of the "from" CI (e.g., AST001450). Must be an existing CI ID.

- [Relationship]{.Strong}\
 The relationship type name (e.g., Allocated To, Connected To, Feeds).\
 ➤ Must exactly match a configured relationship name in **Admin → SACM→ Relationship Types** (spelling/case must match; no free-text).

- [Target Asset ID]{.Strong}\
 The CMDB ID of the "to" CI (e.g., AST001565). Must be an existing CI ID.

[Optional columns (populate when relevant):]{.Strong}

- [Source Port]{.Strong}\
 Port (or interface) on the source CI (e.g., Ethernet1/1, 8080). Free text.

- [Target Port]{.Strong}\
 [Port (or interface) on the target CI (e.g., ]{.inline-comment-marker}[Gi0/2]{.inline-comment-marker_1}[, ]{.inline-comment-marker}[443]{.inline-comment-marker_1}[). Free text.]{.inline-comment-marker}

5. [Click ]{.inline-comment-marker}[Upload]{.Strong}[.]{.inline-comment-marker}

- If a user attempts to upload a file in an unsupported format such as .txt on the Import Data Files page, the system immediately rejects the file and displays an error message.

 ![Import Data Files 1 11](../../Resources/Images/Discovery/Import-Data-Files/Import Data Files_1_11.png)

6. [The system processes the file and displays the imported relationship mappings for review.]{.inline-comment-marker}

 ![Import Data Files 1 22](../../Resources/Images/Discovery/Import-Data-Files/Import Data Files_1_22.png)

7. Click ****Submit**.**

8. Confirmation message pop-up will be displayed \"Success Message - Import added successfully\"

 ![Import Data Files 1 23](../../Resources/Images/Discovery/Import-Data-Files/Import Data Files_1_23.png)

9. [The user can view the relationship under CMDB CI details page.]{.inline-comment-marker}
