# Configuration Items (CI)

Configuration Items (CI) are components or other service assets must be managed to deliver an IT service.

In Virima:

<div class="Bullet1">

Each CI is derived from a Blueprint, with components that can be Network Adapters, Storage Disks, Software Instances, and so forth.

</div>

<div class="Bullet1">

Each CI component is associated with the parent CI using Relationships--for example, Installed On, Component Of, Hosted On, Virtualized On, and so forth.

</div>

<div class="Bullet1">

Each CI/CI Component in Ecosystem Manager has a unique Asset ID.

</div>

CI's are added when they are:

<div class="Bullet1">

Moved manually into CMDB.

</div>

<div class="Bullet1">

Discovered by the Discovery application as they are moved into CMDB.

</div>

<div class="Bullet1">

Imported from a file (.xls, .xlsx, or .csv).

</div>

To configure items, do the following:

<div class="NumList1" madcap:conditions="Default.OldUI" madcap:autonum="1. ">

In the navigation pane, select **ITSM /> Configuration Management /> Configuration Items**. The Configuration Items window displays.

</div>

<div class="NumList2" madcap:conditions="Default.NewUI" madcap:autonum="2. ">

In the navigation pane, select **ITAM /> Configuration Management /> CMDB**. The Configuration Items window displays.

</div>
