# Probe Workflow

Use this function to define a workflow for a probe. A Probe Workflow is a sequence of steps executed while discovering a host by entering appropriate scripts.

In the main window, click **Admin > Probe Workflow**. The Probe Workflow window displays.

!

!


### New GroupProbe

- From the *Select Actions* drop-down list, choose **New GroupProbe**.The New GroupProbe window displays.
- Enter the *Name* and *Description*.
- Enter a *Script* program that invokes the probe. A partial sample is shown below.

!

- When all entries are made, click **Add**.


### Edit Probe Workflow

Not all functions include an option to edit record line items shown in the main window.


- Navigate to the window containing the record to edit.
- Click the line containing the record. The applicable window or dialog box displays.
- Make the necessary modifications.
- Click **Save** or **Add**, as applicable.

When editing an existing item, other options may be available, such as adding [Tasks](/itsm/release_mngmt/tasks_6_1.mdx), [Comments](/discovery/comments_6_1.mdx), and [Attachments](/discovery/attachments_6_1.mdx) and viewing [History](/common_topics/history_6_1.mdx).


The options change based upon the function; therefore, not every option may be shown.


### Delete Probe Workflow

Deleting is a permanent action and cannot be undone. Deleting may affect other functionality and information in the application such as data in configured reports, fields in windows, selectable options, etc. Therefore, be sure to understand the potential effects before making a deletion.


- Click the line that contains the item to delete.
- From the *Select Actions* drop-down list, choose **Delete**. If a confirmation message is displayed, type **Delete** and click **Delete**to confirm.
