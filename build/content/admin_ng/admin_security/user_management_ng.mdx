# User Management

Use this function to create, manage, and configure user accounts within Virima.

User management provides centralized control over user access, profiles, and account settings.

In the main window, click **Admin > Security > User Management**. The User Management window displays.

## User List Overview

View all users in the system:

**User Columns**:

- **User ID**: Unique identifier
- **Username**: Login username
- **Full Name**: User's full name
- **Email**: Email address
- **Role**: Primary role
- **Department**: Department assignment
- **Status**: Active, Inactive, Locked
- **Last Login**: Last login timestamp
- **MFA Enabled**: Multi-factor status

## Create New User

From the *Select Actions* drop-down list, choose **New User**. The New User window displays.

### Basic Information

**User Details**:

- **Username**: Unique login name (required)
- **First Name**: User's first name (required)
- **Last Name**: User's last name (required)
- **Email**: Email address (required, unique)
- **Phone**: Phone number
- **Mobile**: Mobile number

**Account Settings**:

- **User ID**: Auto-generated or custom
- **Employee ID**: HR system employee ID
- **Cost Center**: Accounting cost center
- **Manager**: Select reporting manager

### Authentication

**Password Setup**:

- **Set Password**: Enter initial password
- **Require Password Change**: ☑ On first login
- **Password Never Expires**: ☐ (not recommended)

**Authentication Method**:

- Local authentication (Virima database)
- LDAP/Active Directory
- SAML 2.0 SSO
- OAuth 2.0
- Multi-factor authentication required

**Multi-Factor Authentication**:

- **Require MFA**: ☑ Enforce MFA for this user
- **MFA Methods**: SMS, Authenticator app, Email
- **Backup Codes**: Generate recovery codes

### Role and Permissions

**Primary Role**:

Select from available roles:

- Administrator
- Service Desk Agent
- Service Desk Manager
- Technician
- Asset Manager
- Change Manager
- Problem Manager
- End User
- Custom roles

**Additional Roles**:

Add multiple roles if needed.

**Assignment Groups**:

- Select default assignment group
- Add to multiple groups

**Direct Permissions**:

Override or add specific permissions beyond role.

### Contact Information

**Work Information**:

- **Department**: Select department
- **Location**: Select location/office
- **Building**: Building assignment
- **Floor**: Floor number
- **Room**: Room number
- **Desk**: Desk identifier

**Contact Details**:

- **Office Phone**: Extension number
- **Mobile Phone**: Mobile number
- **Home Phone**: Home contact (optional)
- **Alternate Email**: Secondary email

**Emergency Contact**:

- **Contact Name**: Emergency contact person
- **Relationship**: Relationship to user
- **Phone**: Emergency contact number

### User Preferences

**Regional Settings**:

- **Time Zone**: User's time zone
- **Language**: Preferred language
- **Date Format**: MM/DD/YYYY, DD/MM/YYYY, YYYY-MM-DD
- **Time Format**: 12-hour or 24-hour

**Notification Preferences**:

- **Email Notifications**: Enabled
- **SMS Notifications**: Disabled
- **Push Notifications**: Enabled
- **Digest Frequency**: Daily

### Access Control

**IP Restrictions**:

- Allow from any IP
- Restrict to specific IP ranges
- Require VPN connection

**Session Settings**:

- **Session Timeout**: 30 minutes
- **Concurrent Sessions**: Allow 2 sessions
- **Remember Device**: 30 days

**Access Hours**:

- Allow 24/7
- Restrict to business hours: Monday-Friday, 8 AM - 6 PM
- Custom schedule

### Additional Settings

**User Status**:

- **Active**: User can log in
- **Inactive**: User cannot log in
- **Locked**: Account locked due to failed login attempts

**VIP Status**: ☐ Mark as VIP user

**Service Account**: ☐ Mark as service/system account

**Notes**: Add administrative notes about user

Click **Save** to create the user.

## Edit User

Navigate to the window containing the user.

Click the line containing the user.

Make modifications to any section:

- Basic information
- Contact details
- Role assignments
- Preferences
- Security settings

Click **Save**.

## Reset Password

To reset a user's password:

Select a user.

From the *Select Actions* drop-down list, choose **Reset Password**.

**Reset Options**:

**Set New Password**:

- Enter new password manually
- Generate random password

**Send Reset Link**:

- Email password reset link to user
- Link expires in 24 hours

**Require Change on Next Login**: ☑ Check to force change

Click **Reset Password**.

## Unlock User Account

If account is locked due to failed login attempts:

Select locked user.

From the *Select Actions* drop-down list, choose **Unlock Account**.

Optionally reset password.

Click **Unlock**.

## Activate/Deactivate User

**Deactivate User**:

Select an active user.

From the *Select Actions* drop-down list, choose **Deactivate**.

**Deactivation Options**:

- **Reason**: Terminated, On leave, Transferred, Other
- **Effective Date**: Immediate or future date
- **Reassign Tickets**: Reassign user's tickets to another user
- **Retain Data**: Keep user data for records

Click **Deactivate**.

**Reactivate User**:

Select an inactive user.

From the *Select Actions* drop-down list, choose **Activate**.

Review and update user information.

Click **Activate**.

## Clone User

Create a new user based on existing user:

Select a user to copy.

From the *Select Actions* drop-down list, choose **Clone User**.

Enter new username and email.

Adjust settings as needed.

Click **Create**.

## Bulk User Operations

Manage multiple users simultaneously:

### Bulk Import Users

From the *Select Actions* drop-down list, choose **Import Users**.

**Import Methods**:

**CSV Import**:

- Download CSV template
- Fill in user data
- Upload completed CSV file

**LDAP Sync**:

- Connect to LDAP/AD
- Select users to import
- Map LDAP attributes to Virima fields

**API Import**:

- Import from HR system
- Use REST API
- Scheduled synchronization

Map fields and click **Import**.

### Bulk Update

Select multiple users (checkboxes).

From the *Select Actions* drop-down list, choose **Bulk Update**.

**Update Options**:

- Change role
- Update department
- Change location
- Set MFA requirement
- Update session timeout
- Modify access hours

Apply changes to all selected users.

### Bulk Deactivate

Select multiple users.

From the *Select Actions* drop-down list, choose **Bulk Deactivate**.

Choose deactivation date and ticket reassignment.

Click **Deactivate Selected**.

## User Groups

Organize users into groups:

Click **Manage Groups**.

### Create User Group

Click **New Group**.

**Group Details**:

- **Group Name**: Support Team
- **Description**: Level 1 support technicians
- **Type**: Static or Dynamic

**Add Members**:

**Static Group**:

- Manually select users
- Add/remove members manually

**Dynamic Group**:

- Define membership criteria
- Auto-update based on attributes
- Example: Department = "IT Support" AND Role = "Technician"

Click **Save**.

## User Provisioning

Automate user creation:

Click **Provisioning Settings**.

**Auto-Provisioning**:

- **From LDAP/AD**: Auto-create on first login
- **From HR System**: Sync with HR database
- **From SSO**: Create from SAML/OAuth attributes
- **Just-in-Time Provisioning**: Create on SSO login

**Provisioning Rules**:

- Default role assignment
- Department mapping
- Location assignment
- Group membership

**Deprovisioning**:

- Auto-deactivate when removed from directory
- Grace period before deactivation
- Ticket reassignment rules

## User Licenses

Manage user licenses:

Click **License Management**.

**License Overview**:

- Total licenses: 500
- Used licenses: 387
- Available licenses: 113

**License Types**:

- Full user: $50/month
- Agent: $30/month
- End user: $10/month
- Read-only: Free

**Assign License**:

Select user and assign license type.

**License Optimization**:

- Identify inactive users with licenses
- Reclaim unused licenses
- Auto-downgrade based on usage

## Delete User

⚠️ **Caution**: Deleting a user is permanent and removes all associated data.

**Recommended**: Deactivate instead of delete to preserve records.

If deletion is required:

Select user to delete.

From the *Select Actions* drop-down list, choose **Delete User**.

**Deletion Options**:

- **Reassign Tickets**: Reassign to another user
- **Anonymize Data**: Replace name with "Deleted User"
- **Archive Data**: Export user data before deletion

Confirm deletion.
