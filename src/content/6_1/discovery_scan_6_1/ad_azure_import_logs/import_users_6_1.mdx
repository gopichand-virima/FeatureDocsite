# Import Users

While importing users into Virima, the system checks if the user already exists. If not, Virima creates a new user.

For consecutive AD imports, the value of a user gets updated if the value has been modified in the AD.

After configuring AD, follow these steps:

1. Click the Import icon located to the right of the selected entry.

 ![AD User Import Logs 1 6](../../Resources/Images/Discovery/AD-User-Import-Logs/AD User Import Logs_1_6.png)

2. The [AD Import]{.Strong} dialog box displays.

3. From the [Type of Import]{.Strong} drop-down list, select an option.

 ![AD User Import Logs 1 7](../../Resources/Images/Discovery/AD-User-Import-Logs/AD User Import Logs_1_7.png)

4. Select the other options, as necessary.

 1. [Reflect AD Account Status]{.Strong} -- Enable to match user login status with Azure AD.

 2. [Activate User Account]{.Strong} -- Enable to make imported users active by default.

 3. [Send Activation Notification]{.Strong} -- Enable to email users upon account activation.

5. When all selections/entries are made, click [Import]{.Strong}.

6. A confirmation message displays. Click [OK]{.Strong}.
