# Azure Import

# Adding a schedule

1.  At the top of <span class="Strong">Scheduled Scans and Imports</span>, choose <span class="Strong">AZURE</span> → <span class="Strong">Select Actions </span>→<span class="Strong"> Add</span>.

<span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Scheduled-Scans-and-Imports/Scheduled%20Import%20setup%20AWS_6.png" original-image-title=""></span>

2.  Fill the header:

    - <span class="Strong">Name</span>/*: e.g., Azure import – nightly.

    - <span class="Strong">Credential</span>/*: pick your <span class="Strong">azure_account</span>.

    - <span class="Strong">Timezone</span>: select the correct region.

    - <span class="Strong">Import Frequency</span>: keep <span class="Strong">Current Time Settings</span> or adjust with the wheels below.

    - Toggle <span class="Strong">Active</span> (must be ON to scheduled import) and optionally <span class="Strong">Recurring</span>.

3.  Under <span class="Strong">Select what has to be imported from AZURE</span>/*, tick at least one service (only choose what you need):

    - Virtual Machine, <span class="Strong">Virtual Machine ScaleSets</span>, <span class="Strong">Network Security Group</span>, <span class="Strong">Application Gateway</span>,

    - <span class="Strong">App Services</span>, <span class="Strong">SQL Servers</span>, <span class="Strong">Virtual Network</span>, <span class="Strong">SQL Managed Instance</span>,

    - <span class="Strong">Azure Kubernetes Service (AKS)</span>, <span class="Strong">Azure Functions</span>, <span class="Strong">Azure Cosmos DB</span>, <span class="Strong">Azure Storage Accounts</span>,

    - <span class="Strong">Azure Key Vault</span>, <span class="Strong">Azure PostgreSQL</span>, <span class="Strong">Azure Oracle Database</span>, <span class="Strong">Azure Neo4J</span>.

4.  Set the schedule with the time wheels (<span class="Strong">Second / Minute / Hour / Day / Month / Weekday</span>).

<span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Scheduled-Scans-and-Imports/Scheduled%20Import%20setup%20AWS_7.png" original-image-title=""></span>

5.  Click save

6.  Close the dialog. The scheduled import shall appear in the list.

    - if <span class="Strong">IsActive</span>, is true, then <span class="Strong">NextTriggerTime is calculated and displayed</span>

    - All other columns including credential shall be populated

# Importing a Azure Schedule

- Open <span class="Strong">Select Actions</span>, and click <span class="Strong">Import</span>.

<span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Scheduled-Scans-and-Imports/Scheduled%20Import%20setup%20AWS_8.png" original-image-title=""></span>

- You’ll see the <span class="Strong">Import Azure Schedule</span> popup with a <span class="Strong">Browse</span> button and a link to <span class="Emphasis">Click here to download the sample file</span>.

- <span class="Strong">Download template:</span> In the <span class="Strong">Import Azure Schedule</span> popup, click the link to <span class="Emphasis">download the sample file</span>.

# <span class="Strong">Prepare Your File</span>

- <span class="Strong">Headers:</span> Keep the headers exactly as shown in the sample file.

- <span class="Strong">Fill in mandatory fields:</span>

  - <span class="Strong">Name</span> Must contain a string.

  - <span class="Strong">Credentials Enter Credential Name or Credential ID of Azure Account</span>

  - <span class="Strong">Imported</span> Items to be imported, must be separated by commas

  - <span class="Strong">Timezone</span>

  - <span class="Strong">Schedule fields:</span> second, minute, hour, day, month, weekday

- <span class="Strong">Fill in optional/additional fields as required:</span>

  - <span class="Strong">Active:</span> true/false

  - <span class="Strong">Recurring:</span> true/false

# <span class="Strong">Import the File</span>

1.  Click <span class="Strong">Select Actions → Import</span>.

2.  In the popup:

    - Click <span class="Strong">Browse</span>.

    - Select your prepared file.

    - Click <span class="Strong">Upload</span>.

3.  Close the dialog.

<span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Scheduled-Scans-and-Imports/Scheduled%20Import%20setup%20AWS_9.png" original-image-title=""></span>

# <span class="Strong">Verify Imported Schedule</span>

- The scheduled jobs will appear in the list/grid.

- If <span class="Strong">IsActive = true</span>, the system will automatically calculate and display <span class="Strong">NextTriggerTime</span>.

- All other relevant columns (including credentials) will be populated.

# Deleting a Azure Schedule

- Select the checkbox in the leftmost column for each schedule you want to remove.

- From the <span class="Strong">Select Actions </span>dropdown, click <span class="Strong">Delete</span>.

<span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Scheduled-Scans-and-Imports/Scheduled%20Import%20setup%20AWS_10.png" original-image-title=""></span>

- A <span class="Strong">Delete Confirmation</span> modal appears.

- In the confirmation box, type <span class="Strong">Delete</span> (exactly as shown), then click <span class="Strong">Delete</span>.  
  The selected schedule(s) are removed and disappear from the grid.

- No imports will be executed from the point of delete

- If an active import is currently running, The import will be completed
