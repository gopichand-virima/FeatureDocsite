# Request Types

Use this function to configure different types of service requests with customized color codes for visual identification.

Request types help categorize requests based on their nature, service offering, or handling requirements. Each type can be assigned a unique color code for easy visual recognition in lists and dashboards.

In the main window, click **Admin > Request Management > Request Types**. The Request Type window displays.

## New Request Type

From the *Select Actions* drop-down list, choose **New Request Type**. The New Request Type window displays.

Enter a *Name* and *Description* for this type.

In the *Color* field, click on the color and choose a color from the picker OR type a 6-digit hex code.

Configure additional properties:

**Default Priority**: Set a default priority for this request type (optional).

**Default Category**: Set a default category for this request type (optional).

**Requires Approval**: Check if requests of this type require approval.

**Auto-Assign to Group**: Select a default assignment group (optional).

**Auto-Fulfill**: Check if this request type can be automatically fulfilled.

**Service Catalog Item**: Link to a service catalog offering (optional).

Click **Add**.

## Common Request Types

Examples of request types you might configure:

- **Standard Service**: Pre-approved standard services
- **Access Request**: User access and permissions
- **Provisioning**: Equipment or software provisioning
- **Information Request**: Documentation or information
- **Change in Service**: Modifications to existing services
- **Account Management**: User account operations
- **Training Request**: Training and education requests
- **Consultation**: Expert consultation services
- **Temporary Access**: Time-limited access requests
- **Decommission**: Service or asset decommissioning

## Edit Request Type

Navigate to the window containing the record to edit.

Click the line containing the record. The applicable window displays.

Make the necessary modifications.

Click **Save**.

## Delete Request Type

Deleting is a permanent action and cannot be undone. Ensure no requests are currently using this type before deletion.

Click the line that contains the item to delete.

From the *Select Actions* drop-down list, choose **Delete**.

## Type-Based Routing

To configure automatic routing based on request type:

Select a request type.

Click **Configure Routing**.

Define routing rules:

- **Assignment Group**: Auto-assign to specific group
- **Assignee**: Auto-assign to specific user
- **Workflow**: Attach specific workflow
- **Approval Chain**: Define approval requirements
- **Notification Template**: Use custom notification

Click **Save**.
