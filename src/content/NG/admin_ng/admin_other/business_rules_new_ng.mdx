# New Business Rule

Use this function to accomplish tasks like automatically modifying the ITSM records or creating a new record based on action details when the trigger conditions are met. This reduces the manual effort of creating and updating the records.

Business rules can be set to run when the following actions occur: Add, Edit, and Add and Edit.

In the main window, select **Admin > Others > Business Rules**. The Business Rules window displays.

From the *Select Actions* drop-down list, choose **New Business Rule**. The New Business Rule dialog box displays.

In the *Rule Name* field, type a rule name to be shown in the Rule name box.

In the *Description* field, type details for this rule.

In the *Order/Priority* field, type the order in which this rule should be executed when there are multiple rules set for the selected module.

To execute the next enabled business rule based on the Order/Priority for the same module, check **Turn on cascade execution**.

To enabled (or disable) the rule, check (or uncheck) the **Enable Rule** option.

In the *Module* field, click the drop-down list, select the name of this module for which the rule applies.

To specify the type of action performed, in the *Execute On* field, click the drop-down list and select the applicable action.

To associate a template, click **Add** for the *Associate Templates* field option. The Associate Templates or Blueprints dialog box displays.

Search for and select the applicable template or blueprint, then click **Save**.

When returning to the New Business Rules window, click **Save**.

In the main window, click the **gear** icon and select **Others > Business Rules**.

The Business Rules window displays.

Click **New Business Rule**. The new Business Rule dialog box displays.

In the *Rule Name* field, type an identifying name to be shown in the Rule name box.

In the *Description* field, type information about this rule.

In the *Order/Priority* field, type the order in which this rule should be executed when there are multiple rules set for the selected module.

In the *Module* field, click the drop-down list, select the name of this module for which the rule applies.

To execute the next enabled business rule based on the Order/Priority for the same module, check **Turn on cascade execution**.

To enabled (disable) the rule, check (uncheck) the **Enable Rule** option.

To specify the type of action performed, in the *Execute On* field, click the drop-down list and select the applicable action.

To associate a blueprint, click the **plus + sign** next to the *Associated Blueprints* field. The CMDB dialog box displays.

Search for and select the applicable template or blueprint, then click **Save**.

When returning to the New Business Rules window, click **Save**.

Add Conditions

Use this function to apply a set of conditions to the business rule.

When adding a condition, the following applies:

- Two business rules cannot have the same execution order.
- All the mandatory fields of an ITSM record should be defined under *Action Details*.
- Two or more conditions can be combined using an “AND” or “OR” logical condition(s).


The Rule Name field must have an entry to enable the **Add Condition** button.


In the New Business Rule dialog box, click **Add Condition**.

Configure the **Trigger Conditions**, **Action Details** and **Notification** settings (as described below).

When all selections/entries are made, click **Save**.

Trigger Conditions

Under *Trigger Condition Details*, do the following:

! !

Select **Ignore Conditions** to disregard all the conditions configured.

In the *Property* field, click the drop-down list and select the property for which you want to apply to the property.

In the *Operator* field, click the drop-down list and select the type of action to perform on the triggered condition.

Enter the applicable value for the condition.

To add multiple conditions, click the **plus + icon**.

To add multiple conditions, click “and” or "or” button.

When all selections/entries are made, click **Add Condition**.

To remove a condition, either click **Delete Condition** or the **trashcan icon** at the end of the row for a condition.

Action Details


The *Action Type* determines what additional fields display.


In the *Action Type* field, click the drop-down list and select the applicable type. Based on this selection, the remain fields change. *Refer to the table below for more details on the Action Type and its corresponding fields.*

! !

To add multiple actions, click the **plus + icon**.

To remove an action, click the **minus - icon**.

When all selections/entries are made, click **Add Action**.

To remove the Action Details block from the window, click **Delete Action**.

### Action Type Fields

| Action Type | Fields |

| Update the triggered record | Property, Operator and Value |

| Create new record | Target Module, Create from Model, Property, Operator and Value |

| Move Record | When the module is Discovery, moves the records from Discovery to CMDB. |

| Delete the triggered record | N/A |

| Create task in triggered record | Property, Operator and Value |

| Survey notification | When the module is Incident, Change, Release/Request/Problem in Business rule, a survey is sent to the Assignee, Reporter, and Stake Holders of the record. |

| No action | N/A |

| "Process" Action Types | N/A |

| "Copy To" Action Types

(ServiceNow, Cherwell, Ivanti) | N/A |


Notifications

Click the **plus + sign** next to the *Update* text.

Click the type of list to search (such as USERS, ROLES, and so forth).


Selections can be made of each type to designate all who need to receive a notifcation when an action takes place.


Enter the search criteria and press **Enter**. *See also* [Advanced Search](/common_topics/advanced_search_ng.mdx).

Select choices from the results list, moving to the selected list.

When all selections/entries are made, click **Add**. The window is updated and displays all the selected notifications.
