# Jira Asset Mappings

Use this function to view and add Jira mapping information.


There must be a [Jira Credential](/admin_integrations/jira_credentials_6_1.mdx) configured before a mapping can be done.


In the main window, click **Admin > Integrations > Jira Asset Mappings**. The Jira Asset Mapping window displays.

New Jira Mapping

From the *Select Actions* drop-down list, choose **New Jira Mapping**.

By default, the *Property* selected is Blueprint. Select the *Operator* and the applicable Virima Blueprint.

Select the Jira *Object Schema*, *Object Type*, and *Child Type*.

*Is Enabled* indicates the status of the mapping (true/false). A blueprint copies from Virima to Jira only if the mapping has the "Is Enabled" status set to True.

To add a Jira Property Mapping, click **Add**.

Select the *Virima Property*, *Jira Property*, *Value* and if a *Correlator* applies.

Click **Add**.

Clone Mappings

From the *Select Actions* drop-down list, choose **Clone Mapping**.

Use the checkboxes to select the records you want to clone.

The selected mappings will be duplicated and added to the list.

Re-Sync Metadata

From the *Select Actions* drop-down list, choose **Re-Sync Metadata**.

If the re-sync is successful, a confirmation message displays. Click **OK**.

Reset Mappings

From the *Select Actions* drop-down list, choose **Reset Mappings**.

If the reset is successful, a confirmation message displays.
