# Managing Imported Assets

# Delete Imported Assets

This action permanently removes the selected asset record(s) from the **Imported Assets** list, ensuring that outdated or incorrect entries do not clutter the system.

1. Select the checkbox next to one or more asset records.

2. From the [Select Actions]{.Strong} dropdown, choose [Delete]{.Strong}.

![Imported Assets 2](../../Resources/Images/Discovery/Imported-Assets/Imported Assets_2.png)

3. If no record is selected, an alert message appears: [\"Please Select atleast one Record.\"]{.Emphasis}

![Imported Assets 3](../../Resources/Images/Discovery/Imported-Assets/Imported Assets_3.png)

4. Type ****Delete**, **and then click on [Delete ]{.Strong}to confirm.

5. Click [Cancel ]{.Strong}to cancel the deletion.

![Imported Assets 4](../../Resources/Images/Discovery/Imported-Assets/Imported Assets_4.png)

5. The selected records are removed from the **Imported Assets** list.

# Move Assets to CMDB

The \"[Move to CMDB]{.Strong}\" option in Virima allows users to move selected asset records from the Imported Assets list into the CMDB. This process ensures that only validated and relevant assets are formally tracked and managed as configuration items (CIs).

1. Select the checkbox next to one or more asset records in the Imported Assets list.

2. From the [Select Actions]{.Strong} dropdown, choose [Move to CMDB]{.Strong}.

![Imported Assets 5](../../Resources/Images/Discovery/Imported-Assets/Imported Assets_5.png)

3. A confirmation dialog appears: ["Are you sure you want to move the Asset?"]{.Emphasis}

![Imported Assets 6](../../Resources/Images/Discovery/Imported-Assets/Imported Assets_6.png)

4. Click [Yes]{.Strong} to proceed.

 - The system begins processing and moves the selected records into the CMDB.

5. Click [No]{.Strong} to cancel the operation.

- Wait until the record is moved; the selected record will appear grayed out during processing.

- After moving, records cannot be retrieved from the Imported Assets list.

- You can also move assets to the CMDB from the Details window.

# Re-scan Assets

The [Re-Scan]{.Strong} feature enables users to restart the scanning process for an existing asset record. This is helpful when an asset\'s information has changed or was incomplete during the initial scan.

1. Select the checkbox next to the asset record you want to revalidate in the Imported Assets list.

2. From the [Select Actions]{.Strong} dropdown, choose [Re-scan]{.Strong}.

![Imported Assets 7](../../Resources/Images/Discovery/Imported-Assets/Imported Assets_7.png)

3. The [Re-scan]{.Strong} window opens.

![Imported Assets 8](../../Resources/Images/Discovery/Imported-Assets/Imported Assets_8.png)

4. Enter or select the following details:

 - [IP Address]{.Strong}: The asset's IP address (auto-filled but editable).

 - [Client]{.Strong}: Select the client from the dropdown.

 - [Type of Scan]{.Strong}: Select the scan type probe from the dropdown (e.g., Deep Host Scan, vCenter Scan).

- If the user does not select a client and attempts to perform the scan operation, the system displays the error message: "Please select the client."

 ![Imported Assets 9](../../Resources/Images/Discovery/Imported-Assets/Imported Assets_9.png)

- If the user selects a client but does not select a scan type and attempts to perform the scan operation, the system displays the error message: "**Please select a scan type**."

 ![Imported Assets 10](../../Resources/Images/Discovery/Imported-Assets/Imported Assets_10.png)

5. Click [Scan]{.Strong} to initiate the re-scan.

 - The system refreshes the asset's data and updates the table accordingly.

 - A new scan run will appear on the Recent Scans page; refresh the page to view the latest status updates.

6. Click [Cancel]{.Strong} to cancel without making changes.

The asset must have a valid IP address to be re-scanned.
