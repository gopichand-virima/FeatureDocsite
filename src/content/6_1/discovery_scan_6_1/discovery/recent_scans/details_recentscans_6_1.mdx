# Details Panel (Main Scan Summary)

<span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Recent-Scans/Recent%20Scans_1_12.png" original-image-title=""></span>

<span class="Strong">General Information:</span>

- <span class="Strong">Name</span>: Scan name

- <span class="Strong">Client</span>: Discovery application used to execute the scan

- <span class="Strong">Probe</span>: Probe type used for the scan (Deep Host Scan (WMIC/|SSH/|SNMP))

- <span class="Strong">Activity</span>: Status update on completed actions (e.g., Discovery agent completed the scan)

|                                              |                  |
|----------------------------------------------|------------------|
| Activity                                     | Status           |
| Finding host types                           | Scan in progress |
| Running probe /<ProbeName/> on host /<Host/> | Scan in progress |
| Found /<N/> Hosts up and running             | Scan in progress |
| Discovery agent completed the scan           | Scan completed   |
| Scan aborted                                 | Scan aborted     |

<span class="Strong">IP Range</span>

- The IP address or range targeted by the scan (e.g., 172.16.5.187)

<span class="Strong">Host Details:</span>

- <span class="inline-comment-marker">Summary of discovered devices:</span>

  - <span class="Strong">Total Scan Hosts (255):</span> Number of targets included in this scan (from your IP range).

  - <span class="Strong">Total Discovered Hosts (255):</span> Total number discovered hosts for a given IP range.

  - <span class="Strong">Windows Hosts (17):</span> Successfully fingerprinted as Windows.

  - <span class="Strong">Unix/Linux Hosts (26):</span> Successfully fingerprinted as Unix/Linux.

  - <span class="Strong">Network Devices (4):</span> Routers/switches/firewalls, etc.

  - <span class="Strong">Unknown Hosts (18):</span> The targets are reachable, but none of the supplied credentials authenticated, so the OS couldn’t be identified.

  - <span class="Strong">Unknown Devices (190):</span> Devices discovered but not fully classified (often due to missing credentials/ports).

<span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Recent-Scans/Recent%20Scans_1_13.png" original-image-title=""></span>

# Execution Info

- <span class="Strong">Run by</span>: <span class="inline-comment-marker">Name of the user who initiated the scan (e.g., </span><span class="Emphasis">Admin Virima</span><span class="inline-comment-marker">). Logged in User</span>

- <span class="Strong">Status</span>: <span class="inline-comment-marker">Final scan result, such as "Scan completed" or "Scan in progress."</span>

  - Scan Completed: The master scan is marked <span class="Strong">Completed</span> only after every child scan finishes and no hosts or sub lists remain pending.

  - Scan in Progress: The master scan is marked <span class="Strong">In Progress</span>, if the child scan has not completed or has not been assigned a status (Success, Partial, or Failed).

- <span class="Strong">Triggered a</span><span class="Strong">t / Completed at:</span><span class="inline-comment-marker"> Timestamps indicating when the scan started and finished.</span>

- <span class="Strong">Scan duration</span>: How long the scan took (e.g., <span class="Emphasis">1 minute 20 seconds</span>).

# Recent scan “Result” and “Details” scenarios

|             |                                                        |
|-------------|--------------------------------------------------------|
| Result      | Details (examples shown in UI)                         |
| Success     | Scan completed                                         |
| Partial     | Couldn’t discover the complete details. Scan completed |
| Failed      | Invalid Username/Password;                             |
| Failed      | No working credential                                  |
| Failed      | Scan Has Timed out for /<host/>                        |
| Failed      | Error Executing probe on /<host/>                      |
| Failed      | Insufficient privileges for /<host/>                   |
| Pass / Fail | Pass/Fail with audit-specific message                  |

- <span class="Strong">Device type</span>: Type of device identified (e.g., <span class="Emphasis">Windows Host</span>).

- <span class="Strong">Device summary</span>: Key system information,

- e.g.:

  - Operating system

  - Domain/Workgroup

  - MAC address

- <span class="Strong">Rescan</span>: The circular icon on the<span class="inline-comment-marker"> right is to initiate a </span><span class="Strong">Rescan</span>. Option to change the Probe and Discovery application (Client) shall be provided.

  - In <span class="Strong">Recent Scans</span>, click the <span class="Strong">circular arrow</span> at the far right of the row to open <span class="Strong">Rescan</span>.

  - The dialog shows:

    - <span class="Strong">IP Address</span> – prefilled with the selected host.

    - <span class="Strong">Choose a client</span> – pick the <span class="Strong">Discovery application/tenant</span> to run against.

    - <span class="Strong">Type of scan</span> – choose the <span class="Strong">probe</span> to use (e.g., Deep Host Scan).

  - Adjust <span class="Strong">Client</span> and <span class="Strong">Type of scan (Probe)</span> as needed, then click <span class="Strong">Scan</span> to launch an immediate rescan.

  - A new run will appear on the <span class="Strong">Recent Scans</span> page; refresh to see status updates.

- Rescans appear as the top entry in the recent scans table. The scan name will be <span class="Strong">Rescan on /<IP-Address/></span>  
  <span class="Emphasis">Example:</span> Rescan on 192.168.1.188.

# Available Actions (from Select Actions dropdown)

Users can perform follow-up actions on selected devices using the <span class="Strong">Select Actions</span> dropdown:

<span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Recent-Scans/Recent%20Scans_1_17.png" original-image-title=""></span>

<span class="Strong">Move to CMDB</span>

- Move to CMDB is allowed when the result is <span class="Strong">Success, </span>the discovered item cannot move to CMDB otherwise.

- When the result is <span class="Strong">Failed </span>or <span class="Strong">Partial</span>, the user can select <span class="Strong">Move to CMDB</span>, but the record will not move. No error message displays.

<span class="Strong">Re-scan</span>

- Reinitiates scan on the selected record.

<span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Recent-Scans/Recent%20Scans_1_18.png" original-image-title=""></span>

- Select the scan probe and client to proceed, then click <span class="Strong">Scan </span>to rescan.

<span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Recent-Scans/Recent%20Scans_1_19.png" original-image-title=""></span>

- When a single host is selected and a rescan is performed, the rescan will appear as the top entry in the recent scans table. The scan name will be <span class="Strong">Rescan on /<IP Address/></span>.  
  <span class="Emphasis">Example:</span> <span class="Strong">Rescan on 192.168.1.188</span>

<span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Recent-Scans/Recent%20Scans_1_16.png" original-image-title=""></span>

- When <span class="Strong">Multi-host</span> is selected and a rescan is performed, the rescan will appear as the top entry in the recent scans table. The scan name will be <span class="Strong">Rescan on multiple hosts</span>.
