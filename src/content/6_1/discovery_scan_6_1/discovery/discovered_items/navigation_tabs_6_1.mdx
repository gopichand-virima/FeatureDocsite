# Components Tab

The <span class="Strong">Components</span> tab offers a detailed view of the individual elements that comprise a discovered asset. It breaks down the asset into its technical components, providing insight into the system’s hardware and software configuration. This detail is crucial for effective IT asset management, troubleshooting, and audit readiness.

# Sub-Tabs Under Components

This section features multiple <span class="Strong">sub-tabs</span>, each representing a specific type of system component. The components sub-tabs differ based on the “Related Possible Components” associated with the blueprint of the discovered items.

<span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Discovered-Items/Discovered%20Items_1_24.png" original-image-title=""></span> <span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Discovered-Items/Discovered%20Items_1_25.png" original-image-title=""></span>

- Installed Software

- Listening Processes

- Network Adapters

- Network Storage Disks

- Peripheral Devices

- Processes

- Services

- Software Instances

- Storage Disk

- Windows Updates

- MMC Certificates

<span class="Strong">Use this section to:</span>

- Audit physical/virtual components

- Identify mismatches, missing parts, or outdated hardware/software

# Log On Events Tab

The <span class="Strong">Log On Events</span> tab is part of the detailed view of a discovered asset, tracking <span class="Strong">user login activity</span> on the selected machine. This tab reveals who accessed the system, when, and by what method or protocol. It is useful for auditing user behavior and detecting unauthorized access attempts.

<span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Discovered-Items/Discovered%20Items_1_26.png" original-image-title=""></span>

- <span class="Strong">Client Dropdown:</span> Select type of client used for logon session.

- <span class="Strong">Duration Dropdown:</span> Filter by time range (e.g., Last 24 Hours, Last 7 Days, Last 30 Days).

# Tasks Tab

The <span class="Strong">Tasks</span> tab in the Discovered Item's detail view tracks and manages task assignments related to the specific asset. This feature enables IT teams to collaborate by assigning, documenting, and monitoring tasks linked to a particular configuration item (CI).

- Task list area: The central area displays <span class="Strong">existing tasks</span> related to the discovered item.

  <span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Discovered-Items/Discovered%20Items_1_29.png" original-image-title=""></span>

<!-- -->

- Select <span class="Strong">New Task</span> to create a task for the asset.

- In the task form, enter the required details:

  - <span class="Strong">Task title</span>

  - <span class="Strong">Assignee</span>

  - <span class="Strong">Status</span>

  - <span class="Strong">Due date</span>

- Click on <span class="Strong">Add </span>to add the task to the list.

  <span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Discovered-Items/Discovered%20Items_1_30.png" original-image-title=""></span>

<!-- -->

- Select the task you want to remove.

- Select <span class="Strong">Delete</span>.

- Confirm the deletion.

  - The task is permanently removed from the list.

# Comments Tab

The <span class="Strong">Comments</span> tab enables users to add and view notes or annotations for specific discovered items. It acts as a collaboration tool for teams managing IT assets, allowing them to document findings, track decisions, and leave messages for other administrators.

If no comments exist, the section displays <span class="Strong">No comments.</span>

After comments are added, they appear in this section in chronological order.

- Select <span class="Strong">New Comment</span> to open a text box.

  - Use this to log issues, remarks, or observations about the CI.

- In the <span class="Strong">Text Box</span>, type your comment.

- (Optional) Select <span class="Strong">Attach</span> to include a file, such as a screenshot, log, or document.

- Select <span class="Strong">Add</span> to save the comment.

  - The comment appears under the CI and is visible to all users with access to the record.

- To exit without saving, select <span class="Strong">Cancel</span>.

  <span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Discovered-Items/Discovered%20Items_1_32.png" original-image-title=""></span>

# Attachments Tab

The Attachments tab allows users to upload and manage files linked to a discovered configuration item (CI). It serves as a centralized location for storing supporting documentation, screenshots, reports, logs, and other relevant files.

<span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Discovered-Items/Discovered%20Items_1_33.png" original-image-title=""></span>

- If no files are attached, the tab displays <span class="Strong">“No Attachments.”</span>

- After a file is uploaded, it appears in the list with options to <span class="Strong">view</span>, <span class="Strong">download</span>, or <span class="Strong">delete</span>.

- Select <span class="Strong">New Attachment</span>.

  - A file upload popup opens.

  <span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Discovered-Items/Discovered%20Items_1_34.png" original-image-title=""></span>

<!-- -->

- In the popup window, do the following:

  - <span class="Strong">Upload Document</span> – Select <span class="Strong">Browse</span> to choose a file from your computer.

  - <span class="Strong">Description</span> – (Optional) Enter a note or context for the file.

- Click on <span class="Strong">Upload</span> to save the file and associate it with the discovered item.

  <span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Discovered-Items/Discovered%20Items_1_35.png" original-image-title=""></span>

# Updates Tab

The Updates tab offers a chronological log of changes to a discovered Configuration Item (CI), serving as a critical tool for tracking modifications, updates, and administrative actions over time.

<span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Discovered-Items/Discovered%20Items_1_36.png" original-image-title=""></span>

1.  <span class="Strong">Modified On</span>  
    Displays the date and time of each change.

2.  <span class="Strong">Modified By</span>  
    Shows the username or system account that made the change.

3.  <span class="Strong">Changes</span>  
    Lists the types of modifications made, including:

    - Configuration adjustments

    - Hardware or software updates

    - Scanning events

    - Component associations or dissociations.
