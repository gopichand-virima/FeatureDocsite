# New Business Rule

Use this function to accomplish tasks like automatically modifying the ITSM records or creating a new record based on action details when the trigger conditions are met. This reduces the manual effort of creating and updating the records.

Business rules can be set to run when the following actions occur: Add, Edit, and Add and Edit.

In the main window, select **Admin /> Others /> Business Rules**. The Business Rules window displays.

</div>

Add Conditions

Use this function to apply a set of conditions to the business rule.

When adding a condition, the following applies:

<div class="Bullet1" madcap:conditions="Default.NewUI">

Two business rules cannot have the same execution order.

</div>

<div class="Bullet1" madcap:conditions="Default.NewUI">

All the mandatory fields of an ITSM record should be defined under *Action Details*.

</div>

<div class="Bullet1" madcap:conditions="Default.NewUI">

Two or more conditions can be combined using an “AND” or “OR” logical condition(s).

</div>

|     |                                                                                |
|-----|--------------------------------------------------------------------------------|
|     | The Rule Name field must have an entry to enable the **Add Condition** button. |

<div class="NumList1" madcap:conditions="Default.NewUI" madcap:autonum="1. ">

In the New Business Rule dialog box, click **Add Condition**.

</div>

<div class="NumList2" madcap:conditions="Default.NewUI" madcap:autonum="2. ">

Configure the **Trigger Conditions**, **Action Details** and **Notification** settings (as described below).

</div>

<div class="NumList2" madcap:conditions="Default.NewUI" madcap:autonum="3. ">

When all selections/entries are made, click **Save**.

</div>
