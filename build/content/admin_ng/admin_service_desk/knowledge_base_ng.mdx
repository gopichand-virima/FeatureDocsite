# Knowledge Base Configuration

Use this function to configure knowledge base settings, article templates, and content management options for the service desk.

The Knowledge Base provides a repository of solutions, procedures, and information that helps both end users and support staff resolve issues quickly and consistently.

In the main window, click **Admin > Service Desk > Knowledge Base**. The Knowledge Base Configuration window displays.

## Knowledge Base Settings

Configure general knowledge base settings:

**Enable Knowledge Base**: Check to activate the knowledge base.

**Public Access**: Allow unauthenticated users to view public articles.

**Approval Required**: Require approval before articles are published.

**Version Control**: Enable article versioning and change tracking.

**Rating System**: Allow users to rate article helpfulness.

**Article Feedback**: Enable feedback and comments on articles.

Click **Save**.

## Article Templates

Create templates for consistent article formatting.

### New Article Template

From the *Select Actions* drop-down list, choose **New Template**.

Enter a *Template Name* and *Description*.

Configure template structure:

**Sections**: Define standard sections for this template type.

Common sections:

- **Symptom/Issue**: Description of the problem
- **Environment**: Applicable systems or configurations
- **Cause**: Root cause explanation
- **Resolution**: Step-by-step solution
- **Workaround**: Temporary workaround if available
- **Related Articles**: Links to related content
- **Attachments**: Supporting files or screenshots

**Required Fields**: Mark which sections are mandatory.

**Default Content**: Add placeholder text or instructions.

Click **Add**.

### Common Article Templates

**Solution Article**:
- Problem description
- Solution steps
- Screenshots
- Related information

**How-To Guide**:
- Objective
- Prerequisites
- Step-by-step instructions
- Expected results

**Troubleshooting Guide**:
- Symptom
- Possible causes
- Diagnostic steps
- Resolution options

**FAQ Article**:
- Question
- Detailed answer
- Additional information

## Categories and Tags

Organize knowledge content with categories and tags.

### Manage Categories

Click **Manage Categories**.

Create hierarchical categories:

- Hardware
  - Desktop
  - Laptop
  - Printer
- Software
  - Operating Systems
  - Applications
  - Email
- Network
  - Wi-Fi
  - VPN
  - Connectivity

### Manage Tags

Click **Manage Tags**.

Create tags for cross-category organization:

- Password Reset
- Installation
- Configuration
- Error Message
- Performance

## Article Workflow

Configure the article lifecycle and approval process.

Click **Configure Article Workflow**.

Define workflow stages:

**Draft**: Article being created or edited.

**Review**: Article submitted for review.

**Approved**: Article approved for publication.

**Published**: Article live and available.

**Archived**: Old article retained for reference.

**Retired**: Article no longer valid, hidden from view.

Configure approval chain:

- Define reviewers and approvers
- Set approval criteria
- Configure notification triggers

Click **Save**.

## Article Access Control

Configure who can view and edit articles.

Click **Manage Permissions**.

For each article category, set:

**View Permissions**:

- Public (everyone)
- Authenticated users
- Specific roles or groups
- Internal only

**Edit Permissions**:

- Knowledge managers
- Subject matter experts
- Specific users or groups

**Approval Permissions**:

- Content approvers
- Manager approval required

Click **Save**.

## Search Configuration

Configure knowledge base search functionality.

Click **Configure Search**.

**Search Options**:

- **Full-text search**: Search entire article content
- **Keyword search**: Search titles and tags only
- **Fuzzy matching**: Find similar terms
- **Search suggestions**: Auto-complete search terms

**Search Results**:

- **Relevance ranking**: Order by relevance score
- **Popularity ranking**: Factor in view count and ratings
- **Recency**: Prioritize recently updated articles
- **Results per page**: Number of results to display

Click **Save**.

## Article Metrics and Analytics

Configure analytics tracking for knowledge articles.

Click **Configure Analytics**.

Track the following metrics:

- **View Count**: Number of times article viewed
- **Helpfulness Rating**: User ratings (1-5 stars)
- **Search Rank**: How often found in searches
- **Linked from Tickets**: Times referenced in tickets
- **Feedback Comments**: User comments and suggestions

Set up reports:

- Most viewed articles
- Highest rated articles
- Articles needing updates
- Popular search terms with no results

Click **Save**.

## Integration Settings

Configure knowledge base integration with ticketing.

Click **Configure Integration**.

**Ticket Integration**:

- **Suggest Articles**: Auto-suggest relevant articles when creating tickets
- **Link Articles**: Allow linking articles to tickets
- **Article from Ticket**: Create knowledge articles from resolved tickets
- **Article Effectiveness**: Track if linked articles resolved tickets

**Portal Integration**:

- **Featured Articles**: Display popular articles on portal home
- **Recent Articles**: Show recently published content
- **Search Widget**: Embed search in portal

Click **Save**.
