# Mark as Knowledge

Convert incidents, solutions, and other records into knowledge base articles for future reference.

## Overview

The Mark as Knowledge function allows you to transform resolved incidents, proven solutions, and valuable information into knowledge base articles that can be searched and reused by your team, reducing resolution time and improving service quality.

## Marking Records as Knowledge

### Basic Process
1. Open a resolved incident or record with valuable information
2. Click the **Mark as Knowledge** button or menu option
3. Review auto-populated content
4. Edit and enhance the knowledge article
5. Categorize and tag appropriately
6. Click **Publish** to add to knowledge base

### Eligible Records
Can be converted to knowledge:
- Resolved incidents with documented solutions
- Problem records with root cause analysis
- Service requests with reusable procedures
- Change records with implementation guides
- Custom records (if configured)

## Knowledge Article Components

### Auto-Populated Fields
When marking as knowledge, these fields auto-fill:
- **Title**: From incident/record summary
- **Description**: From incident description
- **Solution**: From resolution notes
- **Tags**: From categories and keywords
- **Related Items**: Linked records and CIs

### Additional Information
Enhance the article by adding:
- Detailed symptoms
- Step-by-step solutions
- Screenshots and diagrams
- Related articles
- Frequently asked questions
- Preventive measures

## Categorization

### Knowledge Categories
Organize articles into categories:
- Hardware Issues
- Software Problems
- Network Connectivity
- Security and Access
- Procedures and How-To
- Best Practices
- Custom categories

### Tagging
Add relevant tags for searchability:
- Product names
- Error codes
- Symptom keywords
- Technology areas
- User groups
- Priority levels

## Article Quality

### Quality Guidelines
Ensure high-quality knowledge articles:
- Clear, concise language
- Step-by-step instructions
- Accurate technical details
- Proper formatting
- Relevant screenshots
- Complete solution path

### Review Process
- Articles may require approval before publishing
- Designated reviewers check quality
- Feedback provided for improvements
- Approval workflow configurable

- User self-service portal

## Best Practices

### Before Marking as Knowledge
- Verify solution effectiveness
- Ensure incident is fully resolved
- Document all troubleshooting steps
- Include relevant screenshots
- Remove sensitive information
- Check for existing similar articles

### Writing Guidelines
- Use clear, professional language
- Write for your target audience
- Include complete information
- Format consistently
- Add relevant images
- Link to related resources
- Test instructions before publishing

### Maintenance
- Review articles regularly (quarterly)
- Update with new information
- Retire outdated articles
- Consolidate duplicate articles
- Monitor usage and feedback
- Respond to user comments
