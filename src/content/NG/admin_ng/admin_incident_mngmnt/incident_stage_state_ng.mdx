# Incident Stage and State

Use this function to manage incident lifecycle stages and their associated states.

The following stages and states are pre-configured. These can be modified to match your organization's incident management process.

| Order | Incident Stage | Incident States | Description |
| 1 | New | - New<br>- In Progress<br>- Pending Info | Initial stage when an incident is reported. |
| 2 | Investigation | - Assigned<br>- In Progress<br>- Pending<br>- On Hold | Stage when the incident is being investigated. |
| 3 | Resolution | - In Progress<br>- Resolved<br>- Pending Validation | Stage when a resolution is being implemented. |
| 4 | Closed | - Closed<br>- Cancelled | Final stage when the incident is closed. |

In the main window, click **Admin > Incident Management > Incident Stage and State**. The Incident Stage and State window displays.

## Stage Functions

### New Incident Stage

From the *Select Actions* drop-down list, choose **New Incident Stage**. The New Incident Stage window displays.

Enter a *Name* and *Description*.

In the *Stage Order* field, click the drop-down list and select the order for this stage.

Specify if this stage should trigger any automated actions:

- Send notifications
- Update SLA timers
- Change assignment

Click **Add**.

### Edit Incident Stage

Navigate to the window containing the stage record to edit.

Click the line containing the stage. The stage window displays.

Make the necessary modifications.

Click **Save**.

### Delete Incident Stage

Click the line that contains the stage to delete.

From the *Select Actions* drop-down list, choose **Delete**.

## State Functions

### New Incident State

Click the line item containing the stage to add a corresponding state(s).

In the new window, click the **State** tab.

Click **New Incident State**.

Enter a *Name*, *Description* and *Action Name*.

Configure state properties:

**Notify**: Click **Add** to select users or roles to notify when an incident enters this state.

**Approver Roles**: Click **Add** to select roles that can approve transitions from this state (if approval is required).

**SLA Pause**: Check if SLA timers should pause in this state.

**Allow Direct Closure**: Check if incidents can be closed directly from this state.

When all selections/entries are made, click **Save**.

### Edit Incident State

Select the stage containing the state.

Click the **State** tab.

Click the line containing the state to edit.

Make the necessary modifications.

Click **Save**.

### Delete Incident State

Select the stage containing the state.

Click the **State** tab.

Select the state to delete.

From the *Select Actions* drop-down list, choose **Delete**.

## State Transitions

Configure which state transitions are allowed:

Click **Manage Transitions**.

For each state, define:

- Allowed next states
- Required fields for transition
- Approval requirements
- Notification triggers

Click **Save**.
