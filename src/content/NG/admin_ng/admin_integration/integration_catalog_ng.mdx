# Integration Catalog

Use this function to browse, discover, and install pre-built integrations with popular third-party applications and services.

The Integration Catalog provides ready-to-use connectors that simplify integration setup and reduce implementation time.

In the main window, click **Admin > Integration > Integration Catalog**. The Integration Catalog window displays.

**Use Cases**:

- Incident management synchronization

- Notification delivery
- Reporting and analytics

**Supported Versions**:

- Compatible with application versions
- Virima version requirements

### Requirements Tab

**Prerequisites**:

- Account/license in target application
- Admin access to both systems
- Network connectivity requirements
- Firewall rules needed

**Permissions Required**:

In target application:

- API access enabled
- Specific permissions or scopes
- Admin or integration user account

In Virima:

- Integration administrator role
- Module access permissions

**Technical Requirements**:

- Minimum Virima version
- Internet connectivity
- IP whitelist requirements

### Setup Tab

**Configuration Steps**:

Step-by-step setup instructions:

1. **Prepare Target System**: Enable API access
2. **Generate Credentials**: API key or OAuth
3. **Configure in Virima**: Enter connection details
4. **Map Fields**: Configure field mappings
5. **Test Connection**: Verify connectivity
6. **Enable Sync**: Activate integration

**Estimated Setup Time**: 30 minutes

**Configuration Guide**: Link to detailed documentation

## Install Integration

Click **Install** button.

### Installation Process

**Step 1: Review Requirements**

Review and confirm:

- Prerequisites met
- Permissions available
- Technical requirements satisfied

Click **Continue**.

**Step 2: Accept Terms**

- Read integration terms of service
- Accept data sharing agreement
- Acknowledge support policies

Click **Accept and Continue**.

**Step 3: Configure Connection**

**Authentication Method**:

- API Key
- OAuth 2.0
- Username/Password
- Certificate

**Connection Details**:

- **Instance URL**: https://your-instance.example.com
- **API Key**: Enter your API key
- **Username**: (if applicable)
- **Password**: (if applicable)

**Advanced Settings**:

- API version
- Timeout settings
- Proxy configuration
- SSL verification

Click **Test Connection**.

Connection successful ✓

Click **Continue**.

**Step 4: Configure Sync Settings**

**Sync Direction**:

- One-way: Virima → Target
- One-way: Target → Virima
- Two-way: Bidirectional

**Sync Frequency**:

- Real-time (event-driven)
- Every 5 minutes
- Every 15 minutes
- Hourly
- Daily
- Manual only

**Sync Scope**:

- All records
- Filtered records only
- Specific categories
- Date range

**Conflict Resolution**:

- Virima wins
- Target system wins
- Most recent update wins
- Manual review required

Click **Continue**.

**Step 5: Field Mapping**

Map fields between Virima and target system:

| Virima Field | Target Field | Transform |
|--------------|--------------|-----------|
| Incident Number | Ticket ID | Direct |
| Priority | Severity | Value mapping |
| Status | State | Value mapping |
| Description | Summary | Direct |
| Assigned To | Owner | User mapping |

Configure value mappings:

**Priority Mapping**:

| Virima | Target |
|--------|--------|
| Critical | P1 |
| High | P2 |
| Medium | P3 |
| Low | P4 |

Click **Continue**.

**Step 6: Review and Activate**

Review configuration summary:

- Connection details
- Sync settings
- Field mappings
- Estimated impact

**Initial Sync Options**:

- Skip initial sync (sync new records only)
- Sync last 30 days
- Sync last 90 days
- Sync all historical data

Click **Activate Integration**.

**Integration Activated Successfully** ✓

View integration in Active Integrations.

## Update Integration

For installed integrations:

Click **Update Available** when new version is released.

**Update Information**:

- Version: 2.0.5 → 2.1.0
- Release date
- What's new
- Bug fixes
- Breaking changes

**Backup Configuration**: Auto-backup before update

Click **Update Now**.

**Update Process**:

1. Backup current configuration
2. Pause active sync
3. Install new version
4. Migrate configuration
5. Resume sync
6. Verify operation
