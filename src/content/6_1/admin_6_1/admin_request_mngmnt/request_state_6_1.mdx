# Request State

Use this function to add a new state that defines status of the request being handled by an Assignee.

In the main window, click **Admin > Request Management > Request State**. The Request State window displays.

New Change Stage

From the *Select Actions* drop-down list, choose **New Request State**. The New Change Stage window displays.

Enter a *Name* and *Description*.

In the *State* field, click the drop-down list and select the applicable stage order.

Click **Add**.

Edit Request State Approval State

Use this function to define a state for raised requests by the application users.

Open an existing state.

Click the **Approval State** tab. The Approval State window displays.

! !

Click **New Approval State**. The Approval State tab updates.

Click **New**. The Approval State tab updates.

Enter a *Name*, *Description* and *Action Name*.

In the *Notify* field, click **Add**, then search for and select the applicable user, role, group and/or department to notify by email on an approvals update.

In the *Notify* field, click the **plus + sign**, then search for and select the applicable user, role, group and/or department to notify by email on an approvals update.

In the *Approver Roles* field, click **Add**. The Approver dialog box displays.

Search for and select the applicable role(s) with permissions to approve the defined state. Then click **Add**.

In the *Approver Roles* field, click the **plus + sign**. The Approver dialog box displays.

Search for and select the applicable role(s) with permissions to approve the defined state. Then click **Add**.

When all selections/entries are made, click **Add**.
