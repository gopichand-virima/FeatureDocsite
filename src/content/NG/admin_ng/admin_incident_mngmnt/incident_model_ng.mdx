# Incident Model

Use this function to configure and populate incident templates that can be reused for defining incident requests. These models can be used for commonly occurring incidents to skip or reduce form completion time.

In the main window, click **Admin > Incident Management > Incident Model**. The Incident Model window displays.

## New Incident Model

From the *Select Actions* drop-down list, choose **New Incident Model**. The New Incident Model window displays.

Enter the incident details that should be pre-populated:

- **Name**: Template name
- **Description**: Detailed description of the incident type
- **Category**: Select incident category
- **Subcategory**: Select incident subcategory
- **Priority**: Set default priority
- **Impact**: Set default impact level
- **Urgency**: Set default urgency level
- **Assignment Group**: Default assignment group
- **Assignee**: Default assignee

Configure additional pre-filled fields as needed.

When all selections/entries are made, click **Add**.

## Edit Incident Model

Navigate to the window containing the record to edit.

Click the line containing the record. The applicable window displays.

Make the necessary modifications.

Click **Save**.

## Modify Properties

While viewing an incident model, click **Modify Properties** from the *Select Actions* menu. The Modify Properties window displays.

Make the necessary changes to the available properties.

When all modifications are complete, click **Save**.

## Delete Incident Model

Click the line that contains the item to delete.

From the *Select Actions* drop-down list, choose **Delete**.

## Import Incident Models

To import incident models from a file:

From the *Select Actions* drop-down list, choose **Import**.

Select the import file format (CSV, Excel, or JSON).

Upload the file.

Map the columns to incident model fields.

Click **Import**.
