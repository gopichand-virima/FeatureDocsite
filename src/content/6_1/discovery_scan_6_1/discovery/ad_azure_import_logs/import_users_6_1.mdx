# Import Users

While importing users into Virima, the system checks if the user already exists. If not, Virima creates a new user.

For consecutive AD imports, the value of a user gets updated if the value has been modified in the AD.

After configuring AD, follow these steps:

1.  Click the Import icon located to the right of the selected entry.

    <span class="image placeholder" original-image-src="../../Resources/Images/Discovery/AD-User-Import-Logs/AD%20User%20Import%20Logs_1_6.png" original-image-title=""></span>

<!-- -->

2.  The <span class="Strong" style="font-weight: bold;">AD Import</span> dialog box displays.

3.  From the <span class="Strong" style="font-weight: bold;">Type of Import</span> drop-down list, select an option.

    <span class="image placeholder" original-image-src="../../Resources/Images/Discovery/AD-User-Import-Logs/AD%20User%20Import%20Logs_1_7.png" original-image-title=""></span>

<!-- -->

4.  Select the other options, as necessary.

    1.  <span class="Strong" style="font-weight: bold;">Reflect AD Account Status</span> – Enable to match user login status with Azure AD.

    2.  <span class="Strong" style="font-weight: bold;">Activate User Account</span> – Enable to make imported users active by default.

    3.  <span class="Strong" style="font-weight: bold;">Send Activation Notification</span> – Enable to email users upon account activation.

5.  When all selections/entries are made, click <span class="Strong" style="font-weight: bold;">Import</span>.

6.  A confirmation message displays. Click <span class="Strong" style="font-weight: bold;">OK</span>.
