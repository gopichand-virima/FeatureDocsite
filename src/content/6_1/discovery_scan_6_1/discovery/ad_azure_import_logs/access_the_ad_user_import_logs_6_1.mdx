# Accessing the Imported Assets

1.  From the left navigation panel, navigate to <span class="Strong" style="font-weight: bold;">Discovery Scan</span>. Click <span class="Strong" style="font-weight: bold;">Imported Assets.</span>

    <span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Imported-Assets/Imported%20Assets.png" original-image-title=""></span>

    </div>

2.  The Imported Assets page opens.

    <div class="step-media">

    <span class="image placeholder" original-image-src="../../Resources/Images/Discovery/Imported-Assets/Imported%20Assets_1.png" original-image-title=""></span>

    </div>

<span class="Strong" style="font-weight: bold;">Column Details:</span>

Displays imported asset details in tabular format. Each row represents a imported asset with columns such as:

- <span class="Strong" style="font-weight: bold;">Blueprint</span>: The type of asset (e.g., Windows Server, Linux Server, IIS Server).

- <span class="Strong" style="font-weight: bold;">CI</span>: Status of the configuration item (e.g., <span class="Emphasis">New</span>, or linked CI ID such as <span class="Emphasis">AST004534</span>).

  - <span class="Strong" style="font-weight: bold;">New</span>: Asset imported into the system, not yet linked to a CMDB record.

  - <span class="Strong" style="font-weight: bold;">Linked CI</span>: Displays a unique CI ID (e.g., AST004534). The asset is linked to an existing Configuration Item in the CMDB.

- <span class="Strong" style="font-weight: bold;">Host Name</span>: The hostname of the device or server.

- <span class="Strong" style="font-weight: bold;">IP Address</span>: The IP address of the asset.

- <span class="Strong" style="font-weight: bold;">Operating System</span>: The Operating System of the asset.

- <span class="Strong" style="font-weight: bold;">Location</span>: The specified location of the asset.

- <span class="Strong" style="font-weight: bold;">Re-scan</span><span class="inline-comment-marker"> icon – Opens the rescan dialog box where you can specify IP address, client, and type of scan to revalidate asset information.</span>

  - <span class="inline-comment-marker">A new scan run will be listed on the Recent Scans page; refresh the page to view the latest status updates.</span>

  - Scanned results will be displayed in Discovered items. See discovered items PRD for rest of the flow.

  - Record in Imported Assets will not be updated.

- <span class="Strong" style="font-weight: bold;">Records per Page Dropdown</span>: Allows users to select how many records are displayed at once (e.g., 10, 25, 100).

# Select Actions Dropdown

The <span class="Strong" style="font-weight: bold;">Select Actions</span> menu provides administrators with operations they can perform on imported assets:

- <span class="Strong" style="font-weight: bold;">Delete</span> – Removes the selected asset record(s) from the Imported Assets list.

- <span class="Strong" style="font-weight: bold;">Move to CMDB</span> – To move the asset to the Configuration Management Database after confirmation.

- <span class="Strong" style="font-weight: bold;">Re-scan</span> – Opens the rescan dialog box where you can specify IP address, client, and type of scan to revalidate asset information.
