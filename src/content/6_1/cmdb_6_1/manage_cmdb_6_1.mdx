# Managing CMDB

The <span class="Strong">"Select Actions"</span> dropdown in the CMDB allows users to perform various operations on selected Configuration Items (CIs).

These actions are available only after items are added to the CMDB (not while in the discovered state).

<span class="image placeholder" original-image-src="../Resources/Images/CMDB%20-%20Left-Nav/Configuration%20Management%20Database_2.png" original-image-title=""></span>

This menu provides tools to audit, update, process, or manage data related to selected CIs. Common actions include:

- <span class="Strong" style="font-weight: bold;">Audits</span>  
  View the complete history of changes made to a CI for accountability and compliance.

- <span class="Strong">Change Attributes</span>  
  Update CI details such as <span class="Strong">status, owner, or location</span>.

- <span class="Strong">Copy to External Tools</span>

  - <span class="Strong">Copy to Ivanti</span> – Export selected CI data to Ivanti.

  - <span class="Strong">Copy to Jira</span> – Send CI records to Jira for tracking.

  - <span class="Strong">Copy to ServiceNow</span> – Sync CI data with ServiceNow ITSM.

- <span class="Strong">Delete / Export / New</span>

  - <span class="Strong">Delete</span> – Remove a CI record from the CMDB.

  - <span class="Strong">Export</span> – Download CI data for external reporting or analysis.

  - <span class="Strong">New</span> – Add a new CI record manually.

- <span class="Strong">Generate Installed Software Report</span>  
  Create a report listing software installed on the selected CI.

- <span class="Strong">Process ADM (Application Dependency Mapping)</span>  
  Automatically discover and map <span class="Strong">application dependencies and relationships</span>.

- <span class="Strong">Process Available Patch Report</span>  
  View pending <span class="Strong">updates and patches</span> available for the CI.

- <span class="Strong">Process Cloud Hierarchy / DevOps / vCenter</span>  
  Organize and map virtual and cloud-based infrastructure, including DevOps and VMware vCenter hierarchies.

- <span class="Strong">Process Missing Components</span>  
  Identify and report <span class="Strong">expected but missing CI components</span>.

- <span class="Strong">Process Network Connections</span>  
  Map and analyze <span class="Strong">network links and communication paths</span> related to the CI.

- <span class="Strong" style="font-weight: bold;">Process Software Installation</span>  
  Track, verify, and update <span class="Strong">software installations</span> for the selected CI.
