# Monitoring Profile: Action Details

Use this function to specify what how the system handles changes found for an item being monitored. For example, if a configuration file is being monitored, and a change is detected, the action may be to create an incident report or send a notification to a user.

 -- ----- -----------------------------------------------------------------------------------
 1. In the main window, click **Admin \> Discovery \> Discovery Monitoring Profile**.
 -- ----- -----------------------------------------------------------------------------------

 -- ----- -------------------------------------------------------------------------------
 2. From the *Select Actions* drop-down list, choose **New**. The window updates.
 -- ----- -------------------------------------------------------------------------------

 -- ----- ---------------------------------------------
 3. Navigate to the **Action Details** section.
 -- ----- ---------------------------------------------

::: {.Ind50
![discovery mon prof action details old](../Resources/Images/Admin - Discovery/discovery-mon-prof-action-details-old.PNG)

 -- ----- ---------------------------------------------------------------------------------------------
 4. In the *Target Module* field, click the drop-down list and select the required ITSM module.
 -- ----- ---------------------------------------------------------------------------------------------

 -- ----- -------------------------------------------------------------------------------------------
 5. In the *Create From Model* field, select the template to be used for the selected module.
 -- ----- -------------------------------------------------------------------------------------------

 -- ----- ----------------------------------------------------------------------------------------------------------------------------------
 6. In the *Property* fields, click the drop-down list, select the applicable property, and then select a matching value (required).
 -- ----- ----------------------------------------------------------------------------------------------------------------------------------

+---+------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| Column 1 | Depending on the type of property, an additional step may be required. For example, if the Property **Assigned User from Group** is selected, a user must be selected in the *User* dialog box. |
| --- | --- |
| | []{.image .placeholder original-image-src="../Resources/Images/Admin%20-%20Discovery/user-dialog-box-old_232x93.png" original-image-title="" |
+---+------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+

 -- ----- -------------------------------------------------------------
 7. To add a new property field row, click the **plus + sign**.
 -- ----- -------------------------------------------------------------

 -- ----- -------------------------------------------------------------
 8. To remove a property field row, click the **minus - sign**.
 -- ----- -------------------------------------------------------------

 -- ----- ------------------------------------------------------
 9. When all selections/entries are made, click **Add**.
 -- ----- ------------------------------------------------------

 --- --------------------------------------------------------------------------------------------------------------------------------------------------------------------
   If required information is missing (such as a property value), an Alert Message displays.  Resolve the error and then attempt to add or save the information again
 --- --------------------------------------------------------------------------------------------------------------------------------------------------------------------
