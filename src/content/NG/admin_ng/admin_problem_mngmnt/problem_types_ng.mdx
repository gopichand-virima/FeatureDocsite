# Problem Types

Use this function to configure different types of problems with customized color codes for visual identification.

Problem types help categorize problems based on their nature, source, or handling requirements. Each type can be assigned a unique color code for easy visual recognition in lists and dashboards.

In the main window, click **Admin > Problem Management > Problem Types**. The Problem Type window displays.

## New Problem Type

From the *Select Actions* drop-down list, choose **New Problem Type**. The New Problem Type window displays.

Enter a *Name* and *Description* for this type.

In the *Color* field, click on the color and choose a color from the picker OR type a 6-digit hex code.

Configure additional properties:

**Default Priority**: Set a default priority for this problem type (optional).

**Default Category**: Set a default category for this problem type (optional).

**Requires RCA**: Check if problems of this type require formal Root Cause Analysis.

**Auto-Assign to Group**: Select a default assignment group (optional).

**Create Known Error**: Automatically create known error record when problem is confirmed.

Click **Add**.

## Common Problem Types

Examples of problem types you might configure:

- **Infrastructure**: Hardware, network, or system infrastructure problems
- **Application**: Software application defects or issues
- **Performance**: System or application performance degradation
- **Security**: Security vulnerabilities or weaknesses
- **Data**: Data integrity or quality issues
- **Process**: Business process failures
- **Integration**: System integration failures
- **Capacity**: Capacity or scalability problems
- **Availability**: Service availability problems
- **Configuration**: Configuration errors causing issues

## Edit Problem Type

Navigate to the window containing the record to edit.

Click the line containing the record. The applicable window displays.

Make the necessary modifications.

Click **Save**.

## Delete Problem Type

Deleting is a permanent action and cannot be undone. Ensure no problems are currently using this type before deletion.

Click the line that contains the item to delete.

From the *Select Actions* drop-down list, choose **Delete**.

## Type-Based Routing

To configure automatic routing based on problem type:

Select a problem type.

Click **Configure Routing**.

Define routing rules:

- **Assignment Group**: Auto-assign to specific group
- **Assignee**: Auto-assign to specific user
- **Workflow**: Attach specific workflow
- **Notification Template**: Use custom notification

Click **Save**.
