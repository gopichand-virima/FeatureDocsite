# Role Management

Use this function to create and manage roles that define user permissions and access levels.

Role-based access control (RBAC) provides centralized permission management, ensuring users have appropriate access based on their job functions.

In the main window, click **Admin > Security > Role Management**. The Role Management window displays.

## Role List Overview

View all roles in the system:

**Role Columns**:

- **Role Name**: Name of the role
- **Description**: Role description
- **Type**: System or Custom
- **Users**: Number of assigned users
- **Permissions**: Permission count
- **Active**: Enabled status
- **Last Modified**: Last update date

**System Roles** (pre-built):

- System Administrator
- Service Desk Manager
- Service Desk Agent
- Asset Manager
- Change Manager
- Problem Manager
- Knowledge Manager
- End User

**Custom Roles**:

Organization-specific roles created by administrators.

## Create New Role

From the *Select Actions* drop-down list, choose **New Role**. The New Role window displays.

### Basic Information

**Role Details**:

- **Role Name**: Descriptive name (e.g., "Network Support Specialist")
- **Role ID**: Auto-generated or custom
- **Description**: Detailed role description
- **Role Type**: Custom role

**Hierarchy**:

- **Parent Role**: Inherit permissions from parent
- **Role Level**: Administrative level (1-10)

**Status**:

- **Active**: ☑ Enable this role

### Permission Configuration

Define what this role can access and do.

Click **Configure Permissions**.

#### Module Permissions

Grant access to specific modules:

**Service Desk**:

- ☑ Incidents
- ☑ Service Requests
- ☑ Problems
- ☑ Changes
- ☑ Knowledge Base

**Asset Management**:

- ☑ Hardware Assets
- ☑ Software Assets
- ☐ Contracts
- ☐ Licenses

**Discovery**:

- ☐ Discovery Jobs
- ☐ Discovery Schedules
- ☐ Discovery Credentials

**Admin Functions**:

- ☐ User Management
- ☐ System Configuration
- ☐ Workflow Designer
- ☐ Reports

#### Operation Permissions

For each module, define allowed operations:

**Incidents**:

- **Create**: ☑ Can create incidents
- **Read**: ☑ Can view incidents
- **Update**: ☑ Can modify incidents
- **Delete**: ☐ Cannot delete incidents
- **Export**: ☑ Can export data
- **Import**: ☐ Cannot import data

**Permission Levels**:

**All Records**: Full access to all records

**Group Records**: Access to assignment group records only

**Assigned Records**: Access to personally assigned records only

**Created Records**: Access to self-created records only

**No Access**: No permission

#### Field-Level Permissions

Control access to specific fields:

**Incidents - Editable Fields**:

- ☑ Subject
- ☑ Description
- ☑ Priority
- ☑ Category
- ☑ Status
- ☐ Cost (read-only)
- ☐ SLA (read-only)
- ☐ Assignment Group (restricted)

**Field Security**:

- **Full Access**: Read and write
- **Read Only**: View but cannot modify
- **Hidden**: Field not visible
- **Restricted**: Conditional access

#### Functional Permissions

Control specific functions:

**Ticket Management**:

- ☑ Reassign tickets
- ☑ Change priority
- ☑ Add comments
- ☑ Add attachments
- ☐ Close tickets
- ☐ Reopen closed tickets
- ☐ Delete tickets
- ☑ Merge tickets
- ☐ Bulk update tickets

**Approval Functions**:

- ☑ Submit for approval
- ☐ Approve requests
- ☐ Reject requests
- ☐ Delegate approvals

**Knowledge Base**:

- ☑ Search knowledge articles
- ☑ View articles
- ☐ Create articles
- ☐ Edit articles
- ☐ Publish articles
- ☐ Retire articles

**Reporting**:

- ☑ View standard reports
- ☐ Create custom reports
- ☐ Modify reports
- ☑ Export reports
- ☐ Schedule reports

#### Administrative Permissions

Control admin functions:

**User Administration**:

- ☐ Create users
- ☐ Modify users
- ☐ Deactivate users
- ☐ Reset passwords
- ☐ Assign roles

**Configuration**:

- ☐ Modify system settings
- ☐ Configure workflows
- ☐ Manage templates
- ☐ Configure integrations

**Security**:

- ☐ Manage roles
- ☐ Manage permissions
- ☐ View audit logs
- ☐ Configure security settings

### Data Access Controls

Define what data this role can access:

**Scope Restrictions**:

**Department Scope**:

- All departments
- Specific departments only
- User's department only

**Location Scope**:

- All locations
- Specific locations only
- User's location only

**Category Scope**:

- All categories
- Specific categories only
- Assigned categories only

**Priority Scope**:

- All priorities
- P1-P3 only (exclude low priority)
- Based on role level

### Conditional Access

Define conditions for access:

**Business Hours**:

- Access anytime
- Business hours only: Monday-Friday, 8 AM - 6 PM
- Extended hours: Monday-Saturday, 7 AM - 8 PM

**IP Restrictions**:

- Allow from any IP
- Corporate network only
- Specific IP ranges

**Device Restrictions**:

- Any device
- Managed devices only
- Desktop only (no mobile)

### Role Delegation

Configure delegation capabilities:

**Can Delegate To**:

- ☑ Team members
- ☐ Manager
- ☐ Any user

**Delegation Scope**:

- Specific functions
- Time-limited delegation
- Approval required

Click **Save** to create the role.

## Common Role Configurations

### Service Desk Agent Role

**Permissions**:

- Create, view, update incidents and requests
- View assigned and group tickets
- Reassign within group
- Add comments and attachments
- Change status (except closed)
- View knowledge base
- Access service catalog
- View standard reports

**Restrictions**:

- Cannot delete tickets
- Cannot change SLA
- Cannot access admin functions
- Cannot view other groups' tickets

### Service Desk Manager Role

**Permissions**:

- All agent permissions plus:
- View all tickets (all groups)
- Reassign to any group
- Close and reopen tickets
- Approve low-risk changes
- Create and modify canned responses
- View all reports
- Bulk update tickets
- Manage team schedules

**Restrictions**:

- Cannot access system configuration
- Cannot modify workflows
- Limited user management

### Asset Manager Role

**Permissions**:

- Full access to asset management
- Create, view, update, delete assets
- Manage contracts and licenses
- Run asset discovery
- Generate asset reports
- Import/export asset data
- Configure asset types
- Manage asset workflows

**Restrictions**:

- Read-only access to service desk
- No user administration
- No system configuration

### Change Manager Role

**Permissions**:

- Full access to change management
- Create, view, update changes
- Schedule changes
- Approve changes (based on risk)
- Manage CAB meetings
- View change calendar
- Generate change reports
- Manage change workflows

**Additional Access**:

- View related incidents and problems
- View affected assets
- Create linked tickets

## Edit Role

Navigate to the window containing the role.

Click the line containing the role.

Make modifications to:

- Basic information
- Permissions
- Data access controls
- Conditional access

Click **Save**.

## Clone Role

Create a new role based on existing:

Select a role.

From the *Select Actions* drop-down list, choose **Clone Role**.

Enter new role name.

Modify permissions as needed.

Click **Save**.

## Role Comparison

Compare permissions between roles:

Click **Compare Roles**.

Select 2-3 roles to compare.

View side-by-side permission comparison:

| Permission | Agent | Manager | Admin |
|------------|-------|---------|-------|
| View Incidents | ✓ | ✓ | ✓ |
| Delete Incidents | ✗ | ✗ | ✓ |
| Manage Users | ✗ | Limited | ✓ |
| System Config | ✗ | ✗ | ✓ |

Identify permission differences quickly.

## Assign Role to Users

Assign role to one or more users:

Select a role.

Click **Assign to Users**.

Search and select users.

**Assignment Options**:

- **Primary Role**: Replace existing primary role
- **Additional Role**: Add as secondary role
- **Effective Date**: Immediate or scheduled

Click **Assign**.

## Role Analytics

View role usage and insights:

Select a role.

Click **View Analytics**.

**Role Metrics**:

- Total users: 47
- Active users: 45
- Inactive users: 2
- Average login frequency
- Feature usage by role

**Permission Usage**:

- Most used permissions
- Unused permissions
- Permission requests denied

**Insights**:

- Identify over-privileged users
- Identify under-utilized permissions
- Optimize role definition

## Permission Testing

Test permissions before assigning:

Select a role.

Click **Test Permissions**.

**Simulation**:

- Select a test user
- Apply role temporarily
- Test various actions
- Verify expected behavior

Review results and adjust permissions.

## Role Inheritance

Configure role hierarchy:

**Parent-Child Relationships**:

```
System Administrator (all permissions)
  ├── Service Desk Manager
  │   ├── Service Desk Agent
  │   └── Service Desk Specialist
  ├── Asset Manager
  └── Change Manager
      └── Change Coordinator
```

**Inheritance Rules**:

- Child inherits parent permissions
- Child can have additional permissions
- Cannot remove parent permissions
- Override specific permissions if needed

## Activate/Deactivate Role

**Deactivate Role**:

Select active role.

From the *Select Actions* drop-down list, choose **Deactivate**.

Note: Users with this role will lose associated permissions.

**Reactivate Role**:

Select inactive role.

From the *Select Actions* drop-down list, choose **Activate**.

## Delete Role

Ensure no users are assigned to this role.

Select role to delete.

From the *Select Actions* drop-down list, choose **Delete**.

**Deletion Impact**:

- Users with this role will need reassignment
- Custom permissions will be lost
- Historical data retained

Confirm deletion.

## Role Templates

Use pre-built role templates:

Click **Browse Role Templates**.

**Available Templates**:

- ITIL-aligned roles
- Industry-specific roles
- Compliance-focused roles
- Department-specific roles

Select template and customize.
